## IAM 유저 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (107·108번과 동일).

```
                    IAM 유저 (aws_iam_user)
                   ↗                    ↗
    유저 그룹 멤버십                    로그인 프로필
(aws_iam_user_group_membership)   (aws_iam_user_login_profile)
         ↗                                  ↗
    IAM 그룹 (aws_iam_group)          PGP 공개키 (파일)
```

| 구성 | Terraform | 종류 | 역할 |
|------|-----------|------|------|
| **IAM 유저** | `aws_iam_user` | **resource** | 유저 **본체** |
| **그룹 멤버십** | `aws_iam_user_group_membership` | **resource** | 유저를 **107 그룹**에 소속 |
| **로그인 프로필** | `aws_iam_user_login_profile` | **resource** | **AWS 콘솔** 로그인 비밀번호 |
| **PGP 공개키** | 로컬 파일 | — | `pgp_key` 로 **비밀번호 암호화** (108) |

- 유저 **권한**은 유저에 직접 붙이지 않고, **그룹 정책**(107)을 **상속**받는 구조가 일반적이다.

## `aws_iam_user` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** * | string | IAM **유저 이름** (필수) |
| **`force_destroy`** | bool | Terraform 관리 **외** 인증 정보가 있어도 **강제 삭제**할지 |
| **`tags`** | object | **태그** |

```hcl
resource "aws_iam_user" "developer" {
  name          = "${var.project}-${var.environment}-developer-iam-user"
  force_destroy = true

  tags = {
    Name = "${var.project}-${var.environment}-developer-iam-user"
  }
}
```

| `force_destroy` | 의미 |
|-----------------|------|
| **`false`** (기본) | 로그인 프로필·액세스 키 등이 남아 있으면 `terraform destroy` **실패** |
| **`true`** | 유저 삭제 시 **관련 인증 정보도 함께 제거** 시도 |

## `aws_iam_user_group_membership`

107에서 만든 **개발용 그룹**에 유저를 **소속**시킨다.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`user`** | string | IAM **유저 이름** |
| **`groups`** | list(string) | 소속시킬 **그룹 이름** 목록 |

```hcl
resource "aws_iam_user_group_membership" "developer" {
  user   = aws_iam_user.developer.name
  groups = [aws_iam_group.developers.name]
}
```

→ `Developers` 그룹의 정책(ReadOnlyAccess + 106 자작 정책 3종)이 유저에게 **적용**된다.

## `aws_iam_user_login_profile` (슬라이드)

**AWS Management Console** 로그인용 **초기 비밀번호**를 설정한다.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`user`** * | string | IAM **유저 이름** (필수) |
| **`pgp_key`** * | string | **Base64 인코딩된 PGP 공개키** (필수) |
| **`password_length`** | number | 비밀번호 **길이** (기본 **20**) |
| **`password_reset_required`** | bool | **첫 로그인 시 비밀번호 변경** 강제 (기본 **true**) |

```hcl
resource "aws_iam_user_login_profile" "developer" {
  user                    = aws_iam_user.developer.name
  pgp_key                 = filebase64("${path.module}/public.key")
  password_length         = 20
  password_reset_required = true
}
```

### `pgp_key` 와 `filebase64()` (슬라이드)

> **`filebase64()`** 를 쓰면 **바이너리 파일을 Base64 인코딩된 상태**로 읽을 수 있다.

| 방법 | 예 |
|------|-----|
| **파일에서 읽기 (권장)** | `pgp_key = filebase64("${path.module}/public.key")` |
| **문자열 직접** | export 한 공개키를 Base64 문자열로 넣기 |

사전 준비 (108):

```bash
gpg --gen-key                                    # RSA 2048, 패스프레이즈 없음
gpg -o public.key --export <NAME>                # 공개키 파일 생성
```

`public.key` 는 **버전 관리에 넣지 않는** 것이 좋다 (`.gitignore`).

## apply 후 비밀번호 확인 (108 연계)

`aws_iam_user_login_profile` 은 **`encrypted_password`** 를 출력한다. **평문 비밀번호는 state에도 평문으로 남지 않는다.**

```hcl
output "developer_encrypted_password" {
  value     = aws_iam_user_login_profile.developer.encrypted_password
  sensitive = true
}
```

복호화:

```bash
terraform output -raw developer_encrypted_password | base64 --decode | gpg -r <NAME> --decrypt
```

| 단계 | 명령 |
|------|------|
| 1 | `terraform apply` |
| 2 | `encrypted_password` 출력 확인 |
| 3 | `base64 --decode` → `gpg --decrypt` → **실제 비밀번호** |

## 실습 구성 예 (`iam_user.tf`)

```hcl
resource "aws_iam_user" "developer" {
  name          = "${var.project}-${var.environment}-developer-iam-user"
  force_destroy = true
}

resource "aws_iam_user_group_membership" "developer" {
  user   = aws_iam_user.developer.name
  groups = [aws_iam_group.developers.name]
}

resource "aws_iam_user_login_profile" "developer" {
  user    = aws_iam_user.developer.name
  pgp_key = filebase64("${path.module}/public.key")
}
```
