## IAM 유저 데이터 구조 (슬라이드)

109번에서 다룰 구조의 **PGP 관련 부분**만 미리 본다.

```
                    IAM 유저 (aws_iam_user)
                   ↗                    ↗
    유저 그룹 멤버십                    로그인 프로필
(aws_iam_user_group_membership)   (aws_iam_user_login_profile)
         ↗                                  ↗
    IAM 그룹 (aws_iam_group)          PGP 공개키 (data / 파일)
```

| 구성 | Terraform | 종류 | 역할 |
|------|-----------|------|------|
| **IAM 유저** | `aws_iam_user` | **resource** | 유저 **본체** |
| **그룹 멤버십** | `aws_iam_user_group_membership` | **resource** | 유저 ↔ 107 그룹 **소속** |
| **로그인 프로필** | `aws_iam_user_login_profile` | **resource** | **콘솔 로그인** 비밀번호 설정 |
| **PGP 공개키** | 파일 또는 `data` | **data** | 비밀번호 **암호화**에 사용 |

참조: **참조 원 → 참조 대상** (107·106번과 동일).

## IAM 유저 비밀번호와 PGP (슬라이드)

> **IAM 유저의 비밀번호는 PGP 공개키로 암호화된 것을 받는다**

| | 평문 비밀번호 | PGP 사용 |
|---|--------------|----------|
| **Terraform 출력** | state·터미널에 **노출 위험** | **`encrypted_password`** 만 저장·출력 |
| **실제 비밀번호 확인** | 그대로 보임 | **비밀키**로 **복호화**해야 함 |

흐름:

```
PGP 공개키 ──▶ AWS/Terraform이 비밀번호 암호화
                    │
                    ▼
            encrypted_password (암호문)
                    │
                    ▼
         GPG 비밀키로 복호화 → 실제 비밀번호
```

- Terraform `aws_iam_user_login_profile` 의 **`pgp_key`** 인자에 **공개키 문자열**을 넘긴다 (109).
- **`password`** 는 평문으로 직접 넣지 않고, AWS가 생성한 초기 비밀번호를 **PGP로 암호화**해 돌려준다.

## PGP란 (슬라이드)

**PGP** = **Pretty Good Privacy** (「かなりよいプライバシー」)

| 항목 | 내용 |
|------|------|
| **방식** | **공개키 암호** 방식의 **암호화·전자서명** |
| **SSL/TLS와 차이** | SSL/TLS와 달리 **키를 이용자가 직접 관리** |
| **구현체** | **GPG** (GNU Privacy Guard), **OpenPGP** 등 |

### 공개키 암호 개념

| 키 | 역할 | 공유 |
|----|------|------|
| **공개키 (Public Key)** | 데이터 **암호화** | Terraform·AWS에 **제공** (`pgp_key`) |
| **비밀키 (Secret Key)** | 데이터 **복호화** | **본인만** 보관 — 절대 공유하지 않음 |

## GPG 명령 치트시트 (슬라이드)

IAM 유저 생성에 **관련되는 명령은 3가지** (슬라이드 강조).

| 종류 | 명령 | 용도 |
|------|------|------|
| **생성** | `gpg --gen-key` | 키 페어 **생성** |
| **공개키 출력** | `gpg -o <KEY_FILE> --export <NAME>` | **공개키**를 파일로 export → `pgp_key` 에 사용 |
| **복호화** | `gpg -r <NAME> --decrypt` | 암호문을 **복호화** → 실제 비밀번호 확인 |

### 전체 치트시트 (참고)

| 종류 | 명령 |
|------|------|
| **생성** | `gpg --gen-key` |
| **목록 확인** | `gpg --list-keys` · `gpg --list-secret-keys` |
| **키 파일 출력** | `gpg -o <KEY_FILE> --export <NAME>` |
| | `gpg -o <KEY_FILE> --export-secret-key <NAME>` |
| **키 삭제** | `gpg --delete-key <NAME>` · `gpg --delete-secret-key <NAME>` |
| **암호/복호** | `gpg -r <NAME> --encrypt` · `gpg -r <NAME> --decrypt` |

### 암호/복호 주의 (슬라이드)

| 항목 | 설명 |
|------|------|
| **입력** | **표준 입력(stdin)** 에서 데이터를 받음 |
| **암호화 결과** | **바이너리 형식** |

```bash
# 암호화 예 (stdin)
echo "hello" | gpg -r <NAME> --encrypt

# 복호화 예 (stdin)
echo "<암호문>" | gpg -r <NAME> --decrypt
```

## GPG를 이용한 암호/복호 실습 (슬라이드)

강의에서 다루는 **기본 3단계**:

### 1. master 키 페어 생성

```bash
gpg --gen-key
```

| 설정 | 강의 값 |
|------|---------|
| **알고리즘** | **RSA 2048bit** |
| **패스워드** | **없음** (패스프레이즈 없이) |

### 2. 임의 문자열 암호화

```bash
echo "test-password" | gpg -r <NAME> --encrypt > encrypted.bin
```

### 3. 암호화된 데이터 복호화

```bash
gpg -r <NAME> --decrypt encrypted.bin
```

### Terraform 연동 시 (109 예고)

```bash
# 1. 키 생성
gpg --gen-key

# 2. 공개키 export → pgp_key 에 붙여 넣기
gpg -o public.key --export <NAME>
cat public.key   # 이 내용을 aws_iam_user_login_profile.pgp_key 에 사용

# 3. terraform apply 후 encrypted_password 복호화
echo "<encrypted_password>" | base64 --decode | gpg -r <NAME> --decrypt
```

> AWS/Terraform이 돌려주는 `encrypted_password` 는 **base64** 인 경우가 많아, **`base64 --decode`** 후 `gpg --decrypt` 한다 (109 실습에서 확인).

## 108번 정리

| 주제 | 핵심 |
|------|------|
| **PGP** | 공개키 암호 — IAM 초기 비밀번호를 **평문으로 노출하지 않음** |
| **Terraform** | `aws_iam_user_login_profile.pgp_key` = **공개키** |
| **GPG 3명령** | `--gen-key` · `--export` · `--decrypt` |
| **키 생성** | RSA 2048 · 패스프레이즈 없음 (강의 설정) |
| **다음** | **109** 에서 `aws_iam_user` · `login_profile` · 그룹 멤버십 작성 |

현재 `tf-user/` 에 **PGP 키·유저 리소스는 아직 없음** — 109에서 추가.
