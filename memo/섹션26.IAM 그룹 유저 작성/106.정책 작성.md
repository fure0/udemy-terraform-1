# 106. 정책 작성

섹션26은 **IAM 그룹·유저**를 Terraform으로 만든다.  
**106번**은 **자작 IAM 정책(Custom Policy)** 의 **데이터 구조·리소스·실습 요건** — **107 그룹 작성** · **109 유저 작성** 에서 이 정책을 **연결**한다.

슬라이드: **IAMポリシーのデータ構造** · **aws_iam_policy** · **aws_iam_policy_document** · **独自ポリシーの作成**

이전: **105 모듈의 정의와 이용** · 섹션25 마지막  
다음: **107 그룹 작성** · **108 PGP란**

## IAM 정책 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (64·94·97번과 동일).

```
IAM 정책 (aws_iam_policy)
         ↗
정책 문서 (data.aws_iam_policy_document)
```

| 구성 | Terraform | 종류 | 역할 |
|------|-----------|------|------|
| **IAM 정책** | `aws_iam_policy` | **resource** | AWS에 **정책 리소스** 등록 |
| **정책 문서** | `data.aws_iam_policy_document` | **data** | 권한 내용 **JSON** 생성 |

- **64번** — 신뢰 정책 → `aws_iam_role.assume_role_policy`
- **94번** — 버킷 정책 → `aws_s3_bucket_policy.policy`
- **106번** — **독립 IAM 정책** → `aws_iam_policy.policy` ← 이번 강의

## `aws_iam_policy` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** | string | IAM **정책 이름** |
| **`description`** | string | 정책 **설명** |
| **`policy`** * | string | 정책 **JSON 문자열** (필수) |

`*` = 필수 인자.

```hcl
resource "aws_iam_policy" "example" {
  name        = "example-policy"
  description = "Example custom policy"
  policy      = data.aws_iam_policy_document.example.json
}
```

- JSON을 **직접 문자열로 넣지 않고**, `data.aws_iam_policy_document`의 **`.json`** 을 넘기는 패턴이 강의 방식이다 (64·94와 동일).

## `data.aws_iam_policy_document` (슬라이드)

정책 문서 = **JSON 데이터**. HCL 블록으로 작성하면 Terraform이 JSON으로 렌더링한다.

### 슬라이드 JSON 예시 (개념)

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Action": ["s3:GetObject"],
      "Effect": "Allow",
      "Resource": "*",
      "Principal": "*"
    }
  ]
}
```

| JSON 필드 | 의미 |
|-----------|------|
| **`Version`** | 정책 언어 버전 (`2012-10-17`) |
| **`Statement`** | 권한 문장 배열 |
| **`Action`** | 허용·거부할 **API 액션** |
| **`Effect`** | `Allow` / `Deny` |
| **`Resource`** | 대상 **리소스** ARN |
| **`Principal`** | 대상 **주체** (리소스 기반 정책·신뢰 정책에서 사용) |

> **106 실습(유저·그룹 정책)** 은 **Identity-based policy** 이므로 `principals` 블록은 **보통 생략**한다. 정책이 **붙는 유저·그룹**이 주체가 된다.

### 데이터 소스 필드 요약 (슬라이드 표)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`version`** | string | 정책 버전 |
| **`policy_id`** | string | 정책 ID |
| **`statement`** | block | 정책 **본문** (여러 개 가능) |

#### `statement` 블록

| 항목 | 타입 | 설명 |
|------|------|------|
| **`sid`** | string | 문장 ID |
| **`effect`** | enum | `"Allow"` / `"Deny"` |
| **`actions`** | string[] | 액션 목록 |
| **`resources`** | string[] | 대상 리소스 ARN |
| **`principals`** | block | 주체 (`type`, `identifiers`) — 리소스·신뢰 정책용 |

#### `principals` 블록

| 항목 | 타입 | 설명 |
|------|------|------|
| **`type`** * | enum | `"AWS"`, `"Service"` 등 |
| **`identifiers`** * | string[] | ARN, 서비스 식별자 등 |

## 자작 정책 실습 요건 (슬라이드)

슬라이드 **独自ポリシーの作成** — 아래 **3가지**를 하나의 정책에 담는다.

| # | 요건 | Effect | Action |
|---|------|--------|--------|
| 1 | **빌링을 모두 거부** | `Deny` | `aws-portal:*` |
| 2 | **EC2 재기동 허용** | `Allow` | `ec2:RebootInstances` |
| 3 | **비밀번호 변경 허용** | `Allow` | `iam:ChangePassword` |

- **`statement` 블록을 3개** 두면 된다 (한 블록 = JSON `Statement` 배열의 한 항목).
- `resources = ["*"]` 로 두는 경우가 많다 (강의 슬라이드 수준).

### HCL 예시

```hcl
data "aws_iam_policy_document" "custom" {
  statement {
    effect    = "Deny"
    actions   = ["aws-portal:*"]
    resources = ["*"]
  }

  statement {
    effect    = "Allow"
    actions   = ["ec2:RebootInstances"]
    resources = ["*"]
  }

  statement {
    effect    = "Allow"
    actions   = ["iam:ChangePassword"]
    resources = ["*"]
  }
}

resource "aws_iam_policy" "custom" {
  name        = "${var.project}-${var.environment}-custom-policy"
  description = "Deny billing, allow EC2 reboot and password change"
  policy      = data.aws_iam_policy_document.custom.json
}
```

## 64·94·106 비교

| 용도 | data | 연결 대상 | `.json` 넘기는 인자 |
|------|------|-----------|---------------------|
| **신뢰 정책** (64) | `aws_iam_policy_document` | `aws_iam_role` | `assume_role_policy` |
| **버킷 정책** (94) | `aws_iam_policy_document` | `aws_s3_bucket_policy` | `policy` |
| **IAM 정책** (106) | `aws_iam_policy_document` | `aws_iam_policy` | `policy` |

공통: **`data.aws_iam_policy_document` → `.json`** 패턴.

## 이 프로젝트와의 관계

| 항목 | 현재 `terraform/iam.tf` |
|------|-------------------------|
| **`data.aws_iam_policy_document`** | **`ec2_assume_role`** (신뢰 정책, 64) |
| **`aws_iam_policy`** | **없음** — 106에서 **자작 정책** 추가 |
| **정책 연결** | Role은 **관리형 ARN** `aws_iam_role_policy_attachment` (66) |
| **그룹·유저** | **없음** — 107·109에서 추가 후 정책 **연결** |
