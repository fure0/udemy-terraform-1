## IAM 그룹 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (66·106번과 동일).

그룹에 정책을 붙이는 방법은 **두 가지**다.

### A. 관리형 정책 연결 (Managed Policy Attachment)

```
IAM 그룹 (aws_iam_group)
         ↗              ↗
그룹 정책 연결          IAM 정책
(aws_iam_group_policy_attachment)  (aws_iam_policy / AWS 관리형)
```

| 구성 | Terraform | 종류 | 역할 |
|------|-----------|------|------|
| **IAM 그룹** | `aws_iam_group` | **resource** | 그룹 **본체** |
| **그룹 정책 연결** | `aws_iam_group_policy_attachment` | **resource** | 그룹 ↔ 정책 **ARN** 연결 |
| **IAM 정책** | `aws_iam_policy` | **resource** | **독립 정책** (106에서 생성) |

- **66번** `aws_iam_role_policy_attachment` 의 **그룹 버전**이다.
- AWS **관리형 정책**도 `policy_arn` 으로 연결 가능 (`arn:aws:iam::aws:policy/...`).

### B. 인라인 정책 (Inline Policy)

```
IAM 그룹 (aws_iam_group)
         ↗
그룹 정책 (aws_iam_group_policy)
         ↗
정책 문서 (data.aws_iam_policy_document)
```

| 구성 | Terraform | 종류 | 역할 |
|------|-----------|------|------|
| **IAM 그룹** | `aws_iam_group` | **resource** | 그룹 **본체** |
| **그룹 정책** | `aws_iam_group_policy` | **resource** | 그룹에 **직접 내장**된 정책 |
| **정책 문서** | `data.aws_iam_policy_document` | **data** | 인라인 정책 **JSON** |

### 콘솔에서의 구분 (슬라이드)

| 종류 | 콘솔 표시 | Terraform |
|------|-----------|-----------|
| **관리형 정책** | **管理ポリシー** — `ReadOnlyAccess` 등 **독립 정책**을 Attach | `aws_iam_group_policy_attachment` |
| **인라인 정책** | **インラインポリシー** — 그룹에 **직접 설정** (`tastylog-dev-developers-policy` 등) | `aws_iam_group_policy` |

강의 실습은 주로 **관리형 연결(A)** + 106에서 만든 **자작 정책**을 붙이는 흐름이다.

## `aws_iam_group` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** * | string | IAM **그룹 이름** (필수) |

```hcl
resource "aws_iam_group" "developers" {
  name = "${var.project}-${var.environment}-developers-iam-group"
}
```

---

## `aws_iam_group_policy_attachment` (슬라이드)

**관리형 정책**(자작 `aws_iam_policy` 또는 AWS 관리형)을 그룹에 **연결**한다.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`group`** * | string | IAM **그룹 이름** |
| **`policy_arn`** * | string | IAM **정책 ARN** |

정책 하나당 **attachment 리소스를 하나** 둔다 (66번 Role과 동일 패턴).

```hcl
resource "aws_iam_group_policy_attachment" "readonly" {
  group      = aws_iam_group.developers.name
  policy_arn = "arn:aws:iam::aws:policy/ReadOnlyAccess"
}

resource "aws_iam_group_policy_attachment" "billing_deny" {
  group      = aws_iam_group.developers.name
  policy_arn = aws_iam_policy.billing_deny.arn
}
```

---

## `aws_iam_group_policy` (슬라이드)

**인라인 정책** — 그룹에만 존재하는 정책. 삭제 시 그룹과 함께 사라진다.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`group`** * | string | IAM **그룹 이름** |
| **`name`** | string | **인라인 정책** 이름 |
| **`policy`** * | string | 정책 **JSON 문자열** (필수) |

```hcl
resource "aws_iam_group_policy" "developers_inline" {
  name   = "${var.project}-${var.environment}-developers-policy"
  group  = aws_iam_group.developers.name
  policy = data.aws_iam_policy_document.developers.json
}
```

| | 관리형 (`_attachment`) | 인라인 (`_group_policy`) |
|---|------------------------|--------------------------|
| **정책 위치** | 독립 `aws_iam_policy` 리소스 | 그룹 **내부**에만 존재 |
| **재사용** | 여러 그룹·Role에 **공유** 가능 | 해당 그룹 **전용** |
| **ARN** | `policy_arn` 으로 참조 | 없음 (이름으로 관리) |

## 개발용 그룹 실습 요건 (슬라이드)

슬라이드 **開発用グループの作成** — **개발용 그룹**에 아래 정책을 붙인다.

| # | 권한 | 정책 |
|---|------|------|
| 1 | **전 리소스 읽기 허용** | AWS 관리형 **`ReadOnlyAccess`** |
| 2 | **빌링 거부** | 106에서 만든 자작 정책 (`aws-portal:*` Deny) |
| 3 | **EC2 재기동 허용** | 106에서 만든 자작 정책 (`ec2:RebootInstances`) |
| 4 | **비밀번호 변경 허용** | 106에서 만든 자작 정책 (`iam:ChangePassword`) |

→ **`aws_iam_group` 1개** + **`aws_iam_group_policy_attachment` 4개**.

### HCL 예시 (관리형 연결)

```hcl
resource "aws_iam_group" "developers" {
  name = "${var.project}-${var.environment}-developers-iam-group"
}

resource "aws_iam_group_policy_attachment" "developers_readonly" {
  group      = aws_iam_group.developers.name
  policy_arn = "arn:aws:iam::aws:policy/ReadOnlyAccess"
}

resource "aws_iam_group_policy_attachment" "developers_billing_deny" {
  group      = aws_iam_group.developers.name
  policy_arn = aws_iam_policy.billing_deny.arn
}

resource "aws_iam_group_policy_attachment" "developers_ec2_reboot" {
  group      = aws_iam_group.developers.name
  policy_arn = aws_iam_policy.ec2_rebootable.arn
}

resource "aws_iam_group_policy_attachment" "developers_change_password" {
  group      = aws_iam_group.developers.name
  policy_arn = aws_iam_policy.iam_change_own_password.arn
}
```

## 66·106·107 비교

| 대상 | 그룹/Role | 연결 리소스 | 정책 종류 |
|------|-----------|-------------|-----------|
| **66 Role** | `aws_iam_role` | `aws_iam_role_policy_attachment` | AWS 관리형 ARN |
| **106 Policy** | — | `aws_iam_policy` | 자작 정책 **생성** |
| **107 Group** | `aws_iam_group` | `aws_iam_group_policy_attachment` | 관리형 + 106 자작 정책 **연결** |

## 이 프로젝트와의 관계

| 항목 | 현재 `tf-user/` |
|------|-----------------|
| **워크스페이스** | `tastylog-user.tfstate` (main.tf backend) |
| **106 자작 정책** | `iam_policy.tf` — `billing_deny`, `ec2_rebootable`, `iam_change_own_password` |
| **`aws_iam_group`** | **아직 없음** — 107에서 `iam_group.tf` 등 추가 |
| **`aws_iam_group_policy_attachment`** | **아직 없음** — ReadOnlyAccess + 위 3정책 연결 |

> 106 슬라이드는 **한 정책에 statement 3개**였지만, `tf-user/iam_policy.tf` 는 **정책 3개로 분리**해 두었음. 그룹 연결은 **attachment 3개**로 동일하게 동작한다.
