# 97. CloudFront 리소스

섹션23은 **CloudFront(CDN)** 를 Terraform으로 만든다.  
**97번**은 CloudFront 관련 **리소스·data 구조와 인자 개요** — **98 ELB 오리진** · **99 S3 오리진** 실습의 기반.

슬라이드: **CloudFrontのデータ構造**

이전: **96 프라이빗 버킷** · **93 버지니아 ACM**  
다음: **98 CloudFront 작성 (ELB 오리진)** · **99 CloudFront 작성 (S3 오리진)**

## CloudFront 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (73·94번과 동일).

```
Route53 레코드 ──→ CloudFront (aws_cloudfront_distribution)
                        ↗              ↗
              ACM 증명서          Origin Access Identity
         (aws_acm_certificate)   (aws_cloudfront_origin_access_identity)
                                          ↓
                              Policy Document (data.aws_iam_policy_document)
                              → S3 bucket policy (99번)
```

| 구성 | Terraform | 참조 |
|------|-----------|------|
| **배포** | `aws_cloudfront_distribution` | (루트) |
| **뷰어 HTTPS** | `viewer_certificate` → **`virginia_cert`** | `acm_certificate_arn` (93) |
| **오리진 (ELB)** | `origin` + **`custom_origin_config`** | `aws_lb.alb.dns_name` (98) |
| **오리진 (S3)** | `origin` + **`s3_origin_config`** | static 버킷 + **OAI** (99) |
| **DNS** | `aws_route53_record` | CloudFront **domain_name** Alias (이후 강의) |
| **S3 접근 제한** | `aws_cloudfront_origin_access_identity` | OAI policy → bucket policy |

**별칭(`aliases`)** 에 커스텀 도메인을 쓰면 **`virginia_cert`** (`*.${var.domain}`) 와 Route53 레코드가 필요하다.

## CloudFront 관련 리소스 한눈에

| 리소스 / data | 역할 |
|---------------|------|
| **`aws_cloudfront_distribution`** | CloudFront **배포 본체** (오리진·캐시·HTTPS·지역 제한) |
| **`aws_cloudfront_origin_access_identity`** | S3 **프라이빗/제한** 오리진용 **OAI** (99) |
| **`data.aws_iam_policy_document`** | OAI가 S3를 읽도록 **bucket policy JSON** (99) |
| **`aws_s3_bucket_policy`** | static 버킷에 OAI policy 부착 (99) |
| **`aws_acm_certificate`** | **`virginia_cert`** — CloudFront는 **us-east-1** 만 (93) |
| **`aws_route53_record`** | `*.domain` 또는 서브도메인 → CloudFront Alias |

현재 `terraform/` 에 **CloudFront 리소스는 아직 없음** — 98·99에서 `cloudfront.tf` 등 추가.

## `aws_cloudfront_origin_access_identity` (슬라이드)

S3를 **CloudFront 경유로만** 읽게 할 때 사용 (99).  
(신규 AWS는 **OAC** 권장이나, 강의는 **OAI** 기준.)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`comment`** | string | **코メント** (관리용) |

```hcl
resource "aws_cloudfront_origin_access_identity" "static_oai" {
  comment = "${var.project}-${var.environment}-static-oai"
}
```

`origin_access_identity` 에는 **`aws_cloudfront_origin_access_identity.xxx.cloudfront_access_identity_path`** 를 넣는다.

## `aws_cloudfront_distribution` 개요 (슬라이드)

콘솔 설정을 **5블록**으로 나눈다:

| # | 슬라이드 | Terraform 블록 |
|---|----------|----------------|
| 1 | **基本設定** (기본 설정) | `enabled`, `comment`, `price_class`, `aliases` 등 |
| 2 | **オリジン** (오리진) | `origin { ... }` |
| 3 | **ビヘイビア** (동작) | `default_cache_behavior` · `ordered_cache_behavior` |
| 4 | **アクセス制限** (접근 제한) | `restrictions { geo_restriction { ... } }` |
| 5 | **証明書** (인증서) | `viewer_certificate { ... }` |

## 1. 기본 설정 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`enabled` *** | bool | 배포 **활성** 여부 |
| **`is_ipv6_enabled`** | bool | **IPv6** 사용 |
| **`comment`** | string | 배포 **코멘트** |
| **`price_class`** | enum | 예: `"PriceClass_All"` — 엣지 로케이션 범위·요금 |
| **`origin` *** | block | **오리진** (아래) |
| **`default_cache_behavior` *** | block | **기본 캐시 동작** (아래) |
| **`ordered_cache_behavior`** | block | **추가** 경로별 동작 (선택) |
| **`restrictions` *** | block | **지역 제한** (아래) |
| **`aliases`** | string[] | **커스텀 도메인** (CNAME/Alternate domain) |
| **`viewer_certificate` *** | block | **뷰어 HTTPS** (아래) |

`*` = 슬라이드상 **필수**.

---

## 2. `origin` 블록 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`domain_name` *** | string | 오리진 **DNS** (ALB DNS · S3 버킷 도메인) |
| **`origin_id` *** | string | 오리진 **고유 ID** — **behavior의 `target_origin_id`가 참조** |

**`custom_origin_config` 와 `s3_origin_config` 중 하나 필수** (*).

### `custom_origin_config` — **커스텀 오리진 (= ELB)** (98)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`origin_protocol_policy`** | enum | `"http-only"`, `"https-only"`, `"match-viewer"` |
| **`origin_ssl_protocols`** | enum[] | `"TLSv1.2"` 등 |
| **`http_port`** | number | HTTP 포트 (예: 80) |
| **`https_port`** | number | HTTPS 포트 (예: 443) |

예: `domain_name = aws_lb.alb.dns_name`, `origin_id = "alb-origin"`.

### `s3_origin_config` — **S3 오리진** (99)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`origin_access_identity`** | string | **OAI** path (`cloudfront_access_identity_path`) |

예: static 버킷 + OAI. 퍼블릭 policy 대신 **OAI + bucket policy** 로 CloudFront만 허용하는 패턴도 강의에서 다룸.

---

## 3. `default_cache_behavior` · `ordered_cache_behavior` (슬라이드)

두 블록 **동일 스키마**. **`default_cache_behavior` 에만 `path_pattern` 없음** (슬라이드 강조).

| 항목 | 타입 | 설명 |
|------|------|------|
| **`path_pattern` *** | string | URL **경로 패턴** — **`ordered_cache_behavior` 전용** |
| **`allowed_methods` *** | string[] | 허용 HTTP 메서드 (`GET`, `HEAD`, …) |
| **`cached_methods` *** | string[] | **캐시** 대상 메서드 |
| **`viewer_protocol_policy` *** | enum | `"allow-all"`, `"https-only"`, `"redirect-to-https"` |
| **`target_origin_id` *** | string | **전달 대상 `origin_id`** |
| **`forwarded_values` *** | block | **캐시 키에 넣을 요청 정보** (아래) |
| **`min_ttl`** | string | 최소 캐시(초) — Terraform은 **number** |
| **`default_ttl`** | string | 기본 캐시(초) |
| **`max_ttl`** | string | 최대 캐시(초) |
| **`compress`** | bool | **Gzip 등 압축** |

### `forwarded_values` (슬라이드·구 provider)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`query_string` *** | bool | **쿼리 스트링** 전달 여부 |
| **`headers`** | string[] | **전달할 헤더** (예: `Host`, `Authorization`) |
| **`cookies` *** | block | **`forward`**: `"all"` / `"none"` / `"whitelist"` · `whitelisted_names` |

슬라이드 표에 `query_string` 설명이 **포트 번호**로 잘못 적힌 경우 있음 — 의미는 **쿼리 스트링 forward** (94·95 실습 메모와 같이 **슬라이드 오타 주의**).

**provider 최신**: `cache_policy_id` / `origin_request_policy_id` 로 대체되는 경우 있음 — **강의·plan 에러**에 맞춰 `forwarded_values` vs managed policy 중 선택.

## 4. `restrictions` · `geo_restriction` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`restriction_type` *** | enum | `"none"`, `"whitelist"`, `"blacklist"` |
| **`locations`** | string[] | 국가 코드 (예: `"JP"`) — whitelist/blacklist 시 |

실습 기본:

```hcl
restrictions {
  geo_restriction {
    restriction_type = "none"
  }
}
```

## 5. `viewer_certificate` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`cloudfront_default_certificate`** | bool | **CloudFront 기본** `*.cloudfront.net` 인증서 |
| **`acm_certificate_arn`** | string | **ACM ARN** — 커스텀 도메인 시 |
| **`minimum_protocol_version`** | enum | `"TLSv1.2_2021"` 등 |
| **`ssl_support_method` *** | enum | `"sni-only"`, `"vip"` |

| 방식 | 설정 |
|------|------|
| **CloudFront 기본 URL** | `cloudfront_default_certificate = true` |
| **커스텀 도메인 (`aliases`)** | `acm_certificate_arn = aws_acm_certificate.virginia_cert.arn` + `ssl_support_method = "sni-only"` |

**SNI-only**: 한 서버에 **여러 인증서** — CloudFront **일반 선택** (슬라이드).  
**`virginia_cert`** 는 **us-east-1** (93). ALB용 **`tokyo_cert`** 와 **ARN 교환 금지**.

## provider·강의 차이 (주의)

| 항목 | 내용 |
