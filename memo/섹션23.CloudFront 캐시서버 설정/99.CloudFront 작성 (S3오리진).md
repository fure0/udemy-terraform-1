# 99. CloudFront 작성 (S3 오리진)

**정적 파일**은 **S3 static 버킷(95)** 을 CloudFront **오리진**으로 추가하고, **`/public/*`** 경로는 **`ordered_cache_behavior`** 로 S3에 보낸다.  
**OAI(Origin Access Identity)** + **bucket policy** 로 S3는 **CloudFront 경유**로만 읽게 한다 (97 데이터 구조).

슬라이드: **CloudFrontの作成/設定**

1. **CloudFront에 S3 오리진 추가**
2. **ビヘイビア(동작)에 S3 추가** — `/public/*`
3. **동작 확인**

이전: **98 ELB 오리진** · 섹션23 **마지막**

---

## 목표 (슬라이드·아키텍처)

```
사용자 ──https──→ CloudFront (dev.${var.domain})
                      │
         ┌────────────┴────────────┐
         │                         │
    /public/*                   デフォルト (*)
         │                         │
   S3 static (95)                  ELB (98)
  静的ファイル                    動的ファイル
  キャッシュあり                   キャッシュなし
```

| 항목 | 99에서 하는 일 |
|------|----------------|
| **오리진 추가** | `aws_s3_bucket.s3_static_bucket` + **`s3_origin_config`** + **OAI** |
| **동작 추가** | **`ordered_cache_behavior`** · `path_pattern = "/public/*"` |
| **S3 policy** | **OAI** `GetObject` 허용 (95 **퍼블릭 `*`** 정책 → **교체·조정**) |
| **default** | **98 그대로** — ELB · TTL 0 |

슬라이드: **静的ファイルはS3へ** — **`/public/*`** · **キャッシュあり**.

## 97번 데이터 구조 (99에서 완성)

```
CloudFront ──→ OAI (aws_cloudfront_origin_access_identity)
                    ↓
         data.aws_iam_policy_document (OAI GetObject)
                    ↓
         aws_s3_bucket_policy (static 버킷)
```

| 리소스 | 역할 |
|--------|------|
| **`aws_cloudfront_origin_access_identity`** | S3 접근 **주체** |
| **`data.aws_iam_policy_document`** | OAI ARN에 **s3:GetObject** |
| **`aws_s3_bucket_policy`** | static 버킷에 부착 (**95 policy 수정**) |
| **`origin` + `s3_origin_config`** | distribution에 S3 오리진 |
| **`ordered_cache_behavior`** | `/public/*` → S3 `origin_id` |

## 1. OAI 작성

```hcl
resource "aws_cloudfront_origin_access_identity" "static_oai" {
  comment = "${var.project}-${var.environment}-static-oai"
}
```

`origin_access_identity` 에 넣는 값:

```hcl
aws_cloudfront_origin_access_identity.static_oai.cloudfront_access_identity_path
```

---

## 2. S3 bucket policy — OAI 허용 (95 수정)

95에서 쓴 **`Principal *` 퍼블릭 읽기**는 OAI 패턴과 **겹치거나 보안상 제거**하고, **OAI만** 허용하는 정책으로 바꾼다 (강의).

```hcl
data "aws_iam_policy_document" "s3_static_bucket" {
  statement {
    effect  = "Allow"
    actions = ["s3:GetObject"]
    resources = [
      "${aws_s3_bucket.s3_static_bucket.arn}/*"
    ]

    principals {
      type        = "AWS"
      identifiers = [aws_cloudfront_origin_access_identity.static_oai.iam_arn]
    }
  }
}
```

| | 95 (단독 공개) | 99 (CloudFront) |
|---|----------------|-----------------|
| **Principal** | `*` | **OAI `iam_arn`** |
| **직접 S3 URL** | 가능 | **불가** (정책상) |
| **CF `/public/*`** | — | **가능** |

`depends_on = [aws_s3_bucket_public_access_block.s3_static_bucket]` 유지 (95).

---

## 3. CloudFront — S3 오리진 추가 (98 distribution 확장)

**98의 `aws_cloudfront_distribution.cf`** 에 **두 번째 `origin` 블록** 추가:

```hcl
  origin {
    domain_name = aws_s3_bucket.s3_static_bucket.bucket_regional_domain_name
    origin_id   = "s3-static-origin"

    s3_origin_config {
      origin_access_identity = aws_cloudfront_origin_access_identity.static_oai.cloudfront_access_identity_path
    }
  }
```

| 항목 | 값 |
|------|-----|
| **`domain_name`** | **`bucket_regional_domain_name`** (리전 엔드포인트) |
| **`origin_id`** | `"s3-static-origin"` — behavior가 참조 |
| **`s3_origin_config`** | **`custom_origin_config` 와 배타** (97) |

ELB `origin` 블록은 **98 그대로** 두고 **origin 블록을 하나 더** 추가.

---

## 4. `ordered_cache_behavior` — `/public/*`

```hcl
  ordered_cache_behavior {
    path_pattern     = "/public/*"
    allowed_methods  = ["GET", "HEAD"]
    cached_methods   = ["GET", "HEAD"]
    target_origin_id = "s3-static-origin"

    forwarded_values {
      query_string = false

      cookies {
        forward = "none"
      }
    }

    viewer_protocol_policy = "redirect-to-https"
    min_ttl                = 0
    default_ttl            = 3600
    max_ttl                = 86400
    compress               = true
  }
```

| 항목 | S3 정적 (99) | ELB 동적 (98 default) |
|------|--------------|------------------------|
| **`path_pattern`** | **`/public/*`** | **(없음)** |
| **캐시** | **あり** — TTL > 0 | **なし** — TTL 0 |
| **쿠키** | `none` | `all` (앱) |
| **메서드** | GET, HEAD | GET, HEAD, … (앱) |

**`path_pattern`은 `ordered_cache_behavior`에만** — `default_cache_behavior`에는 넣지 않음 (97).

## 5. S3 객체 키와 URL 경로

CloudFront 요청 `https://dev.${var.domain}/public/css/app.css`  
→ S3 오리진 객체 키 **`public/css/app.css`** (경로 그대로).

업로드 예 (95 2단계):

```bash
aws s3 cp ./static s3://<static-bucket>/public/ --recursive
```

버킷 안에 **`public/` prefix** 로 올려야 `/public/*` 와 맞는다.

## 동작 확인 (슬라이드 2)

1. static 버킷에 `public/` 아래 파일 업로드
2. 브라우저:

```
https://dev.${var.domain}/public/<key>
```

3. **캐시**: 두 번째 요청에서 `X-Cache: Hit from cloudfront` (TTL 내)
4. **루트 `/`**: 앱(ELB) 응답 — S3 아님

이 프로젝트: **`route53_cloudfront`** → `dev.${var.domain}` (98 `cloudfront.tf`).

## 98·현재 코드와의 관계

| 항목 | 현재 `cloudfront.tf` |
|------|----------------------|
| **ELB 오리진** | `cf` · `custom_origin_config` · default → ALB |
| **aliases** | `dev.${var.domain}` + **`virginia_cert`** |
| **Route53** | `route53_cloudfront` Alias |
| **S3·OAI** | **99에서 추가** |

## 정리

- **99** = **OAI** + **S3 bucket policy** + **S3 origin** + **`ordered_cache_behavior` `/public/*`**.
- 슬라이드 **静的ファイルはS3へ** = 경로 **`/public/*`** · **キャッシュあり**.
- **98 default** 는 ELB·동적·캐시 없음 — **한 distribution에 오리진 2개**.
- 정적 파일은 S3 **`public/` prefix** 로 업로드 후 **`dev.${var.domain}/public/...`** 로 확인.
