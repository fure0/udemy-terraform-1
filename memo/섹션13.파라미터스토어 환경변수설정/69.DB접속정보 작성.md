# 69. DB 접속정보 작성

앱 EC2가 RDS(MySQL)에 붙을 때 쓰는 **접속 정보**를 **Systems Manager Parameter Store**에 둔다.  
Terraform에서는 **`aws_ssm_parameter`** 리소스로 파라미터를 만든다. EC2는 12번에서 붙인 **SSM 읽기 권한**으로 이 값을 가져와 환경 변수처럼 쓴다.

---

## `aws_ssm_parameter` 주요 인자

| 항목 | 타입 | 설명 |
|------|------|------|
| `name` | string | **파라미터 이름(경로)**(필수). 예: `/tastylog/dev/app/MYSQL_HOST` |
| `type` | string | **필수.** `String`, `StringList`, `SecureString` |
| `value` | string | **필수.** 저장할 값 |
| `overwrite` | bool | 이미 있으면 **덮어쓸지** |
| `description` | string | 설명(선택) |
| `tags` | map(string) | 태그(선택) |

- **`SecureString`**: 비밀번호 등은 암호화 저장(슬라이드 표기 `SeureString`은 오타, 올바른 값은 **`SecureString`**).
- `name` / `type` / `value`는 슬라이드에서 `*`로 **필수**로 표시된다.

---

## 강의에서 만드는 파라미터 (경로)

공통 prefix: **`/tastylog/dev/app/`** (이 저장소는 `var.project`=`tastylog`, `var.environment`=`dev`일 때)

| 파라미터 경로 | 용도 | 값 예시(이 프로젝트 RDS) |
|---------------|------|---------------------------|
| `/tastylog/dev/app/MYSQL_HOST` | DB 호스트 | `aws_db_instance.mysql_standalone.address` |
| `/tastylog/dev/app/MYSQL_PORT` | 포트 | `3306` (`port` 또는 문자열) |
| `/tastylog/dev/app/MYSQL_DATABASE` | DB 이름 | `tastylog` (`db_name`) |
| `/tastylog/dev/app/MYSQL_USERNAME` | 사용자 | `admin` (`username`) |
| `/tastylog/dev/app/MYSQL_PASSWORD` | 비밀번호 | `random_string.db_password.result` → **`SecureString` 권장** |

경로는 보통 **`/${var.project}/${var.environment}/app/<이름>`** 형태로 변수화한다.

---

## HCL 예시

```hcl
resource "aws_ssm_parameter" "mysql_host" {
  name  = "/${var.project}/${var.environment}/app/MYSQL_HOST"
  type  = "String"
  value = aws_db_instance.mysql_standalone.address

  overwrite   = true
  description = "MySQL host for app"

  tags = {
    Project = var.project
    Env     = var.environment
  }
}

resource "aws_ssm_parameter" "mysql_port" {
  name  = "/${var.project}/${var.environment}/app/MYSQL_PORT"
  type  = "String"
  value = tostring(aws_db_instance.mysql_standalone.port)
  overwrite = true
}

resource "aws_ssm_parameter" "mysql_database" {
  name  = "/${var.project}/${var.environment}/app/MYSQL_DATABASE"
  type  = "String"
  value = aws_db_instance.mysql_standalone.db_name
  overwrite = true
}

resource "aws_ssm_parameter" "mysql_username" {
  name  = "/${var.project}/${var.environment}/app/MYSQL_USERNAME"
  type  = "String"
  value = aws_db_instance.mysql_standalone.username
  overwrite = true
}

resource "aws_ssm_parameter" "mysql_password" {
  name  = "/${var.project}/${var.environment}/app/MYSQL_PASSWORD"
  type  = "SecureString"
  value = random_string.db_password.result
  overwrite = true
}
```

비밀번호를 Parameter Store에 넣으면 **tfstate·콘솔 접근 권한** 관리가 중요하다(섹션9 RDS 패스워드 메모 참고).

---

## 리소스 태그 (슬라이드)

Parameter뿐 아니라, 강의에서는 아래 태그가 **VPC·EC2 등에 이미 붙어 있어야** 한다고 정리한다.

| 대상 | 태그 |
|------|------|
| **VPC 관련 리소스** | `Project`, `Env` |
| **EC2** | `Type` (앱 서버는 예: `app`) |

이 저장소 `rds.tf`·`appserver.tf` 등에는 `Project`, `Env`가 이미 쓰이고, EC2에는 `Type = "app"`이 있다.

---

## 정리

- DB 접속 5종을 **Parameter Store 경로**로 등록한다.
- Terraform 리소스는 **`aws_ssm_parameter`**, 비밀번호는 **`SecureString`**.
- 값은 **`aws_db_instance.mysql_standalone`**·`random_string.db_password` 등 **기존 RDS 리소스에서 참조**하면 drift를 줄일 수 있다.
- 다음 단계에서는 앱/EC2가 이 파라미터를 **읽어 환경 변수로 쓰는** 흐름으로 이어진다.
