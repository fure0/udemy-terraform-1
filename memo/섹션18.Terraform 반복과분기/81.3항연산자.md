# 81. 삼항 연산자 (3項演算子)

**조건에 따라 값을 하나 고르는** HCL 표현식이다.  
if/else **문(statement)** 이 아니라, **한 줄로 값을 정하는 식(expression)** 이다.

슬라이드 예:

```hcl
var.a != "" ? var.a : "default_value"
```

이전: **80 `for` 표현식** · 섹션18 **분기** 마지막

---

## 문법

```hcl
<조건> ? <참일 때 값> : <거짓일 때 값>
```

| 부분 | 의미 |
|------|------|
| **조건** | `true` / `false` 로 평가되는 식 (`==`, `!=`, `&&`, `||` 등) |
| **`?`** | 조건이 **참**이면 왼쪽 값 사용 |
| **`:`** | 조건이 **거짓**이면 오른쪽 값 사용 |

슬라이드 예 해석:

- `var.a`가 **빈 문자열이 아니면** → `var.a` 그대로
- **빈 문자열이면** → `"default_value"` 사용

변수에 값이 없을 때 **기본값을 넣는** 패턴으로 자주 쓴다.

---

## 어디에 쓰나

`locals`, `variable` 기본값, 리소스 인자, `output` 등 **값이 들어가는 자리** 어디든 가능하다.

```hcl
locals {
  display_name = var.name != "" ? var.name : "unnamed"
}

resource "aws_instance" "app" {
  instance_type = var.use_large ? "t2.small" : "t2.micro"
  ...
}
```

80번 **`for` 표현식**·75번 **`count`** 와 함께 **반복·분기**를 코드로 표현할 때 쓴다.

---

## 조건·타입 주의

| 항목 | 내용 |
|------|------|
| **조건** | 반드시 **bool** (`true` / `false`) |
| **양쪽 값** | **같은 타입**이어야 한다 (string / string, number / number 등) |
| **중첩** | `a ? b : c ? d : e` 처럼 **괄호 없이** 중첩 가능하나, 읽기 어려우면 `locals`로 나눈다 |

```hcl
# OK — 둘 다 string
var.env == "prod" ? "production" : "development"

# 주의 — 타입이 다르면 에러
# var.flag ? 1 : "yes"   # 불가
```

---

## 자주 쓰는 패턴

### 1. 빈 값일 때 기본값 (슬라이드)

```hcl
var.a != "" ? var.a : "default_value"
```

### 2. 환경·플래그에 따라 인자 변경

```hcl
instance_type = var.environment == "prod" ? "t3.small" : "t2.micro"
```

### 3. `count`와 조합 (74번)

리소스를 **조건부로 0개 또는 1개**:

```hcl
resource "aws_instance" "bastion" {
  count = var.create_bastion ? 1 : 0
  ...
}
```

`count` 자리에 **bool → number** 로 바꾸는 전형적인 삼항 연산 사용이다.

### 4. `for` + 삼항 (80번)

```hcl
[for s in var.list : s != "" ? upper(s) : "N/A"]
```

---

## 비슷한 내장 함수 (참고)

| 수단 | 용도 |
|------|------|
| **삼항 `? :`** | **조건 하나**로 두 값 중 선택 |
| **`coalesce(a, b, ...)`** | **첫 번째 non-null·non-empty** 값 (29번) |
| **`try(expr, default)`** | 식이 **에러 나면** 대체값 |

빈 문자열만 걸러낼 때는 슬라이드처럼 `!= "" ? ... : ...` 가 명확하다.  
null만 보면 `coalesce(var.a, "default")` 도 고려한다.

## 이 프로젝트에 적용하는 예 (참고)

현재 `terraform/` 코드에는 **삼항 연산자를 쓰지 않는다**.  
`var.project`, `var.environment`는 이미 문자열 보간으로 쓰고 있다.

예를 들어 dev에서만 작은 인스턴스:

```hcl
resource "aws_instance" "app_server" {
  instance_type = var.environment == "prod" ? "t3.small" : "t2.micro"
  ...
}
```

SSM parameter 이름에 기본 접두사:

```hcl
locals {
  param_prefix = var.param_prefix != "" ? var.param_prefix : "/${var.project}/${var.environment}/app"
}
```

## 80번·74번과의 관계

| 구문 | 역할 |
|------|------|
| **80 `for`** | 컬렉션 **순회·변환** |
| **81 삼항** | 조건에 따른 **값 선택** |
| **74 `count`** | `조건 ? 1 : 0` 과 묶어 **리소스 개수** 제어 |

섹션18은 **데이터 가공(`for`)** 과 **분기(삼항)** 로 HCL에서 반복·조건을 표현하는 흐름이다.

---

## 정리

- **삼항 연산자**: `조건 ? 참값 : 거짓값`
- 슬라이드: `var.a != "" ? var.a : "default_value"` — **빈 값이면 기본값**
- **bool 조건**, **양쪽 같은 타입** 주의
- `count`, `for`와 조합해 **리소스·데이터** 모두 분기 가능
- 섹션18 끝 — 다음 섹션부터는 강의 진도에 맞춰 이어서 정리
