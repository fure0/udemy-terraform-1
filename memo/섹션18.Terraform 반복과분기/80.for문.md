# 80. for 문 (`for` 표현식)

**컬렉션(list, map, set)을 다른 list·map으로 바꾸는 HCL 구문**이다.  
리소스를 여러 개 만드는 **메타 인수 `for_each`(75번)** 와 이름이 비슷하지만, **값을 가공하는 표현식**이다.

슬라이드: 「リスト型（list, map, set）を異なるリスト型（list, map）へ変換する構文」

다음: **81 삼항 연산자**

---

## 기본 형태

```hcl
[for s in var.list : upper(s)]
```

| 부분 | 의미 |
|------|------|
| `[ ... ]` | 결과가 **list** |
| `{ ... }` | 결과가 **map** |
| `for s in var.list` | `var.list`의 각 원소를 `s`로 순회 |
| `:` | 순회와 **변환 식** 구분 |
| `upper(s)` | 각 원소에 적용할 식 |

`locals`, `variable` 기본값, `output`, 다른 인자 값 등 **어디서든** 쓸 수 있다.

---

## 변환 패턴 (슬라이드)

**변환 원(変換元)** → **변환 대상(変換先)** 조합은 아래와 같다.

| 변환 원 | 변환 대상 | 샘플 |
|---------|-----------|------|
| **list, set** | **list** | `[for s in var.list : upper(s)]` |
| **list, set** | **map** | `{for s in var.list : s => upper(s)}` |
| **map** | **list** | `[for k, v in var.map : upper(v)]` |
| **map** | **map** | `{for k, v in var.map : k => upper(v)}` |

```
list, set ──┬──▶ list
            └──▶ map

map ────────┬──▶ list
            └──▶ map
```

---

## 패턴별 설명

### list, set → list

```hcl
[for s in var.list : upper(s)]
```

- `var.list`가 `["a", "b"]` 이면 → `["A", "B"]`
- **set**을 넣어도 문법은 같다. 결과는 **list** (순서는 set 특성상 보장되지 않음).

### list, set → map

```hcl
{for s in var.list : s => upper(s)}
```

- `["a", "b"]` → `{ "a" = "A", "b" = "B" }`
- **`=>`** 로 **키 => 값** 쌍을 만든다.

### map → list

```hcl
[for k, v in var.map : upper(v)]
```

- map을 돌 때는 **`k, v`** 두 변수 (키, 값).
- 키는 버리고 **값만** 변환해 list로 모은다.

### map → map

```hcl
{for k, v in var.map : k => upper(v)}
```

- **키는 유지**, 값만 `upper(v)` 등으로 변환한 **새 map**.

---

## 필터 (スライド: フィルター)

끝에 **`if 조건`** 을 붙이면, 조건을 만족하는 원소만 결과에 넣는다.

```hcl
[for s in var.list : upper(s) if s != ""]
```

- 빈 문자열 `""` 은 제외하고 나머지만 대문자 list로 만든다.
- map 변환에도 동일: `{for k, v in var.map : k => upper(v) if v != ""}`

## `for` 표현식 vs 메타 인수 `for_each` (75번)

이름이 헷갈리기 쉬우니 구분한다.

| | **`for` 표현식** (80번) | **`for_each` 메타 인수** (75번) |
|---|------------------------|--------------------------------|
| 위치 | `locals`, 인자 값 등 **식** | `resource` / `module` 블록 **안** |
| 역할 | list/map **데이터 변환** | 리소스 **복수 생성** |
| 결과 | list 또는 map **값** | `aws_subnet.subnet["키"]` 같은 **리소스 인스턴스** |

연결 예: list를 map으로 바꾼 뒤, 그 map을 `for_each`에 넘기는 식으로 조합할 수 있다.

```hcl
locals {
  subnet_map = {
    "192.168.1.0/24" = "ap-northeast-1a"
    "192.168.2.0/24" = "ap-northeast-1c"
  }
}

resource "aws_subnet" "subnet" {
  for_each = local.subnet_map
  cidr_block        = each.key
  availability_zone = each.value
}
```

75번은 **리소스** 쪽, 80번은 **`local.subnet_map` 같은 데이터** 쪽.

## `terraform console`에서 연습

29번 내장 함수와 같이 콘솔에서 바로 시험한다.

```bash
cd terraform
terraform console
```

```hcl
> [for s in ["tanaka", "sato"] : upper(s)]
[
  "TANAKA",
  "SATO",
]
> {for s in ["a", "b"] : s => upper(s)}
{
  "a" = "A"
  "b" = "B"
}
> [for s in ["a", "", "b"] : upper(s) if s != ""]
[
  "A",
  "B",
]
```

## 이 프로젝트에 적용하는 예 (참고)

현재 `terraform/` 에는 **`for` 표현식을 아직 쓰지 않는다**.  
75번 메모의 IAM policy 목록을 `locals`에서 가공하는 예:

```hcl
locals {
  policy_arns = [
    "arn:aws:iam::aws:policy/AmazonEC2ReadOnlyAccess",
    "arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore",
  ]
  # set으로 바꿔 for_each에 넘기기 (중복 제거)
  policy_arn_set = toset(local.policy_arns)
}
```

또는 태그 map에서 빈 값 제거:

```hcl
locals {
  tags = {
    Name    = "${var.project}-${var.environment}-app"
    Project = var.project
    Env     = var.environment
    Note    = ""
  }
  tags_clean = {for k, v in local.tags : k => v if v != ""}
}
```

## 정리

- **`for` 표현식**: list/map/set → **list(`[]`)** 또는 **map(`{}`)** 변환.
- list/set 순회: `for s in ...` / map 순회: `for k, v in ...`
- map 출력: `키 => 값`, 필터: 끝에 **`if 조건`**
- **리소스 복제**는 75번 **`for_each` 메타 인수** — 80번과 목적이 다름.
