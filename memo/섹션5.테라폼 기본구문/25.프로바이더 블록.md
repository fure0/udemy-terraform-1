## `provider` 블록

**`provider "<PROVIDER_TYPE>" { ... }`** 는 **사용할 프로바이더(클라우드·SaaS 등)에 대한 설정**을 넣는 블록이다. (슬라이드: **프로바이더의 설정**)

- 리소스(`resource`)를 어떤 API·계정·리전으로 쓸지 연결하는 역할에 가깝다.
- 예: `aws` 프로바이더면 리전, 자격 증명 프로필 등.

### 구성에서 자주 보는 블록 순서(예시)

```terraform
locals { ... }

variable "<VAR_NAME>" { ... }

terraform { ... }

provider "<PROVIDER_TYPE>" { ... }

resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" { ... }

output "<OUTPUT_NAME>" { ... }
```

`terraform` 블록 다음, **`resource`보다 앞**에 두는 경우가 많다.

### `provider` 블록에 넣는 것 (예)

| 항목 | 역할 |
|------|------|
| `region` | (AWS 등) 기본 리전 |
| `profile` | AWS CLI 프로필 이름 등 |
| `alias` | 같은 프로바이더를 여러 번 쓸 때 구분 |

프로바이더마다 지원하는 인자가 다르므로, **AWS Provider** 문서 등에서 `provider "aws"` 예제를 보면 된다.
