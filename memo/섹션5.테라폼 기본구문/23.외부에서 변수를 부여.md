## 변수의 덮어쓰기(외부에서 값 주기)

코드의 `default` 말고, **실행 시점에 바깥에서** 변수 값을 넣는 방법은 크게 **세 가지**다.

| 종류 | 설명 |
|------|------|
| **환경 변수** | 미리 셸 등에 설정해 둔 값을 사용한다. 형식: `TF_VAR_<변수이름>` |
| **변수 파일** | 정해진 이름의 파일에 값을 적어 둔다. 대표: `terraform.tfvars` |
| **커맨드 인자** | 실행할 때 인자로 넘긴다. `-var 이름=값`, `-var-file 경로` |

---

### 1. 환경 변수로 덮어쓰기

Terraform은 **`TF_VAR_` + 변수 이름** 형태의 환경 변수를 자동으로 읽어, 같은 이름의 `variable`에 넣는다.

**예: `main.tf`**

```terraform
variable "message" {
  type    = string
  default = "nothing"
}
```

**실행**

```bash
export TF_VAR_message="Hello World !"
terraform apply
```

`apply` 시점에 `message`는 `"Hello World !"`가 되고, `default`의 `"nothing"`은 쓰이지 않는다.

---

### 2. 변수 파일로 덮어쓰기 (`terraform.tfvars`)

같은 디렉터리에 **`terraform.tfvars`** 를 두면, Terraform이 자동으로 읽어 변수에 반영한다(이름·형식이 맞을 때).

**예: `main.tf`** (위와 동일한 `variable "message"`)

**예: `terraform.tfvars`**

```terraform
message = "Hello World !"
```

**실행**

```bash
terraform apply
```

파일에 적은 값이 `default`보다 우선한다.

- 다른 파일 이름을 쓰려면 `terraform apply -var-file=파일경로` 로 지정한다.

---

### 3. 커맨드로 덮어쓰기 (`-var`)

**실행할 때** 한 번에 값을 넘긴다.

```bash
terraform apply -var 'message=Hello World !'
```

또는 슬라이드와 같이:

```bash
terraform apply -var message="Hello World !"
```

- **`-var-file <파일경로>`** 로 특정 `.tfvars` 파일만 지정해서 적용할 수도 있다.

---

## 방법별로 어떻게 쓸지

| 종류 | 이럴 때 쓰면 좋다 |
|------|-------------------|
| **환경 변수** | **비밀값·키**처럼 파일에 남기기 싫은 값. 실행 로그에 남기지 않으려는 목적에 적합(다만 **state 등 다른 경로에 값이 남을 수는 있음**). 운영 서버에서 `dev` / `stg` 같이 **환경별로만** 나눌 때도 자주 쓴다. |
| **변수 파일 (`.tfvars`)** | **Git으로 관리**하기 쉽고, **코드(절차)** 와 **데이터(변수 값)** 를 나눠서 변경 추적이 명확해진다. |
| **커맨드 인자 (`-var`)** | 값이 **실행 로그에 남는 편**이라, **일시적으로**만 바꿀 때(테스트·디버그·한 번만 다른 값 시도)에 적합하다. |
