## 템플릿(초기 뼈대) 작성

새 Terraform 구성을 시작할 때는, 먼저 **기본 구조(템플릿)** 를 만들어 두면 작업이 빨라진다.

### 1) 기본 파일 준비

- `main.tf`
- `terraform.tfvars`
- `.gitignore`

---

### 2) `main.tf` 기본 뼈대

```terraform
variable "<VAR_NAME>" {
  # ...
}

terraform {
  # ...
}

provider "<PROVIDER_TYPE>" {
  # ...
}
```

필요한 리소스(`resource`), 조회(`data`), 출력(`output`)는 이후 요구사항에 맞게 추가한다.

---

### 3) `terraform.tfvars`

변수 값을 분리해서 관리한다.  
처음에는 최소한의 값만 넣고, 환경(dev/stg/prod)별로 파일을 분리해도 된다.

---

### 4) `.gitignore`

Terraform 작업에서는 상태 파일/캐시 등 버전 관리에서 제외할 항목이 많다.  
슬라이드처럼 **`gitignore.io`** 템플릿을 활용하면 빠르게 시작할 수 있다.

예: `terraform.tfstate`, `terraform.tfstate.*`, `.terraform/` 등
