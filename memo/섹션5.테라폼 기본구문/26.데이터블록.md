## `data` 블록

**`data "<DATA_TYPE>" "<DATA_NAME>" { ... }`** 는 **이 Terraform으로 만들거나 관리하지 않는 기존 리소스**의 정보를 **조회해서 가져올 때** 쓰는 블록이다. (슬라이드: **관리 대상 밖의 리소스를 끌어와 쓸 수 있다**)

- `resource`는 생성·변경·삭제의 **대상**이 되고, `data`는 **읽기 전용 조회**에 가깝다.
- 예: 이미 있는 VPC ID, 최신 Amazon Linux AMI, 다른 팀이 만든 S3 버킷 ARN 등.

### 구성에서 자주 보는 블록 순서(예시)

```terraform
locals { ... }

variable "<VAR_NAME>" { ... }

terraform { ... }

provider "<PROVIDER_TYPE>" { ... }

data "<DATA_TYPE>" "<DATA_NAME>" { ... }

resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" { ... }
```

`data`는 보통 **`resource` 앞**에 두고, 조회 결과는 `data.<DATA_TYPE>.<DATA_NAME>.<속성>` 형태로 참조한다.
