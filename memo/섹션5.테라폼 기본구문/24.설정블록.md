## `terraform` 블록

Terraform 설정 파일에서 **`terraform { ... }`** 는 **Terraform 도구·실행 전체에 걸치는 설정**을 넣는 블록이다. (개별 리소스가 아니라 “이 프로젝트에서 Terraform을 어떻게 쓸지”에 가깝다.)

### 구성에서 자주 보는 블록 순서(예시)

```terraform
locals { ... }

variable "<VAR_NAME>" { ... }

terraform { ... }

provider "<PROVIDER_TYPE>" { ... }

resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" { ... }

output "<OUTPUT_NAME>" { ... }
```

`terraform` 블록은 그중에서도 **전역(전체) 설정**을 담당한다.

### `terraform` 블록에 넣는 것 (예)

| 항목 | 역할 |
|------|------|
| `required_version` | 사용할 Terraform CLI 버전 범위 지정 |
| `required_providers` | provider 이름·소스·버전 제약 |
| `backend` | **state 파일**을 어디에 둘지(S3, 로컬 등) |

실제 문법은 강의·공식 문서의 `terraform` 블록 예제를 따르면 된다.
