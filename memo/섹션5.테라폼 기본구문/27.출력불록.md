## `output` 블록

**`output "<OUTPUT_NAME>" { ... }`** 는 **만든 리소스의 값을 밖에서 참조할 수 있게** 노출하는 블록이다. (슬라이드: **생성한 리소스를 외부에서 참조할 수 있게 한다**)

- `terraform apply` 후 터미널에 요약으로 나오거나, `terraform output` 으로 다시 볼 수 있다.
- 다른 스택·모듈에서 **remote state** 등으로 가져다 쓸 값을 정의할 때도 쓴다.

### 구성에서 자주 보는 블록 순서(예시)

```terraform
locals { ... }

variable "<VAR_NAME>" { ... }

terraform { ... }

provider "<PROVIDER_TYPE>" { ... }

data "<DATA_TYPE>" "<DATA_NAME>" { ... }

resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" { ... }

output "<OUTPUT_NAME>" { ... }
```

`output`은 보통 파일 **맨 아래쪽**에 두는 경우가 많다.

### 예시 (EC2 인스턴스 ID를 출력)

`resource`로 만든 리소스의 속성을 `value`에 넣으면, `apply` 후 그 값이 출력된다.

```terraform
resource "aws_instance" "hello-world" {
  ami           = "ami-0ce107ae7af2e92b5"
  instance_type = "t2.micro"
}

output "ec2_instance_id" {
  value = aws_instance.hello-world.id
}
```

- `aws_instance.hello-world.id` : 리소스 타입·이름(따옴표 없음)·속성 순으로 참조한다.
- AMI ID는 **리전마다 다르므로** 실제 사용 시에는 해당 리전에 맞는 AMI로 바꾼다.
