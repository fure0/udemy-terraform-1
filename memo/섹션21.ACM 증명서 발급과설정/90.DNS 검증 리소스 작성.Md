# 90. DNS 검증 리소스 작성

**89번** `aws_acm_certificate` 요청 후, **DNS 검증**을 Terraform으로 자동화한다.  
필요한 것은 **2가지** (슬라이드):

1. **Route53 레코드** — 검증용 **CNAME**
2. **ACM 증명서 검증** — `aws_acm_certificate_validation`

이전: **89 ACM** · 다음: **91 도메인 설정과 검증(수동)** · **92 ELB HTTPS**

---

## DNS 검증 흐름 (슬라이드)

```
Terraform
  ↓
ACM 증명서 (Pending) ──→ 검증용 CNAME 정보 생성
  ↓
Route53 CNAME 레코드 등록
  ↓
AWS가 DNS 조회 → 검증 OK (👍) → 상태 Issued
```

| 단계 | 리소스 |
|------|--------|
| 89 | `aws_acm_certificate.tokyo_cert` |
| **90** | `aws_route53_record` (검증 CNAME) |
| **90** | `aws_acm_certificate_validation` |
| 91·88 | NS·공인 DNS — **조회 가능**해야 Issued |

---

## ACM 데이터 구조 (슬라이드)

```
ACM 증명서 (aws_acm_certificate.tokyo_cert)     ← 89
    ↑                    ↑
Route53 레코드 (검증 CNAME)   ACM 증명서 검증 (aws_acm_certificate_validation)
    ↑                              ↑
    └──────── ACM 증명서 ──────────┘
```

| 리소스 | 참조 | 역할 |
|--------|------|------|
| **검증 Route53 레코드** | ACM | `domain_validation_options` 의 name/type/value |
| **certificate_validation** | ACM + Route53 | 레코드 생성 후 **Issued** 될 때까지 대기 |

---

## `domain_validation_options` (슬라이드)

89 `apply` 후 ACM이 만드는 **검증용 CNAME 정보** (JSON 개념):

```json
"domain_validation_options": [
  {
    "domain_name": "*.tastylog.work",
    "resource_record_name": "_0123...cdef.tastylog.work.",
    "resource_record_type": "CNAME",
    "resource_record_value": "_0123...cdef.xxxxxxxxxx.acm-validations.aws."
  }
]
```

| 필드 | Route53에 넣을 값 |
|------|-------------------|
| `resource_record_name` | 레코드 **이름** (`name`) |
| `resource_record_type` | **`CNAME`** (`type`) |
| `resource_record_value` | **값** (`records`) |

와일드카드 `*.${var.domain}` 이면 **옵션이 1개**(또는 SAN 추가 시 여러 개) — **`for_each`** 로 레코드 생성.

---

## 1. Route53 검증 레코드

`route53.tf` 또는 `acm.tf`에 추가:

```hcl
resource "aws_route53_record" "cert_validation" {
  for_each = {
    for dvo in aws_acm_certificate.tokyo_cert.domain_validation_options : dvo.domain_name => {
      name   = dvo.resource_record_name
      record = dvo.resource_record_value
      type   = dvo.resource_record_type
    }
  }

  allow_overwrite = true
  name            = each.value.name
  records         = [each.value.record]
  ttl             = 60
  type            = each.value.type
  zone_id         = aws_route53_zone.route53_zone.zone_id
}
```

| 항목 | 설명 |
|------|------|
| **`for_each`** | `domain_validation_options` 항목마다 CNAME 1개 (75·80번) |
| **`allow_overwrite`** | 재요청·갱신 시 **같은 이름** 덮어쓰기 |
| **`zone_id`** | **`aws_route53_zone.route53_zone`** (86·현재 코드) |

---

## 2. `aws_acm_certificate_validation` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`certificate_arn`** * | string | ACM **증명서 ARN** |
| **`validation_record_fqdns`** | string[] | DNS 검증에 쓰는 **FQDN** 목록 |

슬라이드: `validation_record_fqdns` = **Route53에 등록한 검증 CNAME의 FQDN**.

```hcl
resource "aws_acm_certificate_validation" "tokyo_cert" {
  certificate_arn = aws_acm_certificate.tokyo_cert.arn

  validation_record_fqdns = [
    for record in aws_route53_record.cert_validation : record.fqdn
  ]
}
```

| 동작 | 내용 |
|------|------|
| **apply** | Route53 CNAME 반영 후 AWS가 검증 → **Issued** 될 때까지 **대기** |
| **92 HTTPS** | `certificate_arn` 은 **`aws_acm_certificate.tokyo_cert.arn`** 또는 **`aws_acm_certificate_validation.tokyo_cert.certificate_arn`** |

---

## 이 프로젝트와 맞추기

| 항목 | 현재 코드 |
|------|-----------|
| ACM | `aws_acm_certificate.tokyo_cert` — `*.${var.domain}` |
| Zone | `aws_route53_zone.route53_zone` — `var.domain` |
| ALB A 레코드 | `aws_route53_record.route53_record` — `dev-elb.${var.domain}` (별도) |

검증 CNAME은 **ACM이 준 `_xxx...` 이름**이라 ALB Alias 레코드와 **충돌하지 않음**.

89 `depends_on = [aws_route53_zone.route53_zone]` — 존 먼저 만든 뒤 인증서 요청(강의 패턴).

---

## apply 동작

```bash
cd terraform
terraform plan
terraform apply
```

| 순서 | 내용 |
|------|------|
| 1 | (이미 있으면) `tokyo_cert` **Pending validation** |
| 2 | **add** `aws_route53_record.cert_validation[...]` |
| 3 | **add** `aws_acm_certificate_validation.tokyo_cert` — **수 분** 걸릴 수 있음 |
| 4 | 완료 후 ACM 콘솔 **`Issued`** |

**plan이 오래 걸리거나 apply가 멈춘 것처럼 보이면** — validation 리소스가 **Issued 대기** 중일 수 있다(정상).

---

## Issued가 안 될 때

| 원인 | 확인 |
|------|------|
| **NS 미설정 (88)** | Route53 존 CNAME이 **공인 DNS에 안 보임** |
| **가짜 도메인** | `kimty1121.com` 등 — 코드 실습은 OK, **Issued·HTTPS 테스트는 불가** |
| **zone_id 오타** | `route53_zone.id` vs **`zone_id`** — 레코드는 **`zone_id`** 권장 |
| **와일드카드·apex 불일치** | 인증서 `*.domain` 과 존 `domain` 이 **같은 존**에 있어야 함 |

91·88에서 **등록업체 NS → Route53 `name_servers`** 반영 후 재시도.

---

## 89·91·92와의 역할

| 강의 | 내용 |
|------|------|
| **89** | `aws_acm_certificate.tokyo_cert` |
| **90 (지금)** | 검증 **CNAME** + **`aws_acm_certificate_validation`** |
| **91·88** | 구매처 **NS·검증 확인** (수동) |
| **92** | HTTPS 리스너 `certificate_arn` |

## 정리

- DNS 검증 = **Route53 CNAME** + **`aws_acm_certificate_validation`**.
- CNAME 내용은 **`domain_validation_options`** 에서 **`for_each`** 로 생성.
- **`validation_record_fqdns`** = 등록한 검증 레코드의 **`fqdn`** 목록.
- **Issued** 후 **92 ELB HTTPS**에 ACM ARN 연결.
- 다음: **91 도메인 설정과 검증(수동)**.
