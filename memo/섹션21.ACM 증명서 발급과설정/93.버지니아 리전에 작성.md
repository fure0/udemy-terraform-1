# 93. 버지니아 리전에 작성

**CloudFront** 등에 쓸 ACM은 **us-east-1(버지니아)** 에만 만들 수 있다.  
도쿄(`ap-northeast-1`)의 **`tokyo_cert`(89~92·ALB)** 와 **별도**로, **provider alias** 로 버지니아에 **`virginia_cert`** 를 추가한다.

슬라이드: **복수 리전에 대한操作** — **provider 여러 정의** + **리소스마다 provider 지정(使い分け)**

이전: **92 ELB HTTPS** · 섹션21 **마지막(CloudFront 대비)**

---

## 왜 버지니아 ACM이 필요한가

| 용도 | ACM 리전 |
|------|----------|
| **ALB / NLB** (92) | **리소스와 같은 리전** → `ap-northeast-1` `tokyo_cert` |
| **CloudFront** | **반드시 `us-east-1`** → `virginia_cert` |

슬라이드: **증명서는 리전마다 생성이 필요.**

Route 53·검증 CNAME은 **글로벌** — 존은 하나(`kimty1121.com`)로 두고, **도쿄·버지니아 인증서 각각** 검증 레코드를 넣을 수 있다(90과 같은 패턴, **CNAME 값이 인증서마다 다를 수 있음**).

---

## 복수 리전操作 (슬라이드)

1. **provider 여러 정의** — 기본 + `alias`
2. **provider 이용(使い分け)** — 리소스에 **`provider = aws.별칭`** (메타 인수, 72번)

```
main.tf (기본)     → ap-northeast-1  → ALB, tokyo_cert, ...
provider aws.virginia → us-east-1       → virginia_cert (CloudFront용)
```

---

## provider 정의 (슬라이드)

`main.tf` 에 **기본 provider** 가 있다:

```hcl
provider "aws" {
  profile = "terraform"
  region  = "ap-northeast-1"
}
```

**별 리전용 provider** 추가 (슬라이드: 別リージョン向けの provider):

```hcl
provider "aws" {
  alias   = "virginia"
  profile = "terraform"
  region  = "us-east-1"
}
```

| provider | region | 용도 |
|----------|--------|------|
| **`aws`** (기본) | `ap-northeast-1` | VPC, ALB, `tokyo_cert` |
| **`aws.virginia`** | `us-east-1` | **`virginia_cert`** (CloudFront) |

`alias` 가 있는 provider는 **기본이 아님** — **`provider = aws.virginia`** 를 쓴 리소스만 버지니아에 생성.

---

## ACM 리소스 — provider 메타 인수 (슬라이드)

슬라이드: **메타 인수 `provider`로 기본 provider를 덮어쓴다.**

```hcl
resource "aws_acm_certificate" "virginia_cert" {
  provider = aws.virginia

  domain_name       = "*.${var.domain}"
  validation_method = "DNS"

  lifecycle {
    create_before_destroy = true
  }

  tags = {
    Name    = "${var.project}-${var.environment}-wildcard-sslcert-virginia"
    Project = var.project
    Env     = var.environment
  }
}
```

| 항목 | 내용 |
|------|------|
| **`provider = aws.virginia`** | 이 블록만 **us-east-1** ACM |
| **`domain_name`** | `tokyo_cert` 와 동일해도 됨 (`*.kimty1121.com`) |
| **`provider` 생략** | **도쿄**에 만들어짐 → CloudFront에 **붙일 수 없음** |

---

## DNS 검증 (90번 패턴 — 버지니아)

버지니아 인증서도 **Issued** 하려면 **90과 동일**하게 Route53 CNAME + validation.

```hcl
resource "aws_route53_record" "virginia_cert_validation" {
  for_each = {
    for dvo in aws_acm_certificate.virginia_cert.domain_validation_options : dvo.domain_name => {
      name   = dvo.resource_record_name
      type   = dvo.resource_record_type
      record = dvo.resource_record_value
    }
  }

  allow_overwrite = true
  zone_id         = aws_route53_zone.route53_zone.zone_id
  name            = each.value.name
  type            = each.value.type
  ttl             = 60
  records         = [each.value.record]
}

resource "aws_acm_certificate_validation" "virginia_cert_valid" {
  provider = aws.virginia

  certificate_arn = aws_acm_certificate.virginia_cert.arn
  validation_record_fqdns = [
    for record in aws_route53_record.virginia_cert_validation : record.fqdn
  ]
}
```

| 주의 | 내용 |
|------|------|
| **`aws_acm_certificate_validation`** | **`provider = aws.virginia`** — 검증 API도 **us-east-1** |
| **Route53 레코드** | **provider 생략** (기본) — Route53은 **리전 무관** |
| **도쿄·버지니아 CNAME** | **이름/값이 둘 다 다를 수 있음** — `for_each` 키·리소스명 **분리** |

`acm.tf` 의 `route53_acm_dns_resolve` 는 **`tokyo_cert` 전용** — 버지니아는 **별 레코드 블록**.

---

## tokyo vs virginia (이 프로젝트)

| | `tokyo_cert` | `virginia_cert` |
|---|--------------|-----------------|
| **provider** | 기본 (`ap-northeast-1`) | **`aws.virginia`** |
| **연결** | `elb_listener_https` (92) | **CloudFront**(후속 섹션) |
| **검증** | `cert_valid` | `virginia_cert_valid` (예) |
| **Route53** | 같은 `route53_zone` | 같은 존, **검증 CNAME 추가** |

CloudFront 섹션 전까지 **`virginia_cert`만 apply** 해두어도 됨.

---

## apply 전·후

```bash
cd terraform
terraform plan
```

| 기대 | 내용 |
|------|------|
| **add** | `aws_acm_certificate.virginia_cert` (us-east-1) |
| **add** | 검증 Route53 + `virginia_cert_valid` (작성 시) |
| **콘솔** | 리전을 **미국东部(버지니아)** 로 바꿔 ACM 확인 |

**88 NS** 가 이미 `kimty1121.com` 에 맞춰져 있으면, **도쿄와 같은 방식**으로 버지니아도 **Issued** 가능.

CloudFront만 안 할 거면 **93은 plan·코드 작성만** 하고 apply 생략해도 강의 목적에는 충분.

---

## 정리

- **CloudFront용 ACM = `us-east-1`만** — `provider "aws" { alias = "virginia" ... }`.
- 리소스에 **`provider = aws.virginia`** 로 **기본(도쿄) 덮어쓰기** (메타 인수).
- **ALB = tokyo_cert**, **CloudFront = virginia_cert** — **리전·인증서 둘 다** 필요할 수 있음.
- 검증은 **90과 동일** — Route53 CNAME + `aws_acm_certificate_validation` (**validation에도 virginia provider**).
- 섹션21 끝 — CloudFront 연동은 이후 섹션.
