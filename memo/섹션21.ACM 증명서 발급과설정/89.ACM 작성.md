# 89. ACM 작성

**AWS Certificate Manager(ACM)** 로 **TLS 인증서**를 Terraform **`aws_acm_certificate`** 로 요청한다.  
섹션21은 **HTTPS(ALB 443)** 를 위한 흐름이고, **89 → 90 DNS 검증 리소스 → 91·88 도메인/검증 → 92 ELB 리스너** 순이다.

슬라이드: **증명서는 리전마다 생성이 필요** — ALB와 **같은 리전**(`ap-northeast-1` 도쿄).

이전: **섹션20 Route53** · 다음: **90 DNS 검증 리소스**

---

## ACM이란 (슬라이드)

- Terraform → **AWS Cloud (Tokyo Region)** 에 **ACM 인증서** 생성
- **리전 리소스**: ALB에 붙일 인증서는 **ALB와 동일 리전**에 있어야 함
- **CloudFront**용은 **us-east-1(버지니아)** — **93번** 별도

이 프로젝트 provider는 **`ap-northeast-1`** (`main.tf`) → **89는 도쿄 ACM**.

---

## 증명서 발행 흐름 (슬라이드)

| 순서 | 내용 | 강의 |
|------|------|------|
| 1 | **증명서 요청 (ACM)** | **89 (지금)** — `aws_acm_certificate` |
| 2 | **DNS 검증 설정 (ACM)** | **90** — Route53 검증 레코드 + `aws_acm_certificate_validation` |
| 3 | **DNS 검증 (도메인 구매처)** | **91·88** — NS·검증이 **공인 DNS**에 보여야 **Issued** |

89만 `apply` 하면 상태는 **`PENDING_VALIDATION`** — **90·검증 완료 전**에는 ALB HTTPS(92)에 **아직 못 붙임**.

---

## ACM 데이터 구조 (슬라이드)

```
ACM 증명서 (aws_acm_certificate)           ← 89번
    ↑                    ↑
Route53 레코드 (검증 CNAME)    ACM 증명서 검증 (aws_acm_certificate_validation)
    ↑                              ↑
    └──────── ACM 증명서 ──────────┘
```

| 리소스 | 참조 | 역할 |
|--------|------|------|
| **검증 Route53 레코드** (90) | ACM | `domain_validation_options` 의 name/value |
| **certificate_validation** (90) | ACM + 레코드 | DNS 전파 후 **Issued** 대기 |
| **ALB HTTPS 리스너** (92) | ACM | `certificate_arn = aws_acm_certificate.xxx.arn` |

**89번은 `aws_acm_certificate`만** — 검증 레코드·validation은 **90**.

---

## `aws_acm_certificate` 주요 인자 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`domain_name`** * | string | 인증서 **도메인** (FQDN) |
| **`validation_method`** * | enum | **`"DNS"`**, `"EMAIL"`, `"NONE"` — 강의·Route53 연동은 **DNS** |
| **`tags`** | map | 태그 |
| **`lifecycle`** | block | 리소스 **생성·삭제·교체** 동작 제어 (76번) |

### `lifecycle.create_before_destroy` (슬라이드 강조)

**ELB에서 인증서를 쓸 때 `create_before_destroy = true` 권장.**

| 값 | 의미 |
|----|------|
| **`true`** | 갱신·교체 시 **새 인증서를 먼저 만든 뒤** 옛것 삭제 → ALB **HTTPS 끊김** 완화 |
| **`false`** | 기본 — 교체 시 **삭제 후 생성** 구간 가능 |

## HCL 예시

`acm.tf` 등 새 파일:

```hcl
resource "aws_acm_certificate" "main" {
  domain_name       = var.domain
  validation_method = "DNS"

  lifecycle {
    create_before_destroy = true
  }

  tags = {
    Name    = "${var.project}-${var.environment}-cert"
    Project = var.project
    Env     = var.environment
  }
}
```

### 이 프로젝트와 맞추기

| 항목 | 값 |
|------|-----|
| **`domain_name`** | `var.domain` — `terraform.tfvars` 의 `kimty1121.com` |
| **리전** | `ap-northeast-1` (ALB `aws_lb.alb` 와 동일) |
| **Route53** | `aws_route53_zone.main` (86) — **90**에서 검증 CNAME 추가 |

`dev-elb.${var.domain}`(87 ALB Alias)과 **인증서 도메인**이 다를 수 있다.  
HTTPS로 **`https://kimty1121.com`** 를 쓰려면 **`domain_name = var.domain`** (apex).  
서브도메인만 쓰면 `domain_name = "dev-elb.${var.domain}"` 등으로 맞춘다(강의·요구에 따름).

`subject_alternative_names` — `www` 등 추가 도메인 필요 시 provider 문서 참고(슬라이드 기본 범위 밖).

## apply 후 상태

```bash
cd terraform
terraform plan
terraform apply
```

| 단계 | 내용 |
|------|------|
| **plan** | `aws_acm_certificate.main` **add** |
| **apply 직후** | ACM 콘솔 상태 **`Pending validation`** |
| **출력** | `domain_validation_options` — **90**에서 Route53 CNAME에 사용 |

```hcl
# 90번에서 참조 (개념)
# aws_acm_certificate.main.domain_validation_options
```

**Issued** 되려면 검증 CNAME이 **인터넷 DNS에서 조회** 가능해야 한다.  
**88 NS 미설정·가짜 도메인**이면 89·90 코드 실습은 되지만 **발급 완료·HTTPS 접속 테스트**는 어렵다(섹션20 대화와 동일).

---

## 90·92와의 역할

| 강의 | 리소스 | 역할 |
|------|--------|------|
| **89 (지금)** | `aws_acm_certificate` | 인증서 **요청** |
| **90** | `aws_route53_record`(검증) + `aws_acm_certificate_validation` | **DNS 검증 자동화** |
| **91·88** | 수동 | 구매처 NS·검증 확인 |
| **92** | `aws_lb_listener` HTTPS | `certificate_arn` 에 ACM ARN |

현재 `terraform/` 에는 **ACM 코드 없음** — 89 실습에서 추가.

---

## 주의

| 항목 | 내용 |
|------|------|
| **리전** | ALB ≠ ACM 리전이면 **리스너에 ARN 붙일 수 없음** |
| **EMAIL 검증** | Route53 Terraform 자동화와 잘 안 맞음 → **DNS 권장** |
| **유료** | ACM **퍼블릭 인증서**는 **무료** (갱신도 ACM) |
| **destroy** | ELB 리스너가 인증서를 쓰는 중이면 삭제 순서 주의 — `create_before_destroy` 와 92 연동 |

---

## 정리

- **`aws_acm_certificate`**: `domain_name`, `validation_method = "DNS"`, ELB용 **`lifecycle { create_before_destroy = true }`**.
- **리전 = ALB 리전** (도쿄).
- 89만으로는 **Pending** — **90 검증** 후 **Issued** → **92 HTTPS**.
- 다음: **90 DNS 검증 리소스 작성**.
