# 79. 리소스 의존 관계 가시성 (연습)

77번 **개요**와 78번 **Graphviz Interactive Preview 설치**를 전제로, 이 저장소 `terraform/` 구성으로 **DOT 생성 → VS Code에서 SVG 확인**까지 한다.

슬라이드 요약:

1. **`terraform graph`로 DOT 파일 생성**
2. **생성된 DOT 파일을 Visual Studio Code에서 확인**

이전: **77 개요** · **78 VS Code 확장** · 섹션17 마지막 실습

---

## 사전 준비

| 항목 | 확인 |
|------|------|
| **78** | VS Code 확장 **Graphviz Interactive Preview** 설치 |
| **Graphviz 본체** | 확장이 `dot` CLI를 쓰는 경우가 많음 — macOS: `brew install graphviz` (미리보기 실패 시 설치) |

## Step 1 — DOT 파일 생성

프로젝트 루트가 아니라 **`terraform/`** 에서 실행한다.

```bash
cd terraform
terraform graph > graph.dot
```

| 확인 | 내용 |
|------|------|
| 파일 | `terraform/graph.dot` 생성 |
| 내용 | 맨 위가 `digraph {` 형태, 노드에 `aws_vpc`, `aws_db_instance` 등이 보임 |

에러 예:

| 메시지 | 대응 |
|--------|------|
| `No configuration files` | `cd terraform` |
| provider / STS 오류 | AWS 프로필·환경 변수 확인 (`plan` 과 동일) |

**코드만 바꾼 뒤**에는 graph를 **다시 생성**해야 반영된다.

```bash
terraform graph > graph.dot
```

`.tf` 수정 → `graph.dot` 덮어쓰기 → VS Code에서 미리보기 갱신.

## Step 2 — VS Code에서 DOT 열기

1. VS Code에서 이 워크스페이스(`udemy_terraform`) 연다.
2. **`terraform/graph.dot`** 파일을 연다.
3. 처음에는 **텍스트만** 보이는 것이 정상이다.

---

## Step 3 — 그래프 미리보기 (SVG)

슬라이드: **「コマンドパレットから起動」** — **명령 팔레트에서 실행**.

1. `graph.dot` 을 연 상태에서 **명령 팔레트** 열기  
   - macOS: `Cmd + Shift + P`  
   - Windows: `Ctrl + Shift + P`
2. `Graphviz` 로 검색해 아래 중 하나 실행 (확장·버전에 따라 표기가 조금 다를 수 있음):
   - **Graphviz: Open Preview**
   - **Graphviz: Open Preview to the Side**
3. 옆/탭에 **다이어그램(SVG)** 이 열리면 성공.

```
코드(.tf) → terraform graph → graph.dot → [명령 팔레트] Graphviz Preview → SVG
```

미리보기가 안 되면:

- 78번 확장 설치 여부
- 시스템에 **Graphviz** (`dot` 명령) 설치 여부
- `graph.dot` 문법 오류(생성 실패 시 파일이 비었는지)

---

## CLI로 SVG 저장 (선택)

VS Code 없이 파일로 남기려면:

```bash
cd terraform
dot -Tsvg graph.dot -o graph.svg
```

`graph.svg` 를 브라우저·미리보기 앱으로 연다. 강의는 **VS Code 연습**이 중심이다.

---

## 그래프 읽는 법 (이 프로젝트)

노드가 많아도 **화살표 방향**만 보면 된다. 대략 아래 **참조·의존**이 보인다.

| 흐름 | 의미 |
|------|------|
| `aws_subnet` → `aws_db_subnet_group` | 서브넷 ID를 subnet group이 사용 |
| subnet group / SG / parameter group → `aws_db_instance.mysql_standalone` | RDS 생성 조건 |
| `mysql_standalone` → `aws_ssm_parameter` (host, port 등) | DB 주소·포트를 Parameter Store에 기록 |
| `aws_vpc`, `aws_subnet`, SG, `data.aws_ami` → `aws_instance.app_server` | AP 서버 위치·보안·AMI |
| `aws_iam_role` → `aws_iam_instance_profile` → `app_server` | EC2에 붙는 Role 경로 |

**같은 `.tf` 파일에 없어도** 한 그래프에 합쳐진다 (`network.tf` + `rds.tf` + …).

### 73번과 연계해 보기

`aws_instance`에 `depends_on`을 추가한 뒤:

```bash
terraform graph > graph.dot
```

미리보기를 다시 열어 **간선·노드**가 늘었는지 비교한다.  
참조(`subnet_id = ...`)만 있는 관계는 원래도 그래프에 있었다.

## 파일 관리 (참고)

`graph.dot` / `graph.svg` 는 **생성물**이라 Git에 넣지 않아도 된다.  
반복 생성 시:

```bash
# terraform/ 에서
terraform graph > graph.dot
```

`.gitignore`에 `*.dot`, `graph.svg` 를 넣는 팀도 있다(강의 필수 아님).
