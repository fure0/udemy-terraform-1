# 77. 리소스 의존 관계 가시성 (개요)

섹션16 **73 `depends_on`** 에서 **의존 관계를 코드로 정의**했다면, 이번 섹션은 그 관계를 **그림으로 보는** 방법이다.  
Terraform CLI와 **Graphviz**로 **DOT → SVG** 흐름을 만든다.

다음: **78 VS Code·Graphviz 설치** · **79 연습**

---

## 왜 가시화하나

`.tf`가 늘면 **누가 누구를 참조하는지** 한눈에 보기 어렵다.  
`terraform graph`로 **실행 계획 그래프**를 뽑아, VPC → 서브넷 → RDS → EC2 같은 **순서·연결**을 확인한다.

- **암시적 의존**: `aws_subnet.xxx.id` 같은 **속성 참조**
- **명시적 의존**: **`depends_on`** (73번)

둘 다 그래프에 반영된다(표현 방식은 DOT 노드·간선).

---

## 전체 흐름 (슬라이드)

```
코드(.tf)  ──terraform graph──▶  DOT 파일  ──Graphviz──▶  SVG 이미지
```

| 단계 | 도구 | 결과 |
|------|------|------|
| 1 | Terraform **코드** | `terraform/` 아래 `.tf` |
| 2 | **`terraform graph`** | **DOT** (그래프 묘사 언어) |
| 3 | **Graphviz** (또는 VS Code 확장) | **SVG** 등 시각화 |

슬라이드 표현:

- DOT = **「그래프 묘사」언어** — 노드·화살표를 텍스트로 기술
- **`terraform graph` 명령을 이용**
- 렌더링은 **Graphviz** 또는 **Visual Studio Code 플러그인** (78번: **Graphviz Interactive Preview** 등)

---

## `terraform graph` — DOT 파일 생성

**리소스 생성·실행 계획의 의존 관계**를 **DOT 형식**으로 **표준 출력**한다.

```bash
cd terraform
terraform graph > graph.dot
```

| 항목 | 내용 |
|------|------|
| **인자 (기본)** | 없음 (슬라이드) |
| **리다이렉트** | `> <파일명>` 으로 `.dot` 저장 |
| **사전 조건** | `terraform init` 완료, 구성이 유효할 것 |

화면에만 보려면:

```bash
terraform graph
```

`graph.dot`을 열면 `digraph` 로 시작하는 텍스트가 보인다. **사람이 읽기엔 어렵고**, 78~79번에서 **미리보기·SVG**로 본다.

---

## Graphviz / VS Code 쪽 (개요만)

DOT 파일만으로는 **다이어그램이 안 보인다**. 아래 중 하나가 필요하다.

| 방법 | 설명 |
|------|------|
| **Graphviz** (`dot` CLI) | `dot -Tsvg graph.dot -o graph.svg` (78번 설치) |
| **VS Code 확장** | **Graphviz Interactive Preview** — `.dot` 미리보기 (슬라이드) |

78번에서 설치, 79번에서 이 프로젝트 `graph.dot`으로 **연습**.

---

## 이 프로젝트에서 기대되는 모습 (참고)

`terraform/` 에는 네트워크·SG·RDS·IAM·EC2·SSM 등이 있어, 그래프에는 대략 이런 **참조 방향**이 잡힌다.

| 예 | 의존 방향 |
|----|-----------|
| `aws_db_subnet_group` | `aws_subnet.private_*` |
| `aws_db_instance` | subnet group, SG, parameter group |
| `aws_ssm_parameter` | `aws_db_instance` 주소·포트 등 |
| `aws_instance.app_server` | subnet, SG, AMI data, IAM instance profile |
| `aws_iam_instance_profile` | `aws_iam_role` |

파일이 여러 개여도 **하나의 그래프**로 합쳐진다.

---

## 73번 `depends_on`과의 관계

| | 73번 | 77~79번 |
|---|------|---------|
| 목적 | **순서를 코드로 강제** | **이미 정의된 그래프를 눈으로 확인** |
| 수단 | `depends_on = [...]` | `terraform graph` + Graphviz |

`depends_on`을 추가·삭제한 뒤 **graph를 다시 생성**하면, 간선·노드 변화를 비교할 수 있다.

---

## 자주 하는 실수

| 증상 | 원인 |
|------|------|
| `No configuration files` | `terraform/` 밖에서 실행 |
| graph가 비거나 에러 | `init` 미실행, 구문 오류 |
| DOT만 있고 그림 없음 | **Graphviz·VS Code 확장** 미설치 (78번) |

---

## 정리

- **가시화 파이프라인**: **코드 → `terraform graph` → DOT → Graphviz/VS Code → SVG**
- **핵심 명령**: `terraform graph > graph.dot` (인자 없음)
- DOT은 **중간 형식**, 최종 확인은 **78 설치 · 79 연습**
- 섹션16 **73 `depends_on`** 과 짝으로 보면, **정의**와 **확인**이 맞는다.
