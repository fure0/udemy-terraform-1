# 96. 프라이빗 버킷 작성

**배포용 자재(소스·아티팩트)** 를 넣을 S3 버킷을 **인터넷에 공개하지 않고** 만든다.  
94번 중 **버킷 + public access block** 만 쓰고, **95와 달리 bucket policy·policy document는 없음**.

슬라이드: **デプロイ用資材配信準備**

1. **소스 코드 배포용 S3 버킷 생성**
2. **S3에 배포용 자재 배치** (2번은 apply 후 수동·별 강의)

이전: **95 일반 공개 버킷** · 다음: 섹션22 이후(CloudFront·CodeDeploy 등)

---

## 목표

| 항목 | 내용 |
|------|------|
| **용도** | 앱 **배포 패키지·소스 ZIP** 등 (EC2/CodeDeploy가 가져감) |
| **접근** | **IAM** (EC2 Role `AmazonS3ReadOnlyAccess` 등) — **익명 URL 불가** |
| **이름** | 전역 유일 — 슬라이드: **`tastylog-dev-deploy-bucket-<RANDOM>`** |

`<RANDOM>` 은 95와 같이 **`random_string`** 으로 suffix 부여.

---

## 버킷 이름 (슬라이드)

```
tastylog-dev-deploy-bucket-<RANDOM>
```

| 부분 | 이 프로젝트 |
|------|-------------|
| prefix | `${var.project}-${var.environment}-deploy-bucket` |
| suffix | `random_string.s3_deploy_unuque_key.result` (예) |

예: `tastylog-dev-deploy-bucket-k9m2p1`

| 버킷 | 용도 |
|------|------|
| **static** (95) | CSS·JS·이미지 — **공개 읽기** |
| **deploy** (96) | 배포 자재 — **프라이빗** |
| **tfstate** | Terraform state — **별도·수동 생성** |

---

## 프라이빗 버킷 구성 (94번 중 2종)

```
aws_s3_bucket (deploy)
    ↑
aws_s3_bucket_public_access_block  ← 4종 전부 true
```

| 리소스 | 96에서의 역할 |
|--------|----------------|
| **`aws_s3_bucket`** | deploy 버킷 본체 |
| **`aws_s3_bucket_public_access_block`** | **퍼블릭 액세스 전부 차단** |
| ~~`data.aws_iam_policy_document`~~ | **없음** |
| ~~`aws_s3_bucket_policy`~~ | **없음** |

읽기 권한은 **버킷 정책이 아니라 IAM Role** (12·13번 `app_iam_role` + `AmazonS3ReadOnlyAccess`).

---

## Public Access Block — 95 vs 96

**96 프라이빗**: 네 가지 **모두 `true`**.

| 항목 | 95 (공개 static) | **96 (프라이빗 deploy)** |
|------|-------------------|--------------------------|
| `block_public_acls` | `true` | **`true`** |
| `ignore_public_acls` | `true` | **`true`** |
| `block_public_policy` | **`false`** | **`true`** |
| `restrict_public_buckets` | **`false`** | **`true`** |

95에서 **`block_public_policy = true`** 를 deploy에 쓰면 OK.  
**95 공개 버킷**에 쓰면 퍼블릭 policy apply **403** (95 실습에서 겪은 케이스).

## HCL 예 (`s3.tf` — 95 아래에 추가)

```hcl
resource "random_string" "s3_deploy_unuque_key" {
  length  = 6
  upper   = false
  special = false
}

# -------------------------------------------------
# S3 deploy bucket (private)
# -------------------------------------------------

resource "aws_s3_bucket" "s3_deploy_bucket" {
  bucket = "${var.project}-${var.environment}-deploy-bucket-${random_string.s3_deploy_unuque_key.result}"
}

resource "aws_s3_bucket_public_access_block" "s3_deploy_bucket" {
  bucket = aws_s3_bucket.s3_deploy_bucket.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```

- **`depends_on` 불필요** — bucket policy가 없으므로 95의 policy ↔ block 순서 이슈 없음.
- **`force_destroy`·`tags`** — 강의·실습에 맞게 95와 동일하게 추가 가능.

## 2단계 — 배포 자재 배치 (슬라이드)

apply 후 **객체 업로드** (강의에서 수동 또는 CLI):

```bash
aws s3 cp ./deploy-artifacts s3://tastylog-dev-deploy-bucket-xxxxx/ --recursive
```

| 접근 방법 | 95 static | **96 deploy** |
|-----------|-----------|---------------|
| **퍼블릭 URL** | 가능 (policy) | **불가** |
| **EC2 + IAM Role** | 가능 (S3ReadOnly) | **가능 (주 사용처)** |
| **내 PC AWS CLI** | 업로드·다운로드 (자격 증명) | 동일 |

EC2에서 확인 예 (Role 부여 후):

```bash
aws s3 ls s3://tastylog-dev-deploy-bucket-xxxxx/
```

---

## apply 전·후

```bash
cd terraform
terraform plan
terraform apply
```

| 확인 | 내용 |
|------|------|
| **plan** | deploy bucket + public_access_block **add** (policy 없음) |
| **콘솔** | Block public access **4개 ON** · 버킷 정책 **없음** |
| **테스트** | 퍼블릭 URL GET → **403** · EC2/CLI(IAM) → **200/list OK** |

---

## 95와 차이 (한눈에)

| | **95 공개 (static)** | **96 프라이빗 (deploy)** |
|---|----------------------|---------------------------|
| **슬라이드 이름** | `static-bucket-<RANDOM>` | **`deploy-bucket-<RANDOM>`** |
| **policy** | `GetObject` Allow `*` | **없음** |
| **public access block** | 정책 **허용** (`false` 2개) | **전부 차단** (`true` 4개) |
| **리소스 수** | 4종 + random | **2종 + random** |
| **용도** | 정적 웹·CDN 오리진 | **배포 패키지 저장** |

---

## 주의

| 항목 | 내용 |
|------|------|
| **95·96 혼동** | deploy는 **`block_public_policy = true`** — static(95) 설정을 그대로 복사하지 않기 |
| **`AmazonS3ReadOnlyAccess`** | **계정 내 모든 S3 읽기** — 실습용. 운영은 버킷 ARN으로 **인라인 policy** 제한 |
| **버킷 이름** | static·deploy **random suffix 각각** — 한 `random_string` 공유해도 되나 분리 권장 |
| **provider 4+** | `versioning { }` 를 `aws_s3_bucket` 안에 넣지 않음 (94·95 참고) |

---

## 정리

- **96** = `tastylog-dev-deploy-bucket-<RANDOM>` + **public access block 4종 ON** + **bucket policy 없음**.
- 슬라이드 2단계: **버킷 Terraform** → **배포 파일 업로드**.
- 접근은 **IAM(EC2 Role)** — 12번 `app_iam_role_s3_readonly` 와 연결.
- **95**는 공개 정적, **96**은 프라이빗 배포 — **이름·block·policy** 세 가지로 구분.
