# 95. 일반 공개 버킷 작성

**정적 컨텐츠 배포**용 S3 버킷을 **인터넷에서 읽을 수 있게** 만든다.  
94번 S3 4종(버킷 · public access block · policy document · bucket policy)을 **공개 읽기** 조합으로 쓴다.

슬라이드: **静的コンテンツ配信準備**

1. **정적 컨텐츠 배포용 S3 버킷 생성**
2. **S3에 정적 컨텐츠 배치** (2번은 apply 후 수동·별 강의)

이전: **94 S3 리소스** · 다음: **96 프라이빗 버킷**

---

## 목표

| 항목 | 내용 |
|------|------|
| **용도** | CSS·JS·이미지 등 **정적 파일** (CloudFront·웹 앞단) |
| **접근** | **익명 `s3:GetObject`** (버킷 정책) |
| **이름** | 전역 유일 — 슬라이드: **`tastylog-dev-static-bucket-<RANDOM>`** |

`<RANDOM>` 은 **다른 계정과 겹치지 않게** 붙인다. Terraform에서는 **`random_string`** (45·RDS 패스워드와 같은 방식).

---

## 버킷 이름 (슬라이드)

```
tastylog-dev-static-bucket-<RANDOM>
```

| 부분 | 이 프로젝트 |
|------|-------------|
| prefix | `${var.project}-${var.environment}-static-bucket` |
| suffix | `random_string.static_bucket_suffix.result` |

예: `tastylog-dev-static-bucket-a1b2c3d4`

**tfstate 버킷**(`tastylog-tfstate-bucket-kimty`)과 **이름·용도 모두 다름**.

---

## 공개 버킷 구성 (94번 조합)

```
aws_s3_bucket (static)
    ↑              ↑
public_access_block   bucket_policy ← data.aws_iam_policy_document (GetObject Allow *)
```

| 리소스 | 95에서의 역할 |
|--------|----------------|
| **`aws_s3_bucket`** | static 버킷 본체 |
| **`aws_s3_bucket_public_access_block`** | **퍼블릭 정책 허용** 쪽으로 조정 (아래 표) |
| **`data.aws_iam_policy_document`** | `Principal *` + `s3:GetObject` |
| **`aws_s3_bucket_policy`** | 정책 부착 + **`depends_on`** public access block |

---

## Public Access Block — 공개 vs 96 프라이빗

**96 프라이빗**: 네 가지 **모두 `true`**.

**95 공개**(강의·정적 배포): **퍼블릭 버킷 정책**을 쓰려면 아래처럼 **정책 관련만 `false`** 인 경우가 많다.

| 항목 | 95 (공개) | 96 (프라이빗) |
|------|-----------|---------------|
| `block_public_acls` | `true` | `true` |
| `ignore_public_acls` | `true` | `true` |
| `block_public_policy` | **`false`** | `true` |
| `restrict_public_buckets` | **`false`** | `true` |

`block_public_policy = true` 이면 **익명 읽기 policy apply 실패**할 수 있음 (94번).

## HCL 예 (`s3.tf`)

```hcl
resource "random_string" "static_bucket_suffix" {
  length  = 8
  special = false
  upper   = false
}

resource "aws_s3_bucket" "static" {
  bucket = "${var.project}-${var.environment}-static-bucket-${random_string.static_bucket_suffix.result}"

  force_destroy = true

  tags = {
    Name    = "${var.project}-${var.environment}-static-bucket"
    Project = var.project
    Env     = var.environment
  }
}

resource "aws_s3_bucket_public_access_block" "static" {
  bucket = aws_s3_bucket.static.id

  block_public_acls       = true
  block_public_policy     = false
  ignore_public_acls      = true
  restrict_public_buckets = false
}

data "aws_iam_policy_document" "static_public_read" {
  statement {
    sid       = "PublicReadGetObject"
    effect    = "Allow"
    actions   = ["s3:GetObject"]
    resources = ["${aws_s3_bucket.static.arn}/*"]

    principals {
      type        = "*"
      identifiers = ["*"]
    }
  }
}

resource "aws_s3_bucket_policy" "static" {
  bucket = aws_s3_bucket.static.id
  policy = data.aws_iam_policy_document.static_public_read.json

  depends_on = [aws_s3_bucket_public_access_block.static]
}
```

**provider 4+** 이면 `aws_s3_bucket_versioning` 등은 강의·plan에 맞게 추가.

## 2단계 — 정적 컨텐츠 배치 (슬라이드)

apply 후 **객체 업로드** (강의에서 수동 또는 CLI):

```bash
aws s3 cp ./local-path s3://tastylog-dev-static-bucket-xxxxx/ --recursive
```

또는 콘솔 **업로드**. URL 예:

```
https://<bucket>.s3.ap-northeast-1.amazonaws.com/<key>
```

CloudFront 연동은 **이후 섹션**에서 이 버킷을 **오리진**으로 쓸 수 있다.

---

## apply 전·후

```bash
cd terraform
terraform plan
terraform apply
```

| 확인 | 내용 |
|------|------|
| **plan** | bucket, public_access_block, policy **add** |
| **콘솔** | 버킷 · 퍼블릭 액세스 · 버킷 정책 |
| **테스트** | 객체 업로드 후 **퍼블릭 URL**로 GET (403이면 policy·block 재확인) |

## 주의

| 항목 | 내용 |
|------|------|
| **보안** | **전 세계 읽기** — 실습용. 운영은 CloudFront·OAC 등으로 제한 |
| **`force_destroy = true`** | destroy 시 **객체 포함** 삭제 — 실습 편의 |
| **버킷 이름 변경** | **replace** (새 버킷) — suffix random 유지 권장 |
| **IAM `s3_readonly`** | EC2 Role은 **별도** — 이 버킷 **정책 공개**와 무관 |

## 96과 차이

| | **95 공개 (static)** | **96 프라이빗** |
|---|----------------------|-----------------|
| **policy** | `GetObject` **Allow *** | 없거나 Deny·IAM만 |
| **public access block** | 정책 **허용** 쪽 | **전부 차단** |
| **용도** | 정적 웹·CDN 오리진 | 앱 전용·내부 데이터 |

## 정리

- **95** = `tastylog-dev-static-bucket-<RANDOM>` + **공개 읽기 policy** + public access **정책 허용**.
- 슬라이드 2단계: **버킷 Terraform** → **파일 업로드**.
- **`depends_on`**: bucket policy → public access block (73·94).
- 다음: **96 프라이빗 버킷**.
