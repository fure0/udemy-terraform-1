# 94. S3 리소스

섹션22는 **S3 버킷(스트레이지)** 를 Terraform으로 만든다.  
**94번**은 S3 관련 **리소스·data 구조와 인자 개요** — **95 일반 공개 버킷** · **96 프라이빗 버킷** 실습의 기반.

슬라이드: **S3のデータ構造**

다음: **95 일반 공개 버킷** · **96 프라이빗 버킷**

---

## S3 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (73번·Route53과 동일).

```
                    버킷 (aws_s3_bucket)
                   ↗              ↗
  Public Access Block          Bucket Policy
  (aws_s3_bucket_public_access_block)   (aws_s3_bucket_policy)
                                              ↑
                                    Policy Document (data.aws_iam_policy_document)
```

| 구성 | Terraform | 참조 |
|------|-----------|------|
| **버킷** | `aws_s3_bucket` | (루트) |
| **퍼블릭 액세스 차단** | `aws_s3_bucket_public_access_block` | `bucket = aws_s3_bucket.xxx.id` |
| **버킷 정책** | `aws_s3_bucket_policy` | `bucket = ...`, `policy = data....json` |
| **정책 JSON** | `data "aws_iam_policy_document"` | bucket policy가 **참조** |

**73번 `depends_on`**: Public Access Block과 Bucket Policy가 **서로 참조하지 않을 때**, policy 쪽에 `depends_on = [aws_s3_bucket_public_access_block.xxx]` 를 두는 패턴(슬라이드·강의).

## S3 관련 리소스 한눈에

| 리소스 / data | 역할 |
|---------------|------|
| **`aws_s3_bucket`** | 버킷 **본체** (이름·태그·force_destroy 등) |
| **`aws_s3_bucket_public_access_block`** | 콘솔 **「퍼블릭 액세스 차단」** 4종 |
| **`data.aws_iam_policy_document`** | 버킷 정책 **JSON** 생성 |
| **`aws_s3_bucket_policy`** | 버킷에 **정책 부착** |

버전 관리·암호화 등은 슬라이드·강의에 따라 **별 리소스**(`aws_s3_bucket_versioning` 등)로 나뉠 수 있음(AWS provider 4+).

## `aws_s3_bucket` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`bucket`** | string | **버킷 이름** (전역 유일) |
| **`force_destroy`** | bool | **객체가 있어도** 버킷 삭제 허용 여부 |
| **`versioning`** | block | **버전 관리** (`enabled` bool) — 슬라이드 |
| **`tags`** | map | 태그 |

### HCL 예 (개념)

```hcl
resource "aws_s3_bucket" "app" {
  bucket = "${var.project}-${var.environment}-app-bucket"

  force_destroy = true

  versioning {
    enabled = true
  }

  tags = {
    Name    = "${var.project}-${var.environment}-app-bucket"
    Project = var.project
    Env     = var.environment
  }
}
```

**provider 4+** 에서 `versioning` 블록이 **`aws_s3_bucket`에서 제거**된 경우:

```hcl
resource "aws_s3_bucket_versioning" "app" {
  bucket = aws_s3_bucket.app.id
  versioning_configuration {
    status = "Enabled"
  }
}
```

강의·`terraform plan` 에러에 맞춰 **슬라이드 방식 vs 분리 리소스** 중 하나를 쓴다.

## `aws_s3_bucket_public_access_block` (슬라이드)

콘솔 **「Block public access (bucket settings)」** 에 해당.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`bucket`** | string | 대상 **버킷 ID/이름** |
| **`block_public_acls`** | bool | **새** 퍼블릭 ACL 차단 |
| **`block_public_policy`** | bool | **새** 퍼블릭 버킷 정책 차단 |
| **`ignore_public_acls`** | bool | **기존** 퍼블릭 ACL 무시 |
| **`restrict_public_buckets`** | bool | 퍼블릭 정책이 있어도 **소유자·AWS 서비스만** 접근 |

### 전부 ON (프라이빗·96)

```hcl
resource "aws_s3_bucket_public_access_block" "app" {
  bucket = aws_s3_bucket.app.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```

**95 공개 버킷**은 일부를 `false` 로 두거나, 정책과 조합(강의 95).

## `data.aws_iam_policy_document` (슬라이드)

버킷 정책 **JSON**을 HCL로 작성. **data** 블록(26번).

| 항목 | 타입 | 설명 |
|------|------|------|
| **`version`** | string | `"2012-10-17"` 등 |
| **`policy_id`** | string | 정책 ID (선택) |
| **`statement`** | block | 정책 본문 (반복) |

### `statement` 블록

| 항목 | 타입 | 설명 |
|------|------|------|
| **`sid`** | string | 문(statement) ID |
| **`effect`** | enum | `"Allow"` / `"Deny"` |
| **`actions`** | string[] | 예: `s3:GetObject` |
| **`resources`** | string[] | 대상 ARN (버킷·객체) |
| **`principals`** | block | **누구에게** (`type`, `identifiers`) |

슬라이드 JSON 예 (공개 읽기 개념):

```json
{
  "Version": "2012-10-17",
  "Statement": [{
    "Action": ["s3:GetObject"],
    "Effect": "Allow",
    "Resource": "*",
    "Principal": "*"
  }]
}
```

Terraform 예:

```hcl
data "aws_iam_policy_document" "public_read" {
  statement {
    sid       = "PublicReadGetObject"
    effect    = "Allow"
    actions   = ["s3:GetObject"]
    resources = ["${aws_s3_bucket.app.arn}/*"]

    principals {
      type        = "*"
      identifiers = ["*"]
    }
  }
}
```

실무에서는 **`Resource`를 버킷 ARN으로 한정** (`arn/*`). 슬라이드 `"*"` 는 개념용.

## `aws_s3_bucket_policy` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`bucket`** | string | 버킷 ID |
| **`policy`** | string | 정책 **JSON** (`data.aws_iam_policy_document.xxx.json`) |

```hcl
resource "aws_s3_bucket_policy" "app" {
  bucket = aws_s3_bucket.app.id
  policy = data.aws_iam_policy_document.public_read.json

  depends_on = [aws_s3_bucket_public_access_block.app]
}
```

**Public Access Block** 이 `block_public_policy = true` 이면 **퍼블릭 정책 apply 실패**할 수 있음 — **95·96**에서 공개/비공개 조합을 맞춘다.

## 이 프로젝트와의 관계

| 항목 | 현재 |
|------|------|
| **S3 버킷 리소스** | `terraform/` 에 **아직 없음** — 섹션22에서 `s3.tf` 등 추가 |
| **tfstate backend** | `main.tf` **S3** `tastylog-tfstate-bucket-kimty` — **Terraform이 만든 앱 버킷과 별개** |
| **IAM** | `app_iam_role_s3_readonly` — EC2가 S3 **읽기** (12·13) |
| **SG** | `data.aws_prefix_list.s3_pl` — AP 서버 **S3 prefix list** 아웃바운드 |

정적 파일·이미지 호스팅용 **앱 버킷**은 95·96에서 위 4종 조합으로 구성.

3. `data.aws_iam_policy_document` (bucket ARN 참조 가능)

## 정리

- S3 Terraform = **버킷 + (퍼블릭 액세스 차단) + policy document(data) + bucket policy**.
- **참조 방향**: block·policy → **버킷**, policy → **iam_policy_document**.
- **`force_destroy`**: 실습 종료 시 빈 버킷 없이 destroy 하려면 `true` (주의).
