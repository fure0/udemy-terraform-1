# 42. 파라미터그룹 작성

## 1) RDS 구성에서 파라미터 그룹의 위치

RDS 인스턴스는 다음과 같은 하위 설정을 참조한다.

- 파라미터 그룹(Parameter Group)
- 옵션 그룹(Option Group)
- 서브넷 그룹(Subnet Group)

이번 단계에서는 DB 엔진 동작값을 제어하는 **파라미터 그룹**을 먼저 생성한다.

## 2) `aws_db_parameter_group` 개념

Terraform 리소스: `aws_db_parameter_group`

주요 속성:

- `name`: 파라미터 그룹 이름
- `family`: 엔진 패밀리 (예: `mysql8.0`, `postgres12`)
- `parameter` 블록: 실제 DB 파라미터 설정 (`name`, `value`)

## 3) 현재 코드 작성 내용

`terraform/rds.tf`

```hcl
resource "aws_db_parameter_group" "db_param_group" {
  name   = "${var.project}-${var.environment}-mysql-standalone-parametergroup"
  family = "mysql8.0"

  parameter {
    name  = "character_set_database"
    value = "utf8mb4"
  }

  parameter {
    name  = "character_set_server"
    value = "utf8mb4"
  }
}
```

## 4) 설정 의도

- `family = "mysql8.0"`  
  → MySQL 8.0 계열 RDS 인스턴스에 맞는 파라미터 그룹 지정

- `character_set_database = utf8mb4`  
  → 기본 DB 문자셋을 UTF-8(4바이트)로 설정

- `character_set_server = utf8mb4`  
  → 서버 기본 문자셋도 UTF-8(4바이트)로 설정

즉, 데이터베이스/서버 문자셋을 모두 `utf8mb4`로 맞춰 한글 및 이모지 등 멀티바이트 문자를 안정적으로 저장하도록 구성한다.

## 5) 체크 포인트

- `family` 값은 실제 RDS 엔진 버전과 반드시 호환되어야 한다.
- 파라미터 변경 시 즉시 반영되지 않고, 재부팅/유지보수 창에 반영되는 항목이 있을 수 있다.
