# 51. RDS 데이터투입(수동)

이 단계는 발판 서버를 통해 RDS에 초기 데이터를 넣는 실습이다.  
핵심은 **SQL 파일을 발판 서버로 옮기고**, RDS 접속 정보에 맞게 수정한 뒤, `mysql`로 실행하는 것이다.

---

## 작업 순서

1. 리소스(SQL) 파일을 발판 서버로 전송
2. 접속 정보(host/user/password/db) 수정
3. RDS에 데이터 투입
4. 데이터 확인
5. 후처리(임시 EC2/키 정리)

---

## 1) SQL 파일 발판 서버로 전송

로컬에서 발판 서버로 파일 복사:

```bash
scp -i ~/.ssh/rds-bastion ./init.sql ec2-user@<BASTION_PUBLIC_IP>:/home/ec2-user/
```

여러 파일이면 디렉터리째 전송:

```bash
scp -i ~/.ssh/rds-bastion -r ./sql ec2-user@<BASTION_PUBLIC_IP>:/home/ec2-user/
```

전송 후 SSH 접속:

```bash
ssh -i ~/.ssh/rds-bastion ec2-user@<BASTION_PUBLIC_IP>
```

---

## 2) 접속 정보 수정

RDS 접속에 필요한 값:

- `RDS_ENDPOINT` (예: `xxxx.ap-northeast-1.rds.amazonaws.com`)
- `PORT` (MySQL 기본 `3306`)
- `USER` (예: `admin`)
- `PASSWORD`
- `DB_NAME` (예: `tastylog`)

SQL 파일 자체를 수정해야 한다면 `vim` 등으로 편집:

```bash
vim init.sql
```

실무에서는 비밀번호를 파일에 하드코딩하지 않고, 실행 시 입력(`-p`) 또는 환경변수/시크릿 매니저를 사용한다.

---

## 3) RDS 데이터 투입

### 방법 A: 파일 실행(권장)

```bash
mysql -h <RDS_ENDPOINT> -P 3306 -u admin -p tastylog < init.sql
```

### 방법 B: 접속 후 source 실행

```bash
mysql -h <RDS_ENDPOINT> -P 3306 -u admin -p
```

```sql
USE tastylog;
SOURCE /home/ec2-user/init.sql;
```

---

## 4) 데이터 확인

접속 후 기본 검증 쿼리:

```sql
SHOW TABLES;
SELECT COUNT(*) FROM <table_name>;
SELECT * FROM <table_name> LIMIT 10;
```

에러가 나면 아래를 우선 확인:

- RDS SG 인바운드 `3306`에 발판 SG가 허용되어 있는지
- DB 이름/계정/비밀번호가 정확한지
- SQL 파일의 문자셋(UTF-8)과 엔진 버전 호환성

---

## 5) 후처리 (중요)

실습이 끝나면 비용/보안 관점에서 임시 자원을 정리한다.

1. 발판 서버(EC2) 종료 또는 삭제
2. 임시 SSH 키 페어 삭제
   - 로컬 키 파일: `~/.ssh/rds-bastion`, `~/.ssh/rds-bastion.pub`
   - AWS에 등록한 키 페어도 삭제
3. 필요 없는 보안 그룹 규칙 제거

---

## 정리

- 프라이빗 RDS 데이터 투입은 보통 `로컬 -> 발판 서버 -> RDS` 경로로 수행한다.
- 먼저 소량 데이터로 검증한 뒤, 본 데이터를 넣는 순서가 안전하다.
- 실습 후 임시 리소스를 반드시 정리해 보안 위험과 과금 위험을 줄인다.
