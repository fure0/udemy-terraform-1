# 46. RDS 작성 (`aws_db_instance`)

Terraform에서 RDS 인스턴스는 `aws_db_instance` 리소스로 정의한다. 슬라이드에서 다룬 인자를 카테고리별로 정리한다.

---

## 기본 설정

| 인자 | 타입 | 설명 |
|------|------|------|
| `engine` | string | DB 엔진 (예: `mysql`, `postgres`) |
| `engine_version` | string | 엔진 버전 (선택) |
| `identifier` | string | RDS 인스턴스 식별자(리소스 이름) |
| `instance_class` | string | 인스턴스 클래스 (예: `db.t3.micro`) |
| `username` | string | 마스터 사용자 이름 |
| `password` | string | 마스터 비밀번호 |
| `tags` | map(string) | 태그 (키–값) |

---

## 스토리지

| 인자 | 타입 | 설명 |
|------|------|------|
| `allocated_storage` | number | 할당 스토리지(GB) |
| `max_allocated_storage` | number | 스토리지 오토스케일 상한(GB). 설정 시 오토스케일 활성 |
| `storage_type` | string | `standard`(마그네틱), `gp2`, `gp3`, `io1` 등 |
| `storage_encrypted` | bool | `true`면 암호화. KMS 키는 `kms_key_id`로 지정(선택) |

슬라이드에는 `storage_encrypted`에 KMS ID 또는 false라고 되어 있으나, Terraform에서는 보통 `storage_encrypted = true/false`이고 별도 키는 `kms_key_id`를 쓴다.

---

## DB 설정

| 인자 | 타입 | 설명 |
|------|------|------|
| `db_name` | string | 인스턴스에 생성할 초기 데이터베이스 이름 |
| `parameter_group_name` | string | DB 파라미터 그룹 |
| `option_group_name` | string | DB 옵션 그룹 |

슬라이드 표기는 `name`이지만 Terraform AWS Provider에서는 초기 DB 이름이 **`db_name`**이다.

---

## 백업·유지보수

| 인자 | 타입 | 설명 |
|------|------|------|
| `backup_window` | string | 자동 백업 수행 시간대(UTC, `hh:mm-hh:mm` 형식) |
| `backup_retention_period` | number | 백업 보관 일수(0이면 자동 백업 비활성에 가깝게 동작) |
| `maintenance_window` | string | 유지보수 시간대 |
| `auto_minor_version_upgrade` | bool | 마이너 버전 자동 업그레이드 여부 |

슬라이드에서 `backup_retention_period`가 bool로 나올 수 있으나, Provider에서는 **숫자(일)**이다.

---

## 삭제·변경 반영

| 인자 | 타입 | 설명 |
|------|------|------|
| `deletion_protection` | bool | `true`면 콘솔/API로 삭제 방지(Terraform destroy는 별도 제약 참고) |
| `skip_final_snapshot` | bool | 삭제 시 최종 스냅샷 생략 여부 |
| `apply_immediately` | bool | `true`면 유지보수 창을 기다리지 않고 변경 즉시 적용 시도 |

삭제 시 `skip_final_snapshot = false`이면 `final_snapshot_identifier`가 필요하다.

---

## 네트워크·가용성(강의에서 자주 같이 씀)

- `db_subnet_group_name`: 서브넷 그룹
- `vpc_security_group_ids`: 보안 그룹
- `publicly_accessible`: 퍼블릭 접근 여부
- `multi_az`: Multi-AZ 배포 여부

이전 레슨에서 만든 `aws_db_parameter_group`, `aws_db_option_group`, `aws_db_subnet_group`의 `name`을 위 인자들에 연결하면 된다.
