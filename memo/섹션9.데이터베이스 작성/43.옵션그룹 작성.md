# 43. 옵션그룹 작성

## 1) RDS 데이터 구조에서 옵션 그룹 위치

RDS 인스턴스는 아래 3가지를 조합해서 동작 설정을 구성한다.

- 파라미터 그룹(Parameter Group): DB 엔진의 세부 파라미터 값(예: 문자셋, 로그 설정 등)
- 옵션 그룹(Option Group): 엔진에 추가/확장되는 옵션(예: 특정 플러그인, 확장 기능 등)
- 서브넷 그룹(Subnet Group): DB가 배치될 서브넷(네트워크 범위)

이번 단계는 그중 **옵션 그룹**을 생성한다.

## 2) `aws_db_option_group` 개념

Terraform 리소스: `aws_db_option_group`

주요 속성:

- `name`: 옵션 그룹 이름
- `engine_name`: 연결할 엔진 이름 (예: `mysql`, `postgres`)
- `major_engine_version`: 엔진의 메이저 버전 (예: `8.0`)
- `option` 블록: 실제로 추가할 옵션(옵션 이름/설정값). 필요할 때만 사용

## 3) 현재 코드 작성 내용

`terraform/rds.tf`

```hcl
resource "aws_db_option_group" "mysql_standalone_optiongroup" {
  name                 = "${var.project}-${var.environment}-mysql-standalone-optiongroup"
  engine_name          = "mysql"
  major_engine_version = "8.0"
}
```

현재는 `option { ... }` 블록을 넣지 않았고, 엔진/버전과 이름만 정의되어 있다.

## 4) 설정 의도(왜 만들어도 옵션 블록이 없을까?)

- 옵션 그룹은 “엔진/버전 조합”에 대해 RDS가 참조할 수 있도록 기본 골격을 제공한다.
- 강의 흐름에서 다음 단계로 특정 기능(예: audit/extension/plugin 등)을 추가할 때
  이 옵션 그룹에 `option` 블록을 더하는 방식으로 확장하기 좋다.

## 5) 체크 포인트

- `engine_name`과 `major_engine_version`은 RDS 인스턴스 설정과 반드시 일치해야 한다.
- 옵션 블록이 없다면 기능 자체는 추가되지 않지만, 이후 확장/구성 연결을 위해 미리 만들어두는 패턴이다.
