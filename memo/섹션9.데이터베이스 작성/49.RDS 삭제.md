# 49. RDS 삭제

RDS는 데이터 유실 위험이 크기 때문에, `aws_db_instance`에는 삭제 관련 보호 옵션이 있다.  
강의 흐름은 **평소에는 보호를 켜두고**, 정말 삭제할 때만 옵션을 잠시 바꿔서 삭제하는 방식이다.

---

## 삭제 관련 주요 인자 (`aws_db_instance`)

| 항목 | 타입 | 설명 |
|------|------|------|
| `deletion_protection` | bool | 인스턴스 삭제를 막을지 여부 |
| `skip_final_snapshot` | bool | 삭제 시 최종 스냅샷 생성 생략 여부 |
| `apply_immediately` | bool | 변경사항 즉시 반영 여부 |

---

## 오조작 방지 권장값 (평소 운영 시)

| 항목 | 값 | 이유 |
|------|----|------|
| `deletion_protection` | `true` | 실수로 DB를 삭제하지 않도록 보호 |
| `skip_final_snapshot` | `false` | 삭제 시 최종 스냅샷을 남겨 복구 가능성 확보 |
| `apply_immediately` | `false` | 즉시 반영으로 인한 예기치 않은 영향 최소화 |

---

## 실제 삭제 직전 임시값

삭제하려면 한 번 보호 옵션을 바꿔 적용해야 한다.

| 항목 | 값 | 목적 |
|------|----|------|
| `deletion_protection` | `false` | 삭제 가능 상태로 전환 |
| `skip_final_snapshot` | `true` | 실습/빠른 삭제 시 스냅샷 생성 생략 |
| `apply_immediately` | `true` | 위 변경을 즉시 반영 |

> 포인트: **삭제 방지 속성은 먼저 수정해서 apply**해야 하며, 곧바로 삭제가 되지 않을 수 있다.

---

## 삭제 절차 (강의 순서)

1. `deletion_protection`, `skip_final_snapshot` 값을 삭제용 설정으로 변경  
2. `terraform apply`로 먼저 반영  
3. RDS 리소스 삭제(예: 코드에서 제거 후 `terraform apply`, 또는 `terraform destroy -target=...`)  
4. 필요 시 동일 설정으로 RDS 재생성

---

## 현재 실습 코드 기준 예시

`terraform/rds.tf`에서는 현재 다음과 같이 되어 있어 **보호 모드**에 가깝다.

```hcl
deletion_protection = true
skip_final_snapshot = false
apply_immediately   = true
```

삭제가 필요하면 일시적으로 아래처럼 바꾼 뒤 apply하고 삭제를 진행한다.

```hcl
deletion_protection = false
skip_final_snapshot = true
apply_immediately   = true
```

---

## 정리

- 기본은 `deletion_protection = true`로 운영한다.
- 삭제 시에는 보호 옵션을 **한 번 먼저** 수정/적용한 뒤 삭제한다.
- 운영 환경에서는 최종 스냅샷(`skip_final_snapshot = false`)을 남기는 전략이 일반적으로 안전하다.
