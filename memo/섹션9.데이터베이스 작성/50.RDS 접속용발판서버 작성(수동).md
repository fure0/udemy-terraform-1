# 50. RDS 접속용 발판서버 작성(수동)

RDS가 프라이빗 서브넷에 있으면 로컬 PC에서 바로 접속하기 어렵다.  
그래서 퍼블릭 서브넷에 **임시 발판 서버(Bastion Host)**를 만들고, 그 서버에서 RDS로 접속을 확인한다.

---

## 작업 순서

1. 일회성 SSH 키 페어 생성
2. 발판 서버(EC2) 수동 생성
3. 발판 서버에 MySQL 클라이언트 설치
4. 발판 서버에서 RDS 접속 테스트

---

## 1) 일회성 키 생성

로컬에서 실습용 키를 만들고 권한을 제한한다.

```bash
ssh-keygen -t rsa -b 4096 -f ~/.ssh/rds-bastion -N ""
chmod 400 ~/.ssh/rds-bastion
```

- 개인키: `~/.ssh/rds-bastion`
- 공개키: `~/.ssh/rds-bastion.pub`

> 실습 종료 후에는 키 파일을 삭제하는 것이 안전하다.

---

## 2) 발판 서버 수동 생성 (EC2 콘솔)

대표 설정:

- AMI: Amazon Linux 2 또는 Ubuntu
- 배치: 퍼블릭 서브넷
- 퍼블릭 IP: 활성화
- 키 페어: 위에서 만든 키(또는 콘솔에서 생성한 키)
- 보안 그룹 인바운드:
  - `22/tcp` : 내 공인 IP만 허용
- 보안 그룹 아웃바운드:
  - 기본 허용(또는 RDS 포트 3306으로 제한)

추가로 **RDS 보안 그룹 인바운드**에 발판 서버 SG를 소스로 허용해야 한다.

- `3306/tcp` from `bastion_sg`

---

## 3) 발판 서버에 MySQL 클라이언트 설치

EC2 접속:

```bash
ssh -i ~/.ssh/rds-bastion ec2-user@<BASTION_PUBLIC_IP>
```

Amazon Linux 2 예시:

```bash
sudo yum update -y
sudo yum install -y mysql
```

Ubuntu 예시:

```bash
sudo apt update
sudo apt install -y mysql-client
```

---

## 4) 발판 서버에서 RDS 접속 확인

RDS 엔드포인트로 접속 테스트:

```bash
mysql -h <RDS_ENDPOINT> -P 3306 -u admin -p
```

정상 연결되면 비밀번호 입력 후 MySQL 프롬프트가 뜬다.

확인 명령:

```sql
SELECT VERSION();
SHOW DATABASES;
```

---

## 트러블슈팅 체크포인트

- 발판 서버에 퍼블릭 IP가 있는지
- 발판 SG `22` 인바운드가 내 IP로 열려 있는지
- RDS SG `3306` 인바운드 소스가 발판 SG인지
- RDS 엔드포인트/포트/계정 정보가 맞는지
- RDS가 `available` 상태인지

---

## 정리

- 프라이빗 RDS 접속은 보통 **발판 서버 경유**로 확인한다.
- 핵심은 SG 경로(`내 IP -> 발판(22) -> RDS(3306)`)를 정확히 여는 것이다.
- 이 단계가 끝나면 다음 실습(데이터 투입)으로 이어진다.
