# 45. 랜덤 문자열 생성

## 1) 왜 `random_string`을 쓰는가

RDS 마스터 비밀번호처럼 **매 배포마다 달라져야 하는 값**을 코드에 평문으로 넣지 않으려면, Terraform에서 난수를 만들어 두고 `aws_db_instance` 등에 넘기는 방식이 일반적이다.  
이때 사용하는 리소스가 **`random_string`**(HashiCorp **random** 프로바이더)이다.

생성된 문자열은 `random_string.<이름>.result`로 참조한다.

## 2) `random_string` 주요 인자 (강의 자료 기준)

| 항목 | 타입 | 설명 |
| :--- | :--- | :--- |
| `length` * | number | 난수 문자열 길이(필수) |
| `upper` | bool | 대문자 사용 여부(기본: `true`) |
| `lower` | bool | 소문자 사용 여부(기본: `true`) |
| `number` | bool | 숫자 사용 여부(기본: `true`) |
| `special` | bool | 특수문자 사용 여부(기본: `true`) |
| `override_special` | string | 사용할 특수문자 집합을 직접 지정. 미지정 시 기본 집합: `!@#$%&*()-_=+[]{}<>:?` |

`*` 표시는 필수 인자.

## 3) RDS 비밀번호에 맞춘 설정 팁

MySQL RDS 비밀번호는 **일부 특수문자가 제한**되는 경우가 있어, 강의에서는 `special = false`로 두고 영문·숫자만 쓰는 패턴이 자주 나온다.

## 4) 현재 코드 작성 내용

`terraform/rds.tf`

```hcl
resource "random_string" "db_password" {
  length  = 16
  special = false
}
```

## 5) 프로바이더 선언

`random_string`을 쓰려면 Terraform에 **random** 프로바이더가 필요하다. `terraform` 블록의 `required_providers`에 예를 들어 다음을 추가한다.

```hcl
random = {
  source  = "hashicorp/random"
  version = "~> 3.0"
}
```

추가 후 `terraform init`으로 프로바이더를 받는다.
