# 65. IAM Role 작성

64번에서 만든 **신뢰 정책**(`data.aws_iam_policy_document`)을 붙여 **`aws_iam_role`** 리소스로 IAM Role 본체를 만든다.  
다이어그램(63번)에서 **IAM Role**이 중심이고, 아래쪽 **신뢰 정책(data)**을 참조하는 구조다.

---

## `aws_iam_role` 주요 인자

| 항목 | 타입 | 설명 |
|------|------|------|
| `name` | string | IAM Role **이름** |
| `assume_role_policy` | string | **신뢰 정책 JSON**(필수). 보통 `data.aws_iam_policy_document.xxx.json` |
| `description` | string | Role 설명(선택) |
| `tags` | map(string) | 태그(선택) |

`assume_role_policy`에 `*`가 붙어 있는 것은 **반드시 지정**해야 한다는 뜻이다. EC2가 맡을 수 있게 하려면 64번의 Trust Policy 내용이 여기 들어간다.

---

## HCL 예시

```hcl
resource "aws_iam_role" "app" {
  name               = "${var.project}-${var.environment}-app-role"
  assume_role_policy = data.aws_iam_policy_document.ec2_assume_role.json

  description = "IAM role for app EC2"

  tags = {
    Name    = "${var.project}-${var.environment}-app-role"
    Project = var.project
    Env     = var.environment
  }
}
```

이 저장소 `iam.tf`에는 이미 **`data.aws_iam_policy_document.ec2_assume_role`**만 있으므로, 위 `aws_iam_role` 블록을 같은 파일에 추가하면 된다.

---

## 63번 다이어그램과의 위치

```
인스턴스 프로필 ──→ IAM Role  ←── (여기서 작성)
                        │
                        └──→ 신뢰 정책(data, 64번)
```

- **권한 정책(S3 등)**은 아직 Role에 붙지 않는다 → **66. 정책 연결**에서 `aws_iam_role_policy_attachment`로 연결한다.
- EC2에 붙이려면 **67. 인스턴스 프로필**, **68. EC2** 단계가 이어진다.

---

## 실습 팁

- 강의에서는 **인스턴스 프로필 이름과 Role 이름을 같게** 두는 패턴을 쓰기도 한다(63번 메모).
- `assume_role_policy`는 **문자열 JSON**이므로, 직접 JSON을 넣지 말고 **`data.aws_iam_policy_document`의 `.json`**을 쓰면 HCL로 관리하기 쉽다.

---

## 정리

- **`aws_iam_role`** = 역할 리소스 + **`assume_role_policy`**에 신뢰 정책 연결.
- 다음은 **권한 Policy를 Role에 attach**(66번) → **인스턴스 프로필**(67번) → **EC2의 `iam_instance_profile`**(68번).
