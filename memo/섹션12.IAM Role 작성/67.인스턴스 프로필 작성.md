# 67. 인스턴스 프로필 작성

EC2는 IAM Role을 **직접** 붙이지 않고, **인스턴스 프로필(Instance Profile)**을 통해 Role을 전달한다.  
Terraform에서는 **`aws_iam_instance_profile`** 리소스로 만들고, 그 안에 **어떤 IAM Role을 넣을지** 지정한다(63번 다이어그램 최상단).

---

## 왜 인스턴스 프로필이 필요한가

- **IAM Role** = 권한 묶음(신뢰 정책 + 권한 정책).
- **인스턴스 프로필** = EC2가 부팅 시 **맡을 Role을 담는 용기**.
- EC2의 `iam_instance_profile` 인자에는 **프로필 이름**을 넣는다(68번).

---

## `aws_iam_instance_profile` 주요 인자

| 항목 | 타입 | 설명 |
|------|------|------|
| `name` | string | **인스턴스 프로필 이름** |
| `role` | string | 연결할 **IAM Role 이름** (`aws_iam_role.xxx.name`) |

### 슬라이드 팁

| 항목 | 팁 |
|------|-----|
| **`name`** | **IAM Role 이름과 같게** 맞추면 관리가 쉽다(강의 권장). |
| **`role`** | 문서상은 **옵션**이지만, EC2에 쓰려면 **반드시 지정**한다. Role 없는 프로필은 실무에서 거의 쓰지 않는다. |

---

## HCL 예시

이 저장소의 Role 이름은 **`${var.project}-${var.environment}-app-iam-role`** 이다. 프로필도 같은 이름으로 맞추는 예:

```hcl
resource "aws_iam_instance_profile" "app" {
  name = aws_iam_role.app_iam_role.name
  role = aws_iam_role.app_iam_role.name
}
```

또는 `name`을 Role과 동일한 문자열로 직접 지정:

```hcl
resource "aws_iam_instance_profile" "app" {
  name = "${var.project}-${var.environment}-app-iam-role"
  role = aws_iam_role.app_iam_role.name
}
```

`iam.tf`에는 아직 인스턴스 프로필 블록이 없으므로, 66번 정책 연결 다음에 위 리소스를 추가하면 된다.

## 63번 다이어그램에서의 위치

```
인스턴스 프로필  ──→  IAM Role
                         ├── 신뢰 정책(64)
                         └── 정책 연결(66) → Policy
```

- **67번**: 프로필 생성 + Role 연결  
- **68번**: `aws_instance`의 **`iam_instance_profile`**에 이 프로필 이름 지정

---

## 정리

- **`aws_iam_instance_profile`**: `name` + `role`로 EC2용 **Role 운반체**를 만든다.
- 강의에서는 **`name` = IAM Role 이름** 패턴을 권장한다.
- **`role`은 옵션이지만 실습·운영에서는 필수**로 본다.
