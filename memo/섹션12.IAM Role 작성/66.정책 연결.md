# 66. 정책 연결

65번에서 만든 **IAM Role**에 **권한 정책(Policy)**을 붙인다.  
신뢰 정책(64번)이 **“누가 역할을 맡을 수 있는지”**라면, 여기서 연결하는 Policy는 **“맡은 뒤 무엇을 할 수 있는지”**다.

Terraform에서는 **`aws_iam_role_policy_attachment`**가 **Role과 Policy를 연결**하는 리소스다(63번 다이어그램의 **ロールポリシーアタッチ**).

---

## `aws_iam_role_policy_attachment` 주요 인자

| 항목 | 타입 | 설명 |
|------|------|------|
| `role` | string | **IAM Role 이름**(필수). `aws_iam_role.xxx.name` |
| `policy_arn` | string | 붙일 **Policy ARN**(필수) |

Policy 하나당 **attachment 리소스를 하나** 두는 패턴이 일반적이다(관리형 정책 4개면 attachment 4개).

---

## 강의에서 Role에 붙이는 AWS 관리형 정책

| 정책 이름 | 용도(슬라이드) |
|-----------|----------------|
| **AmazonSSMManagedInstanceCore** | **Session Manager**로 인스턴스 접속 |
| **AmazonS3ReadOnlyAccess** | **S3**에서 파일 가져오기(읽기) |
| **AmazonEC2ReadOnlyAccess** | **EC2 태그** 정보 조회 |
| **AmazonSSMReadOnlyAccess** | **Parameter Store**에서 환경 변수 등 조회(읽기) |

모두 **AWS가 제공하는 관리형 정책**이므로 ARN은 보통 다음 형식이다.

`arn:aws:iam::aws:policy/<정책이름>`

---

## HCL 예시

`data "aws_iam_policy"`로 ARN을 가져오거나, ARN 문자열을 직접 넣을 수 있다.

```hcl
data "aws_iam_policy" "ssm_core" {
  arn = "arn:aws:iam::aws:policy/AmazonSSMManagedInstanceCore"
}

resource "aws_iam_role_policy_attachment" "ssm_core" {
  role       = aws_iam_role.app_iam_role.name
  policy_arn = data.aws_iam_policy.ssm_core.arn
}

resource "aws_iam_role_policy_attachment" "s3_readonly" {
  role       = aws_iam_role.app_iam_role.name
  policy_arn = "arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess"
}

resource "aws_iam_role_policy_attachment" "ec2_readonly" {
  role       = aws_iam_role.app_iam_role.name
  policy_arn = "arn:aws:iam::aws:policy/AmazonEC2ReadOnlyAccess"
}

resource "aws_iam_role_policy_attachment" "ssm_readonly" {
  role       = aws_iam_role.app_iam_role.name
  policy_arn = "arn:aws:iam::aws:policy/AmazonSSMReadOnlyAccess"
}
```

이 저장소 `iam.tf`의 Role 리소스 이름은 **`aws_iam_role.app_iam_role`**이다. 강의 코드에 맞춰 `role` 인자만 동일하게 맞추면 된다.

---

## 63번 다이어그램에서의 위치

```
IAM Role  ←── aws_iam_role_policy_attachment
    ↑              │
    │              └──→ Policy (관리형 ARN / data)
    └──→ 신뢰 정책(64번)
```

- **인스턴스 프로필**(67번)과 **EC2**(68번)는 이후 단계에서 Role을 EC2에 연결한다.

---

## 정리

- **`aws_iam_role_policy_attachment`**: `role` + `policy_arn`으로 **권한 정책을 Role에 연결**.
- 강의 EC2용 Role에는 **SSM·S3 읽기·EC2 읽기·SSM 읽기** 관리형 정책 4개를 붙인다.
- Trust(64) → Role(65) → **정책 연결(66)** → 인스턴스 프로필(67) 순서다.
