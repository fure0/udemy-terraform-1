# 64. 신뢰 정책 작성

**신뢰 정책(Trust Policy)**은 IAM Role에 붙는 JSON으로, **“누가 이 역할을 맡을(AssumeRole 할) 수 있는가”**만 정한다.  
S3 읽기 같은 **실제 API 권한**은 **별도의 권한 정책(Policy)**에서 정의한다(63번 다이어그램의 Trust vs Policy 구분).

EC2가 역할을 쓰려면, 신뢰 정책에서 **주체(Principal)로 EC2 서비스**를 허용하고 **`sts:AssumeRole`**을 허용하면 된다.

---

## 콘솔에서 보는 JSON 예시 (EC2)

슬라이드에 나온 형태는 다음과 같다.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "",
      "Effect": "Allow",
      "Principal": {
        "Service": "ec2.amazonaws.com"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

| 필드 | 의미 |
|------|------|
| `Effect` | `Allow` — 이 문장은 허용 |
| `Principal.Service` | **`ec2.amazonaws.com`** — EC2 서비스가 이 역할을 맡을 수 있음 |
| `Action` | **`sts:AssumeRole`** — 역할 맡기에 필요한 STS 액션 |

`Version`은 IAM 정책 문서에서 흔히 **`2012-10-17`**을 쓴다.

---

## Terraform `data "aws_iam_policy_document"`

신뢰 정책 JSON을 손으로 붙여 넣지 않고, **`data "aws_iam_policy_document"`** 블록으로 만들어 `aws_iam_role`의 **`assume_role_policy`**에 넘기는 패턴이 강의에서 다루는 방식이다.

### 데이터 소스 필드 요약 (슬라이드 표)

| 항목 | 타입 | 설명 |
|------|------|------|
| `version` | string | 정책 언어 버전 |
| `policy_id` | string | 정책 식별자(선택에 가깝게 사용) |
| `statement` | 블록 | **문장(Statement)** 단위의 내용. 여러 개 둘 수 있음 |

#### `statement` 안

| 항목 | 타입 | 설명 |
|------|------|------|
| `sid` | string | 문장 ID |
| `effect` | `Allow` / `Deny` | 허용 또는 거부 |
| `actions` | list(string) | 액션 목록 |
| `resources` | list(string) | 대상 리소스 ARN 등 |
| `principals` | 블록 | **맡을 수 있는 주체**(Trust에서 중요) |

#### `principals` 안

| 항목 | 타입 | 설명 |
|------|------|------|
| `type` | string | **필수.** `"AWS"`, `"Service"` 등 |
| `identifiers` | list(string) | **필수.** 계정 ARN, **`ec2.amazonaws.com`** 같은 서비스 식별자 등 |

---

## HCL 예시 (EC2 신뢰 정책)

```hcl
data "aws_iam_policy_document" "ec2_assume_role" {
  statement {
    effect = "Allow"

    principals {
      type        = "Service"
      identifiers = ["ec2.amazonaws.com"]
    }

    actions = ["sts:AssumeRole"]
  }
}
```

다음 단계(65. IAM Role 작성)에서 `aws_iam_role`에 다음처럼 연결한다.

```hcl
resource "aws_iam_role" "example" {
  name               = "example-role"
  assume_role_policy = data.aws_iam_policy_document.ec2_assume_role.json
  # ...
}
```

---

## 정리

- 신뢰 정책 = **“EC2(서비스)가 이 역할을 맡아도 된다”**는 선언.
- Terraform에서는 **`data.aws_iam_policy_document`**로 JSON을 생성해 **`assume_role_policy`**에 넣는다.
- **`principals`**의 `type` / `identifiers`가 Trust에서 EC2를 가리키는 핵심이다.
