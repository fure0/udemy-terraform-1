# 68. EC2에 IAM Role 작성

64~67번에서 만든 **IAM Role·정책·인스턴스 프로필**을 **EC2에 연결**하는 단계다.  
슬라이드 제목은 **「EC2へIAMロールを適用」**이지만, AWS/Terraform에서는 **Role을 EC2에 직접 붙이지 않고**, **인스턴스 프로필**을 EC2에 지정한다.

**강의 1단계:** 만들어 둔 IAM Role(프로필 경유)을 EC2에 **関連付け(연결)** 한다.

---

## 연결 구조 (슬라이드)

```
Terraform
    ↓
[ 인스턴스 프로필 ] ──→ EC2
        │
        └── IAM Role
                ├── 신뢰 정책
                └── 정책 연결 → Policy
```

- 화살표는 **인스턴스 프로필 → EC2** 방향이다.
- EC2가 받는 것은 **프로필 이름**이고, 그 안에 들어 있는 **Role**의 권한이 인스턴스에 적용된다.

---

## `aws_instance`에서 지정

| 인자 | 설명 |
|------|------|
| `iam_instance_profile` | 붙일 **인스턴스 프로필 이름**(문자열). `aws_iam_instance_profile.xxx.name` |

Role ARN이 아니라 **프로필의 `name`**을 넣는다(67번에서 Role 이름과 같게 둔 경우, 그 이름과 동일).

### HCL 예시

```hcl
resource "aws_instance" "app_server" {
  ami           = data.aws_ami.app.id
  instance_type = "t2.micro"
  # ... subnet_id, security groups, key_name 등 ...

  iam_instance_profile = aws_iam_instance_profile.app_ec2_profile.name

  tags = {
    Name = "${var.project}-${var.environment}-app-ec2"
    # ...
  }
}
```

이 저장소에서는 `iam.tf`에 **`aws_iam_instance_profile.app_ec2_profile`**이 있고, `appserver.tf`의 **`aws_instance.app_server`**에는 아직 `iam_instance_profile`이 없으므로 위 한 줄을 추가한 뒤 `terraform apply`하면 된다.

---

## apply 시 참고

- **이미 떠 있는 EC2**에 프로필을 **처음** 붙이거나 바꾸면, Terraform/AWS에 따라 **인스턴스 교체(replace)** 플랜이 나올 수 있다. `plan`으로 **destroy/create**인지 확인한다.
- 별도로 **`data.aws_ami.app` + `most_recent`** 때문에 **AMI 변경으로 교체**가 잡히는 경우도 있다(섹션11·EC2 메모 참고). IAM만 넣으려 할 때 EC2 replace가 보이면 **AMI drift**인지 함께 본다.

## 정리

- EC2에 IAM 권한을 주려면 **`iam_instance_profile = 인스턴스 프로필 이름`**.
- 콘솔에서 “IAM Role 연결”이라고 해도, 실제로는 **인스턴스 프로필**이 EC2와 연결된다.
- `apply` 전 **`terraform plan`**으로 EC2 교체 여부를 확인한다.
