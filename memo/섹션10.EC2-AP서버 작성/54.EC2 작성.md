# 54. EC2 작성

Terraform의 **`aws_instance`**로 VPC 안 **퍼블릭 서브넷**에 애플리케이션용 EC2를 띄운다. 다이어그램처럼 **서브넷 + 보안 그룹** 안에 인스턴스가 들어가고, AMI·키페어 등과 연결된다.

---

## EC2를 구성할 때 필요한 조각

| 요소 | 역할 |
|------|------|
| **AMI** | OS·루트 볼륨 템플릿 (`data "aws_ami"` 등으로 조회) |
| **키페어** | SSH 접속용 (`aws_key_pair`, `key_name`) |
| **서브넷** | 인스턴스가 붙는 네트워크 위치 (`subnet_id`) |
| **보안 그룹** | 인·아웃바운드 방화벽 (`vpc_security_group_ids`) |

52·53에서 준비한 AMI 검색·키페어와 여기서의 **네트워크·SG**가 맞물려야 한다.

---

## `aws_instance` 설정 영역(개념)

1. **기본 설정**: `ami`, `instance_type` 등
2. **네트워크**: `subnet_id`, 퍼블릭 IP, `vpc_security_group_ids`
3. **그 외**: `key_name`, `iam_instance_profile`, `user_data`, `tags` 등

---

## 기본 설정

| 항목 | 타입 | 설명 |
|------|------|------|
| `ami` | string | AMI ID(필수). 이 프로젝트는 `data.aws_ami.app.id` 등 |
| `instance_type` | string | 인스턴스 크기(필수). 예: `t2.micro`, `t3.micro` |
| `tags` | map(string) | 태그(선택) |

---

## 네트워크

| 항목 | 타입 | 설명 |
|------|------|------|
| `subnet_id` | string | 배치할 서브넷 ID(퍼블릭이면 공인 경로 설계와 함께) |
| `associate_public_ip_address` | bool | 자동 할당 퍼블릭 IP 여부 |
| `vpc_security_group_ids` | list(string) | 적용할 보안 그룹 ID 목록 |

퍼블릭 서브넷 + `associate_public_ip_address = true` 조합이면 SSH 등으로 인터넷에서 붙기 쉽다(보안 그룹 `22`는 본인 IP 등으로 제한).

---

## 그 외(자주 쓰는 인자)

| 항목 | 타입 | 설명 |
|------|------|------|
| `iam_instance_profile` | string | 인스턴스에 붙이는 IAM 인스턴스 프로파일 이름 |
| `key_name` | string | EC2에 등록된 **키페어 이름**(`aws_key_pair.key_name`과 동일) |
| `user_data` | string | 부팅 시 실행할 스크립트(베이스64 인코딩이 필요한 경우도 있음) |

---

## 예시 스켈레톤

```hcl
resource "aws_instance" "app" {
  ami                         = data.aws_ami.app.id
  instance_type               = "t2.micro"
  subnet_id                   = aws_subnet.public_subnet_1a.id
  vpc_security_group_ids = [
    aws_security_group.app_sg.id,
    aws_security_group.opmng_sg.id,
  ]
  associate_public_ip_address = true
  key_name                    = aws_key_pair.keypair.key_name

  tags = {
    Name = "${var.project}-${var.environment}-app"
  }
}
```

이 저장소에서는 앱 트래픽용 `app_sg`와 운영·SSH용 `opmng_sg`를 같이 붙이는 구성이 자연스럽다. 강의에서는 SG 하나만 쓸 수도 있으니 슬라이드와 맞춘다.

`appserver.tf`에 `aws_key_pair`가 있으므로 `key_name = aws_key_pair.keypair.key_name`으로 연결하면 된다. 서브넷·AMI 데이터 소스 이름은 실제 `.tf` 정의에 맞춘다.

---

## 정리

- EC2 한 대는 **AMI + 타입 + 서브넷 + SG**가 최소 뼈대이고, SSH용이면 **키페어·퍼블릭 IP·SG 22**를 맞춘다.
- 슬라이드의 `aws_instance`는 퍼블릭 서브넷 안 SG에 올린 그림과 동일한 구조로 코드에 옮기면 된다.
