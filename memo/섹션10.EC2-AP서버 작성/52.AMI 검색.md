# 52. AMI 검색

Terraform에서 AMI를 고를 때 쓰는 **`data "aws_ami"`**의 조건 지정은, AWS CLI의 **`aws ec2 describe-images`**와 **같은 필터·소유자 개념**을 쓴다. CLI로 먼저 검색해 보고 Terraform에 옮기면 이해가 빠르다.

---

## AWS CLI로 AMI 검색

```bash
aws ec2 describe-images \
  [--image-ids <AMI_ID>] \
  [--owners <OWNER>] \
  [--filters "Name=<KEY1>,Values=<VALUE1>,<VALUE2>,..."]
```

| 인자 | 설명 |
|------|------|
| `--image-ids` | 특정 AMI ID를 지정 |
| `--owners` | AMI 소유자. 예: 계정 ID, `self`(내 계정), `amazon`(공식 Amazon 이미지), `aws-marketplace`(마켓플레이스) |
| `--filters` | `"Name=키,Values=값1,값2,..."` 형식으로 검색 조건 지정 |

### filters에서 자주 쓰는 키

| 키 | 설명 |
|----|------|
| `name` | 이미지 이름 |
| `description` | 이미지 설명 |
| `root-device-type` | 루트 볼륨 유형. 보통 `ebs` |
| `virtualization-type` | 가상화 방식. 보통 `hvm` |

그 외 조건은 AWS CLI 레퍼런스를 본다.

- [describe-images (AWS CLI Command Reference)](https://docs.aws.amazon.com/cli/latest/reference/ec2/describe-images.html)

---

## Terraform `data "aws_ami"` 주요 인자

| 항목 | 타입 | 설명 |
|------|------|------|
| `owners` | `list(string)` | **필수에 가깝게 사용**. `self`, `amazon`, `aws-marketplace`, `microsoft` 또는 계정 ID 등 |
| `most_recent` | `bool` | 여러 개가 매칭될 때 **가장 최근** AMI 하나만 쓸지 |
| `executable_users` | `list(string)` | 지정한 주체가 **실행(launch) 권한**을 가진 AMI로 제한. `self` 또는 계정 ID |
| `filter` | 블록 | `name`(필터 키)와 `values`(값 목록)로 검색 조건 추가. CLI `--filters`와 대응 |

```hcl
data "aws_ami" "amazon_linux_2" {
  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*-x86_64-gp2"]
  }

  filter {
    name   = "virtualization-type"
    values = ["hvm"]
  }

  filter {
    name   = "root-device-type"
    values = ["ebs"]
  }
}
```

이후 EC2 등에서 `ami = data.aws_ami.amazon_linux_2.id`처럼 참조하면 된다.

---

## 정리

- Terraform AMI 검색 ≈ **`describe-images` + filters/owners**.
- 운영체제·세대별로 `name` 패턴과 `owners`를 맞추고, `most_recent = true`로 최신만 고르는 패턴이 흔하다.
- 필터 키 전체는 위 CLI 문서에서 확인한다.
