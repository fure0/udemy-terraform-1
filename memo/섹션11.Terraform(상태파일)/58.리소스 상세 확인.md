# 58. 리소스 상세 확인

코드(`main.tf` 등)에는 **의도한 설정**이 있고, 실제로 만들어진 리소스의 **속성·식별자**는 상태 **`tfstate`**에도 기록된다(슬라이드: 생성한 리소스 정보도 tfstate에 있다).  
특정 한 리소스가 상태에 어떻게 저장되어 있는지 보려면 **`terraform state show`**를 쓴다.

---

## 명령

```bash
terraform state show <ADDRESS>
```

| 인자 | 설명 |
|------|------|
| `<ADDRESS>` | **필수.** 상세를 보고 싶은 **리소스 주소**(예: `terraform state list`로 확인한 한 줄) |

출력에는 Terraform이 상태에 보관한 **인스턴스 속성 전체**(민감 값은 마스킹될 수 있음)가 표시된다.

---

## 예시 (EC2)

```bash
terraform state show aws_instance.<NAME>
```

`<NAME>`은 코드에 적은 `resource "aws_instance" "<NAME>"`의 `<NAME>`과 같다.

---

## 흐름

1. `terraform state list`로 주소 확인  
2. `terraform state show <ADDRESS>`로 해당 리소스만 상세 출력  

구성 루트(`terraform/` 등)에서 실행한다.

---

## 정리

- `terraform state show`는 **상태에 올라간 한 리소스의 스냅샷**을 본다.
- 콘솔에서 보는 값과 비교하거나, drift·트러블슈팅할 때 자주 쓴다.
