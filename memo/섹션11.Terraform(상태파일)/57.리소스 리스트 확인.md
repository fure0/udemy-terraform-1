# 57. 리소스 리스트 확인

`main.tf` 등으로 정의한 리소스가 실제로 어떻게 배치되어 있는지, Terraform이 기억하는 **현재 상태**는 **`tfstate`**에 담긴다(슬라이드: 클라우드 위의 상태 정보는 tfstate에 기술된다).  
**어떤 주소들이 상태에 올라와 있는지** 보려면 `terraform state list`를 쓴다.

---

## 명령

```bash
terraform state list [<ADDRESS>]
```

| 인자 | 설명 |
|------|------|
| `<ADDRESS>` | (선택) **필터링할 리소스 주소**. 생략하면 상태에 있는 **전체 리소스 주소**를 한 줄에 하나씩 출력한다. |

`<ADDRESS>`는 보통 **접두사**로도 동작한다. 예를 들어 `aws_instance`로 시작하는 것만 보고 싶으면:

```bash
terraform state list aws_instance
```

특정 리소스 한 개만:

```bash
terraform state list aws_instance.app
```

---

## 실행 위치

Terraform 구성 루트(`.tf`가 있는 디렉터리, 이 프로젝트는 `terraform/`)에서 실행한다.

```bash
cd /Users/kimtaeyoun/Desktop/Dev/udemy_terraform/terraform
terraform state list
```

원격 백엔드(S3)를 쓰는 경우에도 **같은 명령**으로, 실제로는 백엔드에 있는 상태를 기준으로 목록이 나온다.

---

## 정리

- `terraform state list`는 **상태 파일에 등록된 리소스 주소 목록**을 본다.
- 선택 인자로 **이름을 좁혀** 출력할 수 있다.
- 다음 단계(상세 확인)는 보통 `terraform state show <ADDRESS>`로 이어진다.
