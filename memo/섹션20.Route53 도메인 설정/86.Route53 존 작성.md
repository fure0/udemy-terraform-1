# 86. Route53 존 작성

**85번**에서 취득한 도메인을 AWS **Route 53 호스트 영(Zone)** 으로 관리하기 시작한다.  
이번 강의는 **`aws_route53_zone`** 만 작성한다. **A 레코드(87)** · **등록업체 NS 설정(88)** 은 다음 단계다.

이전: **85 도메인 취득** · 다음: **87 Route53 레코드**

---

## Route53 설정 흐름 (슬라이드)

| 순서 | 내용 | 강의 |
|------|------|------|
| 1 | **호스트 영(Zone) 생성** | **86 (지금)** |
| 2 | **A 레코드 생성** (ALB 등으로 연결) | 87 |
| 3 | **도메인 등록업체**에 NS(네임서버) 설정 | 88 (수동) |

Zone만 만들고 **등록업체 NS를 바꾸지 않으면** Route 53 레코드가 **인터넷에 반영되지 않는다**.

---

## Route53 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (73번·82번과 동일).

```
Route53 레코드 (aws_route53_record)  ← 87번
        ↓ zone_id 참조
Route53 존 (aws_route53_zone)        ← 86번
```

| 리소스 | 참조 대상 | 코드(예) |
|--------|-----------|----------|
| **레코드** | **존** | `zone_id = aws_route53_zone.main.zone_id` |

**86번은 존만** 만든다. 레코드는 존 **ARN/zone_id** 가 필요하므로 **존 → 레코드** 순서다.

---

## `aws_route53_zone` 주요 인자 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** * | string | Route 53 **존 이름** = **도메인 FQDN** (끝에 `.` 없이 `example.com`) |
| **`force_destroy`** | bool | 존 **삭제 시** 레코드도 함께 지울지 |
| **`tags`** | map | 태그 |

`*` = 필수.

### `force_destroy` (슬라이드 보충)

슬라이드: 「Terraform管理外のリソースがあるとき強制削除するか」

| 값 | 동작 |
|----|------|
| **`false`** (기본) | 존 안에 **Terraform이 만든 것 외 레코드**가 있으면 **`destroy` 실패** |
| **`true`** | 존 삭제 시 **레코드를 포함해 강제 삭제** (실습·dev에서 편함, **운영은 주의**) |

강의·개발 환경에서는 **`force_destroy = true`** 를 쓰는 경우가 많다.

---

## 존 생성 후 AWS가 주는 것

`apply` 후 **이 존 전용 네임 서버(NS) 4개**가 할당된다.

- 콘솔: Route 53 → 해당 호스트 영 → **네임 서버**
- Terraform: `aws_route53_zone.xxx.name_servers` (list)

**88번**에서 **お名前.com(85)** 등 등록업체의 NS를 이 값으로 **수동 변경**한다.  
86번만으로는 **도메인이 Route 53을 아직 안 바라본다**.

---

## HCL 예시

`route53.tf` 등 새 파일:

```hcl
resource "aws_route53_zone" "main" {
  name          = "example.com"   # 85에서 취득한 도메인
  force_destroy = true

  tags = {
    Name    = "${var.project}-${var.environment}-zone"
    Project = var.project
    Env     = var.environment
  }
}
```

도메인을 변수로 두는 예:

```hcl
variable "domain_name" {
  type = string
}

resource "aws_route53_zone" "main" {
  name          = var.domain_name
  force_destroy = true

  tags = {
    Name    = "${var.project}-${var.environment}-zone"
    Project = var.project
    Env     = var.environment
  }
}
```

`terraform.tfvars` 예: `domain_name = "your-domain.example"`

---

## 이 프로젝트와의 관계

- **`terraform/`** 에는 **아직 Route53 코드 없음** — 86 실습에서 추가.
- **ALB** (`aws_lb.alb`, 82~84)는 이미 있다. **87**에서 `aws_route53_record`로 **ALB DNS를 Alias A** 로 연결하는 흐름이 이어진다.
- Route 53은 **글로벌** 서비스. provider `region`은 `ap-northeast-1` 그대로 두어도 된다.

## apply 전·후 확인

```bash
cd terraform
terraform plan
terraform apply
```

| 단계 | 확인 |
|------|------|
| **plan** | `aws_route53_zone.main` **add** |
| **apply 후** | 콘솔에 **호스트 영** 생성, **NS 4개** 표시 |
| **output (선택)** | `name_servers` 를 output에 넣어 88번 수동 작업에 사용 |

```hcl
output "route53_name_servers" {
  value = aws_route53_zone.main.name_servers
}
```

## 85·87·88과의 역할

| 강의 | 내용 |
|------|------|
| **85** | 도메인 **취득** (お名前.com 등) |
| **86 (지금)** | Route 53 **호스트 영** — DNS 관리의 뿌리 |
| **87** | **A / Alias** 레코드 → ALB 등 |
| **88** | 등록업체에 **NS 등록** (Terraform 밖, 수동) |

---

## 정리

- **`aws_route53_zone`**: `name` = 도메인, `force_destroy` = destroy 시 레코드 처리.
- **존 → 레코드** 참조 구조; 86은 **존만**.
- Route 53 설정 3단계 중 **1단계**; NS는 **88**에서 등록업체에 반영.
- 다음: **87 Route53 레코드** (ALB Alias A 등).
