# 87. Route53 레코드 작성

**86번** 호스트 영 안에 **DNS 레코드**를 Terraform **`aws_route53_record`** 로 만든다.  
이 강의·프로젝트의 목표는 **도메인 → ALB** 연결(**A 레코드 + Alias**)이다. 슬라이드에는 **IP 지정 A**, **AWS 리소스 Alias A**, **CNAME** 패턴도 함께 다룬다.

이전: **86 Route53 존** · 다음: **88 등록업체 NS 설정(수동)**

---

## Terraform과 Route53 (슬라이드)

Terraform이 **Route 53** 안에 레코드를 정의한다.

- **A 레코드** — 도메인 → IPv4 (또는 **Alias**로 AWS 리소스)
- **CNAME 레코드** — 도메인 → 다른 **도메인 이름**

이 프로젝트 실습의 **본편**은 **ALB용 Alias A** (82~84 `aws_lb.alb`).

---

## 데이터 구조 (슬라이드)

```
Route53 레코드 (aws_route53_record)  ← 87번
        ↓ zone_id
Route53 존 (aws_route53_zone.main)   ← 86번
```

| 리소스 | 참조 | 코드 |
|--------|------|------|
| **레코드** | **존** | `zone_id = aws_route53_zone.main.zone_id` |

슬라이드 패턴(강의에서 유형별 확인):

1. **A 레코드 (IP 지정)**
2. **A 레코드 (AWS 리소스 지정 = Alias)**
3. **CNAME 레코드**

---

## `aws_route53_record` 공통 인자 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`zone_id`** * | string | 레코드를 넣을 **호스트 영 ID** |
| **`name`** * | string | **레코드 이름** (`www`, apex는 존 이름/`var.domain`) |
| **`type`** * | enum | `"A"`, `"CNAME"`, `"AAAA"` 등 |
| **`ttl`** (*) | number | 캐시 유효 시간(초) |
| **`records`** (*) | string[] | **라우팅 대상** (IP 등) |
| **`allow_overwrite`** | bool | 동일 이름 레코드 **덮어쓰기** 허용 |

(*) **`alias` 블록을 쓰지 않을 때** `ttl`·`records` **필수**.  
Alias A/CNAME to ALB 등 AWS 리소스일 때는 **`alias`만** 쓰고 `ttl`/`records` **생략**.

### `alias` 블록 (AWS 리소스 대상)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** | string | 대상 리소스 **DNS 이름** |
| **`zone_id`** | string | 대상 리소스 **호스트 영 ID** |
| **`evaluate_target_health`** | bool | 대상 **헬스** 반영 여부 |

---

## 패턴 1 — A 레코드 (IP 지정)

고정 **IPv4**에 직접 연결할 때 (EC2 Elastic IP 등).

```hcl
resource "aws_route53_record" "example_a" {
  zone_id = aws_route53_zone.main.zone_id
  name    = "app.${var.domain}"
  type    = "A"
  ttl     = 300
  records = ["203.0.113.10"]
}
```

| 필수 | `zone_id`, `name`, `type`, `ttl`, `records` |

---

## 패턴 2 — A 레코드 (AWS 리소스 = Alias) ★ 이 프로젝트

**ALB** 앞에 도메인을 붙일 때. **CNAME이 아니라 Alias A**가 일반적(apex 도메인도 가능).

### ALB일 때 alias 속성 (슬라이드)

| | ALB | CloudFront (참고) |
|---|-----|-------------------|
| **name** | `<리소스>.dns_name` | `<리소스>.domain_name` |
| **zone_id** | `<리소스>.zone_id` | `<리소스>.hosted_zone_id` |

### HCL 예시 — apex → ALB

현재 코드: **`aws_route53_zone.main`**, **`aws_lb.alb`**, **`var.domain`** (`terraform.tfvars`).

```hcl
resource "aws_route53_record" "alb" {
  zone_id = aws_route53_zone.main.zone_id
  name    = var.domain
  type    = "A"

  alias {
    name                   = aws_lb.alb.dns_name
    zone_id                = aws_lb.alb.zone_id
    evaluate_target_health = true
  }
}
```

| 항목 | 값 |
|------|-----|
| `name` | `kimty1121.com` — **루트(apex)** 도메인 |
| `alias.name` | `aws_lb.alb.dns_name` |
| `alias.zone_id` | `aws_lb.alb.zone_id` (ALB용 **고정 zone id**) |

`allow_overwrite = true` — 존에 같은 이름 레코드가 있을 때 apply 충돌 방지(강의·환경에 따라 추가).

---

## 패턴 3 — CNAME 레코드

**다른 DNS 이름**으로 보낼 때 (`www` → apex 등).

```hcl
resource "aws_route53_record" "www" {
  zone_id = aws_route53_zone.main.zone_id
  name    = "www.${var.domain}"
  type    = "CNAME"
  ttl     = 300
  records = [var.domain]
}
```

ALB DNS를 CNAME으로 직접 가리키는 방식도 가능하나, **apex는 CNAME 불가** → ALB는 **Alias A** 권장.

---

## 트래픽 경로 (완성 시)

```
브라우저 → var.domain (Route53 A Alias)
         → aws_lb.alb (80 HTTP 리스너)
         → alb_target_group :3000
         → app_server
```

**88 NS 미설정**이면 Route 53 레코드는 **apply 되어도** 공인 DNS에서 **도메인 조회는 안 됨**.  
Terraform 실습만 할 때는 **코드·apply·콘솔 레코드 확인**으로 충분 (86·87 대화 참고).

---

## `route53.tf`에 추가하는 예

86번 존 아래에 레코드 블록 추가:

```hcl
resource "aws_route53_record" "alb" {
  zone_id = aws_route53_zone.main.zone_id
  name    = var.domain
  type    = "A"

  alias {
    name                   = aws_lb.alb.dns_name
    zone_id                = aws_lb.alb.zone_id
    evaluate_target_health = true
  }
}
```

의존: **존 + ALB**가 먼저 있어야 한다 (`zone_id`, `dns_name` 참조).

| 강의 | 내용 |
|------|------|
| **86** | `aws_route53_zone.main` |

---

## 정리

- **`aws_route53_record`**: `zone_id`로 **86 존** 참조.
- **Alias A**: `ttl`/`records` 없이 **`alias { name, zone_id }`** — ALB는 **`dns_name` + `zone_id`**.
- **IP A / CNAME**: `ttl` + `records` 필수.
- 이 프로젝트: **`var.domain` → `aws_lb.alb`**.
- 다음: **88 도메인 서비스측 NS 등록** (수동).
