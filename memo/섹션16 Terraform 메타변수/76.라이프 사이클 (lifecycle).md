# 76. 라이프 사이클 (`lifecycle`)

Terraform이 코드와 state를 맞추며 리소스에 하는 **Create / Update / Destroy / Replace(삭제 후 재생성)** 동작을, **리소스 단위로 세밀하게 바꾸는** 메타 인수다.

슬라이드: 「リソース操作の詳細制御方法を指定する」— **리소스 조작의 상세 제어 방법**을 지정한다.

이전: **75 `for_each`** · 섹션16 메타 인수 마지막 강의

---

## Terraform 리소스 조작 (슬라이드)

코드·plan 결과에 따라 Terraform은 대략 아래 네 가지로 움직인다.

| 종류 | 설명 |
|------|------|
| **Create** | 클라우드에 **아직 없는** 리소스를 새로 만든다 |
| **Destroy** | 만들어 둔 리소스를 **삭제**한다 |
| **Update** | 이미 있는 리소스의 속성을 **수정**한다(in-place) |
| **Destroy & Re-Create** | 변경이 Update로 안 되어 **한 번 지우고 다시 만든다** (plan의 `must be replaced`) |

`lifecycle`은 이 기본 흐름을 **덮어쓰거나 일부를 무시**할 때 쓴다.

---

## `lifecycle` 블록 위치·문법

`resource` 블록 **안**에 중첩 블록으로 둔다.

```hcl
resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" {
  ...

  lifecycle {
    create_before_destroy = false
    prevent_destroy       = false
    ignore_changes        = [tags]
  }
}
```

| 항목 | 타입 | 설명 |
|------|------|------|
| **`create_before_destroy`** | bool | replace 시 **먼저 새 리소스를 만들고** 나중에 옛것을 지울지 |
| **`prevent_destroy`** | bool | **삭제를 막을지** (`terraform destroy`·replace 포함) |
| **`ignore_changes`** | string[] | 나열한 **속성의 drift·코드 변경을 plan에서 무시** |

슬라이드 기본값 예는 모두 `false` / `tags` 무시 예시이며, 실무에서는 **`true`나 다른 속성**으로 바꿔 쓴다.

---

## `create_before_destroy`

| 값 | 동작 |
|----|------|
| `false` (기본) | replace 시 **삭제 → 생성** (다운타임·이름 충돌 가능 구간) |
| `true` | replace 시 **생성 → 삭제** (이전 리소스가 잠깐 둘 다 존재) |

### 주의 (슬라이드)

**이름이 전역·계정 내에서 유일해야 하는 리소스**(S3 버킷, 일부 SG 이름 등)는  
`create_before_destroy = true` 로 **새 것과 옛것이 동시에 같은 이름**을 쓰려 하면 **유일성 제약 위반**이 난다.

- 대안: replace 시에만 **다른 이름**을 쓰도록 설계, `name_prefix` 사용, 또는 `false` 유지.

**적합한 예**: LB 뒤 ENI, 일부 attachment처럼 **이름이 자동·고유**이거나 replace 시 이름이 바뀌는 타입.

---

## `prevent_destroy`

| 값 | 동작 |
|----|------|
| `false` (기본) | destroy·replace 허용 |
| `true` | 이 리소스가 **삭제되면 plan/apply가 에러**로 중단 |

슬라이드: **RDS처럼 지우고 싶지 않은 것**에 붙인다.

```hcl
resource "aws_db_instance" "mysql_standalone" {
  ...
  lifecycle {
    prevent_destroy = true
  }
}
```

### 이 프로젝트와의 관계

`rds.tf`의 `mysql_standalone`은 AWS 쪽 **`deletion_protection = true`** 로 삭제를 막고 있다.  
Terraform **`prevent_destroy`** 와는 별개다.

| | `deletion_protection` (AWS) | `prevent_destroy` (Terraform) |
|---|---------------------------|-------------------------------|
| 막는 것 | AWS API로 DB 삭제 | `terraform destroy`·코드에서 리소스 제거·replace |
| 설정 위치 | `aws_db_instance` 인자 | `lifecycle` 블록 |

둘 다 쓰면 **콘솔·Terraform 양쪽**에서 실수 삭제를 줄일 수 있다.

---

## `ignore_changes`

plan/apply 시 **지정한 인자의 차이를 무시**한다. 코드·실제·state가 달라도 Terraform이 **그 필드만큼은 update/replace 하지 않는다**.

슬라이드에서 말하는 이용 케이스:

1. **나중에 코드로 맞출 예정이지만, 지금은 drift를 무시**하고 싶을 때  
2. **콘솔·다른 도구·타 팀**이 같은 리소스를 건드려 **Terraform 밖에서 바뀔 수 있을 때**

### `tags` 무시 (슬라이드 예)

```hcl
lifecycle {
  ignore_changes = [tags]
}
```

운영팀이 콘솔에서만 태그를 붙이는 경우, 다음 apply가 태그를 **코드 값으로 되돌리지 않게** 할 때 쓴다.

### RDS 비밀번호 (48번 → 76번 연결)

콘솔·CLI로 master password를 바꾼 뒤, Terraform이 **옛 비밀번호로 되돌리려 하지 않게**:

```hcl
resource "aws_db_instance" "mysql_standalone" {
  password = random_string.db_password.result
  ...

  lifecycle {
    ignore_changes = [password]
  }
}
```

현재 `rds.tf`에는 **아직 `lifecycle` 없음**. 필요 시 위처럼 추가(필드명은 provider 문서 기준).

### EC2 `ami` + `data.aws_ami` (실무에서 자주)

`most_recent = true` 이면 **최신 AMI가 바뀔 때마다** plan에 **replace**가 잡힐 수 있다(61번 plan 메모).

```hcl
resource "aws_instance" "app_server" {
  ami = data.aws_ami.app.id
  ...

  lifecycle {
    ignore_changes = [ami]
  }
}
```

- **의미**: 이미 띄운 인스턴스의 AMI는 Terraform이 **강제로 안 바꿈**.  
- **주의**: 보안 패치·AMI 갱신은 **별도 절차**로 해야 한다.

---

## Replace가 나는 이유 (복습)

`lifecycle` 없이도 provider가 **Update 불가**라고 판단하면 **Destroy & Re-Create**다.

- 예: `aws_instance`의 `ami` 변경 → `must be replaced`
- `create_before_destroy`는 **replace 순서**만 바꾼다. replace 자체를 없애지는 않는다.

---

## 다른 메타 인수와 함께

```hcl
resource "aws_instance" "app_server" {
  count = 1
  ...

  depends_on = [aws_iam_role_policy_attachment.app_iam_role_ssm_managed]

  lifecycle {
    ignore_changes       = [ami]
    prevent_destroy      = false
    create_before_destroy = false
  }
}
```

`depends_on` · `count`/`for_each`와 **같은 `resource` 블록**에 둘 수 있다.

---

## 정리

| 메타 인수 | 역할 |
|-----------|------|
| `create_before_destroy` | replace 시 **생성·삭제 순서** (이름 유일 리소스는 주의) |
| `prevent_destroy` | Terraform 경로 **삭제 방지** (RDS 등) |
| `ignore_changes` | **지정 속성** 변경·drift 무시 (`tags`, `password`, `ami` 등) |

- Terraform 기본 동작: **Create / Update / Destroy / Replace**.
- `lifecycle`은 그중 **삭제·교체·특정 속성 반영**을 조정한다.
- 섹션16: **72 메타 인수 개요 → 73 depends_on → 74 count → 75 for_each → 76 lifecycle**.
