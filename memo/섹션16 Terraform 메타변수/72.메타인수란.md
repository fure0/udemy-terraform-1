# 72. 메타 인수(メタ引数)란

강의·폴더에서는 **메타변수**라고 부르기도 하지만, 슬라이드·Terraform 공식 표현은 **메타 인수(meta-argument)** 에 가깝다.  
**모든 `resource` 블록에 공통으로 쓸 수 있는 특수 속성**으로, AWS 리소스 타입마다 다른 일반 인자(`ami`, `instance_type` 등)와는 별개다.

---

## 어디에 쓰나

슬라이드는 **`resource` 블록 안**을 가리킨다.

```hcl
resource "<RESOURCE_TYPE>" "<RESOURCE_NAME>" {
  # 일반 인자 (리소스 타입별)
  # 메타 인수 (Terraform 공통)
}
```

`variable`, `terraform`, `provider`, `output` 블록이 아니라 **`resource`(및 일부는 `module`)** 안에서 동작한다.

---

## 일반 인자 vs 메타 인수

| 구분 | 예 | 역할 |
|------|-----|------|
| **일반 인자** | `aws_instance`의 `ami`, `subnet_id` | 해당 리소스·프로바이더 스키마에 정의된 설정 |
| **메타 인수** | `depends_on`, `count`, `lifecycle` | **Terraform이 리소스를 만들·갱신·삭제하는 방식**을 제어 |

---

## 지정 가능한 메타 인수 (슬라이드)

| 종류 | 설명 |
|------|------|
| **`depends_on`** | 리소스 **간 의존 관계**를 명시적으로 정의 |
| **`count`** | 지정한 **횟수만큼** 리소스를 복제 |
| **`for_each`** | **맵·세트**를 순회하며 리소스를 복제 |
| **`lifecycle`** | create/update/destroy 시 **동작 규칙** (`ignore_changes`, `create_before_destroy` 등) |
| **`provider`** | 이 리소스만 **다른 provider 설정**을 쓰게 함 |

섹션16 후반 메모에서 항목별로 자세히 다룬다(73~76).

---

## 예시 (개념만)

```hcl
resource "aws_instance" "app" {
  count = 2

  ami           = data.aws_ami.app.id
  instance_type = "t2.micro"

  lifecycle {
    ignore_changes = [ami]
  }

  depends_on = [aws_iam_role_policy_attachment.app_iam_role_ssm_managed]
}
```

---

## 정리

- **메타 인수** = `resource`에 붙이는 **Terraform 공통 제어용** 속성.
- 의존·복수 생성·수명 주기·프로바이더 지정을 코드로 표현할 때 쓴다.
- 다음: **73 `depends_on`**, **74 `count`**, **75 `for_each`**, **76 `lifecycle`**.
