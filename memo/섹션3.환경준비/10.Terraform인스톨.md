## tfenv

- terraform 버전관리
- 버전 변경이 가능

## tfenv 인스톨

- Github 리포지토리 클론
- .bashrc추가

## Terraforn 인스톨

- tfenv를 사용해서 인스톨

## windows의경우

- 유저 경로 에서 진행
  - /C/Users//
- git 리포지토리클론 (GitBash 터미널)
  - git clone [https://github.com/tfutils/tfenv.git](https://github.com/tfutils/tfenv.git) .tfenv
- .bashrc 파일 작성
  - 내용 작성
    - export PATH=$PATH:/C/Users//.packer/bin
- GitBash 터미널을 닫았다가 열어서 반영
  - 처음에는 워닝 나오는데 괜찮음
  - tfenv명령어가 작동하면 정상
- Terraform인스톨 (GitBash 터미널)
  - tfenv list-remote
    - 현재 이용가능한 버전 출력. 최신 버전 이용하면 됨
  - tfenv install 
  - tfenv use 
    - 버전 선택
- 동작확인 (GitBash)
  - terraform version

## Mac의경우

- 유저 경로 에서 진행
  - cd ~
- git 리포지토리클론 (GitBash)
  - git clone [https://github.com/tfutils/tfenv.git](https://github.com/tfutils/tfenv.git) .tfenv
- .zshrc추가
  - export PATH="$HOME/.tfenv/bin:$PATH"
- Terraform인스톨 (GitBash)
  - tfenv list-remote
    - 1.14.8 현재
  - tfenv install 
  - tfenv use 
- 동작확인 (GitBash)
  - terraform version

### tfenv / Terraform 설치가 안 될 때 (Mac)

**1) `command not found: tfenv`**

- `~/.zshrc`에 `export PATH="$HOME/.tfenv/bin:$PATH"` 추가
- 저장 후 `source ~/.zshrc` 또는 터미널 새로 열기
**2) `GNU Grep is a requirement` / `Failed to resolve … version`**
- `brew install grep`
- `~/.zshrc`에 추가 (Apple Silicon):  
`export PATH="/opt/homebrew/opt/grep/libexec/gnubin:$PATH"`
- `source ~/.zshrc` 후 다시 `tfenv install …`
