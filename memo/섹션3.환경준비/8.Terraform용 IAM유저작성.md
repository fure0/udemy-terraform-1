## 필요한 IAM유저와 권한

### AWS CLI 관리용
- 모든리소스에 관한 FullAccess

### Terraform용
- 모든리소스에 관한 FullAccess


## 프로파일
### AWS CLI 초기화
- aws configure
- USER > .aws > credentials
- 프로파일명을 바꿔가면서 사용
```bash
[default]
aws_access_key_id = AKIASKDFSDF
aws_secret_access_key = wAFDSF

[profile1]
aws_access_key_id = AKIASKDFSDF
aws_secret_access_key = wAFDSF
```
- 외부에 유출되면 안됨


## 계정 설정
### AWS CLI 관리용 그리고 Terraform용 생성
- 유저 추가
  - IAM > 사용자 > 사용자 생성
  - 1단계 : 이름지정
  - 2단계 : 직접 정책 연결
    - AdministratorAccess 선택
  - 3단계 : 태그 지정 없음
- IAM > 사용자 > 사용할 아이디
  - 보안 자격 증명 탭
  - 액세스 키 만들기
    - 한번밖에 표시 안됨

## configure등록
```bash
## 첫번째 등록 CLI 관리용
aws configure

AWS Access key ID
AWS secret Access Key
Default region name: ap-northeast-1
Default output format: jsom

## 두번쨰 등록 Terraform용
aws configure --profile terraform
AWS Access key ID
AWS secret Access Key
Default region name: ap-northeast-1
Default output format: jsom
```

### 등록확인
```bash
cat ~/.aws/credentials
```
