# 84. 리스너 작성

ALB가 **어느 포트·프로토콜**로 요청을 받아 **어느 타겟 그룹**으로 넘길지 정하는 리소스다.  
**`aws_lb_listener`** 가 **82 ALB** 와 **83 타겟 그룹**을 이어 주면, 인터넷 → ALB → EC2:3000 경로가 완성된다.

이전: **82 ALB** · **83 타겟 그룹** · 섹션19 **마지막**

---

## 트래픽 흐름 (슬라이드)

```
인터넷
  ↓
Security group (web_sg)
  ↓
Public subnet
  ↓
로드 밸런서 (ALB)  ← 리스너가 80/443 수신 규칙
  ↓
타겟 그룹 (TG)
  ↓
EC2 (app_server :3000)
```

| 구간 | 이 프로젝트 |
|------|-------------|
| 진입 | `web_sg` — 80·443 **0.0.0.0/0** 인바운드 |
| ALB 위치 | `public_subnet_1a` / `1c` |
| 백엔드 | `alb_target_group` — **HTTP 3000**, `app_server` 등록됨 |

리스너가 없으면 ALB DNS로 접속해도 **처리 규칙이 없어** 응답이 없다.

---

## ELB 데이터 구조 — 리스너 위치 (슬라이드)

```
로드 밸런서 (aws_lb.alb)
    ↑
리스너 (aws_lb_listener) ──→ 타겟 그룹 (aws_lb_target_group.alb_target_group)
                                    ↑
              타겟 그룹 attachment (instance)
                    ↑
              EC2 (aws_instance.app_server)
```

| 리소스 | 참조 대상 | 코드 |
|--------|-----------|------|
| **리스너** | LB | `load_balancer_arn = aws_lb.alb.arn` |
| **리스너** | TG | `default_action.target_group_arn = aws_lb_target_group.alb_target_group.arn` |

리스너는 **LB·TG 양쪽을 참조**하므로, plan 순서는 **LB·TG 생성 후 리스너**가 된다.

---

## `aws_lb_listener` — HTTP (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`load_balancer_arn`** * | string | 로드 밸런서 **ARN** |
| **`port`** | number | 포트 (HTTP는 **80**) |
| **`protocol`** | enum | **`"HTTP"`** |
| **`default_action`** * | block | 규칙에 맞지 않을 때 **기본 동작** |

### `default_action` 블록

| 항목 | 타입 | 설명 |
|------|------|------|
| **`type`** * | enum | `"forward"`, `"redirect"` 등 — TG로 보낼 때 **`forward`** |
| **`target_group_arn`** | string | `type = "forward"` 일 때 **전달 대상 TG ARN** |

`*` = 필수.

### HCL 예시 (HTTP 80)

```hcl
resource "aws_lb_listener" "http" {
  load_balancer_arn = aws_lb.alb.arn
  port              = 80
  protocol          = "HTTP"

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.alb_target_group.arn
  }
}
```

브라우저에서 **`http://<ALB DNS>`** 로 접근할 때 사용한다.

---

## `aws_lb_listener` — HTTPS (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`load_balancer_arn`** * | string | 로드 밸런서 ARN |
| **`port`** | number | **443** |
| **`protocol`** | enum | **`"HTTPS"`** |
| **`ssl_policy`** | enum | SSL 정책. 예: `"ELBSecurityPolicy-2016-08"` |
| **`certificate_arn`** | string | **ACM** 인증서 ARN |
| **`default_action`** * | block | HTTP와 동일 (`type`, `target_group_arn`) |

HTTPS는 **ACM 인증서**가 있어야 한다. 강의에서 HTTP만 실습하면 **위 HTTP 리스너만** 추가해도 된다.

### HCL 예시 (HTTPS 443 — 참고)

```hcl
resource "aws_lb_listener" "https" {
  load_balancer_arn = aws_lb.alb.arn
  port              = 443
  protocol          = "HTTPS"
  ssl_policy        = "ELBSecurityPolicy-2016-08"
  certificate_arn   = aws_acm_certificate.example.arn  # 강의·환경에 맞게

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.alb_target_group.arn
  }
}
```

`web_sg` 에 **443 인바운드**는 이미 있다 (`security_group.tf`).

---

## HTTP vs HTTPS (정리)

| | HTTP | HTTPS |
|---|------|-------|
| **port** | 80 | 443 |
| **protocol** | `"HTTP"` | `"HTTPS"` |
| **추가 인자** | 없음 | `ssl_policy`, `certificate_arn` |
| **default_action** | `forward` → TG | 동일 |

같은 TG(`alb_target_group`)로 **80·443 둘 다** forward 할 수 있다.

---

## 이 프로젝트 `elb.tf`와 맞추기

현재 코드:

| 리소스 | Terraform 이름 |
|--------|----------------|
| ALB | `aws_lb.alb` |
| TG | `aws_lb_target_group.alb_target_group` |
| attachment | `aws_lb_target_group_attachment.instance` |

`elb.tf` 에 **리스너 블록 추가** 예:

```hcl
# ------------
# listener
# ------------

resource "aws_lb_listener" "http" {
  load_balancer_arn = aws_lb.alb.arn
  port              = 80
  protocol          = "HTTP"

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.alb_target_group.arn
  }
}
```

## apply 후 확인

```bash
cd terraform
terraform apply
terraform output   # ALB DNS를 output에 넣었다면
# 또는 AWS 콘솔 EC2 → 로드 밸런서 → DNS 이름
```

| 확인 | 내용 |
|------|------|
| **plan** | `aws_lb_listener.http` **add** |
| **접속** | `http://<alb-dns-name>` → AP(3000) 응답 (앱 기동·헬스체크 OK일 때) |
| **502** | TG에 EC2 미등록, 앱 미기동, SG 3000 불일치 등 점검 |

ALB **DNS 이름**은 `aws_lb.alb.dns_name` 으로 참조할 수 있다 (`output` 블록에 넣기 좋음).

## 섹션19 전체 흐름

| 강의 | 리소스 | 역할 |
|------|--------|------|
| **82** | `aws_lb` | ALB 본체 |
| **83** | `aws_lb_target_group` + `attachment` | 백엔드 풀·EC2 등록 |
| **84 (지금)** | `aws_lb_listener` | **80/443 수신 → TG forward** |

세 가지가 있어야 **인터넷 → ALB → app_server:3000** 이 동작한다.

---

## 정리

- **리스너** = ALB에서 **port + protocol** 로 받아 **`default_action`** 으로 TG에 전달.
- HTTP: **80**, `protocol = "HTTP"`, `type = "forward"`, `target_group_arn`.
- HTTPS: **443** + **`ssl_policy`**, **`certificate_arn`** (ACM).
- 이 저장소: `aws_lb.alb` + `aws_lb_target_group.alb_target_group` 참조.
- 섹션19 ELB 부하 분산 설정 **완료**.
