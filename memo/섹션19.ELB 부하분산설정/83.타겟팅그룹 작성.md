# 83. 타겟 그룹 작성

ALB가 받은 트래픽을 **어느 EC2·어느 포트**로 보낼지 정의하는 리소스다.  
**`aws_lb_target_group`** 로 TG를 만들고, **`aws_lb_target_group_attachment`** 로 **EC2를 TG에 등록**한다.

82번에서 만든 **`aws_lb.alb`** 다음 단계. 84번 **리스너**가 이 TG를 가리키면 HTTP 경로가 완성된다.

이전: **82 ALB** · 다음: **84 리스너**

---

## ELB 구조에서 타겟 그룹 위치 (슬라이드)

슬라이드는 **타겟 그룹**을 중심으로 참조 관계를 그린다.

```
로드 밸런서 (aws_lb)
    ↑
리스너 (aws_lb_listener) ──→ 타겟 그룹 (aws_lb_target_group)  ← 83번
                                    ↑
              타겟 그룹 attachment ──┘
                    ↑
              EC2 (aws_instance.app_server)
```

| 리소스 | 참조 대상 | 코드 |
|--------|-----------|------|
| **리스너** (84) | LB | `load_balancer_arn = aws_lb.alb.arn` |
| **리스너** (84) | TG | `target_group_arn = aws_lb_target_group.xxx.arn` |
| **attachment** (83) | TG | `target_group_arn = aws_lb_target_group.xxx.arn` |
| **attachment** (83) | EC2 | `target_id = aws_instance.app_server.id` |

TG는 **LB와 직접 참조하지 않는다**. **리스너**가 LB와 TG를 이어 준다.

---

## `aws_lb_target_group` 주요 인자 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** | string | 타겟 그룹 이름 |
| **`port`** | number | 트래픽을 보낼 **포트 번호** |
| **`protocol`** | enum | `"HTTP"`, `"HTTPS"`, `"TCP"`, `"UDP"`, `"TCP_UDP"` |
| **`vpc_id`** | string | **VPC ID** (슬라이드 표기는 `string[]` 이지만 provider에서는 **string** 1개) |
| **`tags`** | map | 태그 |

ALB + HTTP 앱이면 **`protocol = "HTTP"`**, AP 포트 **`3000`** 이 이 프로젝트와 맞다 (`app_sg` 3000, `web_sg` → `app_sg`).

---

## `aws_lb_target_group_attachment` 주요 인자 (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`target_group_arn`** * | string | 등록할 **타겟 그룹 ARN** |
| **`target_id`** * | string | 등록할 대상 ID — **EC2 인스턴스 ID** |

`*` = 필수. EC2 1대면 attachment 리소스 **1개**. 여러 대면 attachment를 늘리거나 `for_each`/`count`로 반복(75·74번).

---

## 이 프로젝트에 맞는 값

| 항목 | 값 | 이유 |
|------|-----|------|
| **`port`** | `3000` | tastylog AP (Node 등) |
| **`protocol`** | `"HTTP"` | ALB L7, 리스너도 HTTP(84) |
| **`vpc_id`** | `aws_vpc.vpc.id` | TG는 VPC 소속 |
| **`target_id`** | `aws_instance.app_server.id` | `appserver.tf` AP EC2 |

보안: **`web_sg`** 가 **`app_sg` 3000** 으로 나가고, **`app_sg`** 가 **`web_sg`에서 3000** 인바운드를 허용한다. ALB ENI는 **web_sg**를 쓰므로 TG → EC2:3000 경로와 SG가 맞는다.

---

## HCL 예시 (`elb.tf`에 추가)

현재 ALB 리소스명은 **`aws_lb.alb`** (`elb.tf`).

```hcl
resource "aws_lb_target_group" "app" {
  name     = "${var.project}-${var.environment}-app-tg"
  port     = 3000
  protocol = "HTTP"
  vpc_id   = aws_vpc.vpc.id

  tags = {
    Name    = "${var.project}-${var.environment}-app-tg"
    Project = var.project
    Env     = var.environment
  }
}

resource "aws_lb_target_group_attachment" "app" {
  target_group_arn = aws_lb_target_group.app.arn
  target_id        = aws_instance.app_server.id
}
```

| 리소스 | 이름 예 |
|--------|---------|
| TG | `tastylog-dev-app-tg` |
| attachment | TG ARN + `app_server` 인스턴스 ID |

---

## 의존 관계

| 관계 | 감지 |
|------|------|
| TG → VPC | `vpc_id = aws_vpc.vpc.id` (암시적) |
| attachment → TG | `target_group_arn = ...` (암시적) |
| attachment → EC2 | `target_id = aws_instance.app_server.id` (암시적) |

EC2가 **replace** 되면 attachment도 **같은 `target_id` 참조**로 plan에 잡힐 수 있다. 82번 plan에서 AMI 교체가 같이 뜬 경우, apply 후 **새 인스턴스 ID**로 attachment가 맞춰진다.

---

## apply 전 확인

```bash
cd terraform
terraform plan
```

기대:

- **add**: `aws_lb_target_group.app`, `aws_lb_target_group_attachment.app`
- (82 미적용 시) `aws_lb.alb` 도 함께 add

**리스너(84) 전**에는 ALB DNS로 접속해도 **라우팅 규칙이 없어** 응답이 없을 수 있다. 83은 **TG + EC2 등록**까지.

---

## 82·84와 역할 분담

| 강의 | 리소스 | 역할 |
|------|--------|------|
| **82** | `aws_lb` | ALB 본체 |
| **83 (지금)** | `aws_lb_target_group` | **백엔드 풀** (포트·프로토콜·VPC) |
| **83 (지금)** | `aws_lb_target_group_attachment` | **EC2를 풀에 등록** |
| **84** | `aws_lb_listener` | **80/443** 수신 → TG로 forward |

TG만 있고 리스너가 없으면 트래픽이 TG에 **도달하지 않는다**. attachment만 있고 앱이 3000에서 안 뜨면 **헬스체크 실패·502** 가능(헬스체크 상세는 강의·AWS 콘솔 참고).

---

## 정리

- **타겟 그룹** = 트래픽 **목적지(포트·프로토콜·VPC)** 정의 — `aws_lb_target_group`.
- **attachment** = **EC2 인스턴스 ID**를 TG ARN에 연결 — `aws_lb_target_group_attachment`.
- 이 프로젝트: **HTTP 3000**, `vpc_id`, `app_server` 등록.
- 다음: **84 리스너** — `aws_lb.alb` + `aws_lb_target_group.app` 참조.
