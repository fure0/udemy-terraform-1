# 100. 기동 템플릿 리소스

섹션24는 **EC2 Auto Scaling** 을 Terraform으로 만든다.  
**100번**은 **기동 템플릿(Launch Template)** 관련 **리소스·data 구조와 인자 개요** — **101 기동 템플릿 작성** · **102·103 Auto Scaling Group** 실습의 기반.

슬라이드: **起動テンプレートのデータ構造**

이전: **99 CloudFront S3 오리진** · 섹션23 마지막  
다음: **101 기동 템플릿 작성** · **102 ASG 리소스**

---

## 기동 템플릿 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (73·94·97번과 동일).

```
기동 템플릿 (aws_launch_template)
    ↗        ↗         ↗           ↗
  AMI    키페어   보안 그룹      IAM Role
(data)  (resource) (resource)   (resource)
```

| 구성 | Terraform | 참조 |
|------|-----------|------|
| **기동 템플릿** | `aws_launch_template` | (루트) |
| **AMI** | **`data.aws_ami.app`** | `image_id` |
| **키 페어** | **`aws_key_pair.keypair`** | `key_name` |
| **보안 그룹** | **`aws_security_group.app_sg`** 등 | `network_interfaces.security_groups` |
| **IAM Role** | **`aws_iam_instance_profile.app_ec2_profile`** | `iam_instance_profile` 블록 |

기존 **`aws_instance.app_server`** (appserver.tf) 와 **같은 구성 요소**를 템플릿으로 **재사용**한다 (101).

---

## Launch Template 관련 리소스 한눈에

| 리소스 / data | 역할 |
|---------------|------|
| **`aws_launch_template`** | ASG가 띄울 EC2 **설정 템플릿** (버전 관리) |
| **`data.aws_ami.app`** | 최신 Amazon Linux 2 AMI 조회 (data.tf) |
| **`aws_key_pair.keypair`** | SSH 키 (appserver.tf) |
| **`aws_security_group.app_sg`** · **`opmng_sg`** | 앱·운영 SG (security_group.tf) |
| **`aws_iam_instance_profile.app_ec2_profile`** | EC2 Role 프로필 (iam.tf) |

현재 `terraform/` 에 **`aws_launch_template` 은 아직 없음** — 101에서 추가.

## `aws_launch_template` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`update_default_version`** | bool | 변경 시 **기본 버전을 최신으로 자동 갱신** |
| **`name`** | string | 기동 템플릿 **이름** |
| **`image_id`** | string | **AMI** ID |
| **`instance_type`** | enum | **인스턴스 타입** |
| **`key_name`** | string | **키 페어** 이름 |
| **`block_device_mappings`** | block | **디스크(EBS)** 설정 |
| **`tag_specifications`** | block | 기동 인스턴스에 붙일 **태그** |
| **`network_interfaces`** | block | **네트워크** 설정 |
| **`iam_instance_profile`** | block | **IAM Role** (인스턴스 프로필) |
| **`user_data`** | string | **부팅 시 스크립트** (base64) |

### `update_default_version` (슬라이드 주석)

| 설정 | 의미 |
|------|------|
| **`update_default_version = true`** | 템플릿 수정 시 **default 버전이 최신으로** |
| **버전 고정** | ASG에서 **`launch_template` + `version`** 으로 **특정 버전 지정** (102·103) |

슬라이드: **バージョン固定したい場合 default_version に設定** — ASG 쪽에서 버전을 고정할 수 있다.

### `instance_type` (슬라이드 주석)

슬라이드: **オートスケーリンググループで上書きできる = 指定しなくても良い**

| | Launch Template | Auto Scaling Group (102) |
|---|-----------------|--------------------------|
| **`instance_type`** | 템플릿에 지정 가능 | **ASG에서 덮어쓰기 가능** → **생략 가능** |

실습에서는 템플릿에 **`t2.micro`** 를 두는 경우가 많다 (`app_server` 와 동일).

---

## `block_device_mappings` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`device_name`** | string | **마운트 경로** (예: `/dev/xvda`) |
| **`ebs`** | block | EBS 볼륨 설정 |

### `ebs` 블록

| 항목 | 타입 | 설명 |
|------|------|------|
| **`delete_on_termination`** | bool | 인스턴스 종료 시 **볼륨 삭제** 여부 |
| **`volume_type`** | enum | `"gp2"`, `"gp3"`, `"standard"` 등 |
| **`volume_size`** | number | **용량(GB)** |

AMI 루트 디스크만 쓰면 **생략** 가능. 강의에서 명시하면 위 블록 사용.

---

## `tag_specifications` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`resource_type`** | enum | `"instance"`, `"volume"` 등 |
| **`tags`** | map | **태그** key-value |

`aws_instance.app_server` 의 `tags` 와 동일 패턴을 **instance** 에 지정:

```hcl
tag_specifications {
  resource_type = "instance"

  tags = {
    Name    = "${var.project}-${var.environment}-app-ec2"
    Project = var.project
    Env     = var.environment
    Type    = "app"
  }
}
```

---

## `network_interfaces` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`associate_public_ip_address`** | bool | **퍼블릭 IP** 부여 여부 |
| **`security_groups`** | string[] | **보안 그룹** ID 목록 |
| **`delete_on_termination`** | bool | 인스턴스 종료 시 **ENI 삭제** |
| **`description`** | string | 네트워크 인터페이스 **설명** |

`aws_instance` 의 `subnet_id` + `associate_public_ip_address` + `vpc_security_group_ids` 를 **network_interfaces** 로 옮기는 패턴 (101).

| 이 프로젝트 (`app_server`) | Launch Template |
|-----------------------------|-----------------|
| `subnet_id` | ASG **vpc_zone_identifier** (103) — 템플릿에는 SG·퍼블릭 IP |
| `associate_public_ip_address = true` | `network_interfaces.associate_public_ip_address = true` |
| `app_sg`, `opmng_sg` | `security_groups = [app_sg.id, opmng_sg.id]` |

---

## `iam_instance_profile` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`arn`** | string | 인스턴스 프로필 **ARN** |
| **`name`** | string | 인스턴스 프로필 **이름** |

**`arn` 또는 `name` 중 하나** — 이 프로젝트:

```hcl
iam_instance_profile {
  name = aws_iam_instance_profile.app_ec2_profile.name
}
```

`aws_instance` 의 `iam_instance_profile = ...name` 과 동일 (12·13 IAM).

---

## `user_data` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`user_data`** | string | 부팅 시 실행 스크립트 — Terraform에서는 **`base64encode()`** |

앱 설치·SSM Parameter 읽기 등 — **101** 에서 `app_server` 와 맞춰 추가 여부 결정.

---

## 이 프로젝트와의 관계

| 항목 | 현재 (`appserver.tf` 등) |
|------|-------------------------|
| **단일 EC2** | `aws_instance.app_server` — ASG 전환 전 |
| **AMI** | `data.aws_ami.app` (data.tf) |
| **키 페어** | `aws_key_pair.keypair` |
| **SG** | `app_sg`, `opmng_sg` |
| **IAM** | `app_ec2_profile` |
| **ALB 타겟** | `aws_lb_target_group_attachment.instance` → **103에서 ASG 연동** |

101 이후 **단일 EC2·attachment** 를 ASG·Launch Template 로 **대체**하는 흐름 (강의).

---

## HCL 개념 예 (101 미리보기)

```hcl
resource "aws_launch_template" "app" {
  name                   = "${var.project}-${var.environment}-app-lt"
  image_id               = data.aws_ami.app.id
  instance_type          = "t2.micro"
  key_name               = aws_key_pair.keypair.key_name
  update_default_version = true

  iam_instance_profile {
    name = aws_iam_instance_profile.app_ec2_profile.name
  }

  network_interfaces {
    associate_public_ip_address = true
    security_groups = [
      aws_security_group.app_sg.id,
      aws_security_group.opmng_sg.id,
    ]
  }

  tag_specifications {
    resource_type = "instance"
    tags = {
      Name    = "${var.project}-${var.environment}-app-ec2"
      Project = var.project
      Env     = var.environment
      Type    = "app"
    }
  }
}
```

## apply 시 순서 (암시적 의존)

1. **AMI data** · **keypair** · **SG** · **IAM profile** (기존)
2. **`aws_launch_template`** (101)
3. **`aws_autoscaling_group`** (103) — `launch_template { id, version }`

---

## 주의

| 항목 | 내용 |
|------|------|
| **AMI `most_recent`** | plan 시 **replace** 유발 가능 — 72 lifecycle·고정 AMI |
| **템플릿 버전** | 수정마다 **새 버전** — ASG가 **$Latest** vs **고정 번호** |
| **`instance_type` 생략** | ASG **mixed instances** 등에서 덮어쓰기 — 102 |
| **단일 EC2와 중복** | 101·103 후 **`app_server` 제거** 여부는 강의 순서 따름 |

---

## 정리

- Launch Template = **AMI(data) + 키페어 + SG + IAM** 을 **한 템플릿**으로 묶음.
- 슬라이드 블록: **`block_device_mappings` · `tag_specifications` · `network_interfaces` · `iam_instance_profile` · `user_data`**.
- **`update_default_version`** / ASG **`version`** 으로 버전 관리.
- 다음: **101 기동 템플릿 작성**.
