# 102. 오토 스케일링 그룹 리소스

섹션24는 **EC2 Auto Scaling** 을 Terraform으로 만든다.  
**102번**은 **Auto Scaling Group(ASG)** 관련 **리소스 구조와 인자 개요** — **103 오토 스케일링 그룹 작성** 실습의 기반.

슬라이드: **AutoScalingGroupのデータ構造**

이전: **101 기동 템플릿 작성**  
다음: **103 오토 스케일링 그룹 작성**

---

## Auto Scaling Group 데이터 구조 (슬라이드)

참조: **참조 원 → 참조 대상** (73·94·97·100번과 동일).

```
오토 스케일링 그룹 (aws_autoscaling_group)
    ↗              ↗                    ↗
 서브넷      ELB 타겟 그룹          기동 템플릿
(resource)     (resource)           (resource)
```

| 구성 | Terraform | 참조 |
|------|-----------|------|
| **오토 스케일링 그룹** | `aws_autoscaling_group` | (루트) |
| **서브넷** | **`aws_subnet.public_subnet_1a`** · **`public_subnet_1c`** | `vpc_zone_identifier` |
| **ELB 타겟 그룹** | **`aws_lb_target_group.alb_target_group`** | `target_group_arns` |
| **기동 템플릿** | **`aws_launch_template.app_lt`** | `launch_template { id, version }` |

ASG가 **어디에** · **몇 대** · **어떤 설정으로** EC2를 띄울지 정하고, ALB 타겟 그룹에 **자동 등록**한다.  
단일 EC2 + `aws_lb_target_group_attachment` (elb.tf, 주석 처리됨) 를 **ASG로 대체**하는 흐름 (103).

## ASG 관련 리소스 한눈에

| 리소스 | 역할 |
|--------|------|
| **`aws_autoscaling_group`** | EC2 **대수·가용 영역·헬스체크** 관리, Launch Template 기반 기동 |
| **`aws_subnet.public_subnet_*`** | ASG가 인스턴스를 띄울 **서브넷** (network.tf) |
| **`aws_lb_target_group.alb_target_group`** | ALB가 트래픽을 보낼 **타겟 그룹** (elb.tf) |
| **`aws_launch_template.app_lt`** | 인스턴스 **기동 설정** (appserver.tf, 101) |

현재 `terraform/` 에 **`aws_autoscaling_group` 은 아직 없음** — 103에서 추가.

## `aws_autoscaling_group` (슬라이드)

| 항목 | 타입 | 설명 |
|------|------|------|
| **`name`** | string | Auto Scaling Group **이름** |
| **`max_size` \*** | number | **최대** 인스턴스 수 |
| **`min_size` \*** | number | **최소** 인스턴스 수 |
| **`desired_capacity`** | number | **희망** 인스턴스 수 |
| **`capacity_rebalance`** | bool | **용량 리밸런싱** 활성화 여부 |
| **`health_check_grace_period`** | number | 헬스체크 **시작 전 대기 시간**(초) |
| **`health_check_type`** | enum | `"EC2"` 또는 `"ELB"` |
| **`vpc_zone_identifier` \*** | string[] | **서브넷** ID 목록 |
| **`target_group_arns`** | string[] | **ELB 타겟 그룹** ARN 목록 |
| **`mixed_instances_policy` (\*)** | block | 혼합 인스턴스 타입 정책 |

### 필수 조건 (슬라이드 각주)

**`launch_configuration` · `launch_template` · `mixed_instances_policy` 중 하나는 반드시 지정.**

| 방식 | 비고 |
|------|------|
| **`launch_template`** | **권장** — 이 강의·프로젝트 (101 `app_lt`) |
| **`launch_configuration`** | 레거시 — 신규는 Launch Template 사용 |
| **`mixed_instances_policy`** | 여러 인스턴스 타입·Spot 혼합 — `instance_type` 을 ASG에서 **덮어쓰기** (100번) |

## `launch_template` 블록 (슬라이드)

ASG 본문에 넣는 **`launch_template` 블록** (101의 `aws_launch_template` 리소스와 구분).

| 항목 | 타입 | 설명 |
|------|------|------|
| **`id`** | string | 기동 템플릿 **ID** |
| **`version`** | string | 템플릿 **버전** — `"$Latest"`, `"$Default"`, 또는 **숫자** |

```hcl
launch_template {
  id      = aws_launch_template.app_lt.id
  version = "$Latest"
}
```

| `version` 값 | 의미 |
|--------------|------|
| **`"$Latest"`** | 항상 **최신** 템플릿 버전 사용 |
| **`"$Default"`** | **기본(default)** 버전 사용 (`update_default_version = true` 와 연동, 100) |
| **숫자** (예: `"1"`) | **특정 버전 고정** — 변경 시 ASG를 명시적으로 갱신 |

---

## `mixed_instances_policy` · `launch_template` 블록 (슬라이드)

여러 인스턴스 타입을 쓸 때. **`launch_template` 블록** 안에 중첩.

| 항목 | 타입 | 설명 |
|------|------|------|
| **`launch_template` \*** | block | 기동 템플릿 지정 |
| **`launch_template_specification` \*** | block | 기본 템플릿 |
| **`launch_template_id`** | string | 기동 템플릿 **ID** |
| **`version`** | string | `"$Latest"` · `"$Default"` · 숫자 |
| **`override`** | block | 템플릿 값 **덮어쓰기** |
| **`instance_type`** | enum | **인스턴스 타입** (템플릿 `instance_type` 대체) |

100번 슬라이드: Launch Template의 **`instance_type` 은 ASG에서 덮어쓸 수 있어 생략 가능** — `override` 로 지정.

```hcl
mixed_instances_policy {
  launch_template {
    launch_template_specification {
      launch_template_id = aws_launch_template.app_lt.id
      version            = "$Latest"
    }

    override {
      instance_type = "t2.micro"
    }
  }
}
```

이 프로젝트 실습(103)은 단순 **`launch_template` 블록** 만으로 충분한 경우가 많다.

## 주요 인자 상세

### 용량 (`min_size` · `max_size` · `desired_capacity`)

| 항목 | 역할 |
|------|------|
| **`min_size`** | ASG가 **유지할 최소** 대수 |
| **`max_size`** | **늘릴 수 있는 최대** 대수 |
| **`desired_capacity`** | **현재 목표** 대수 (생략 시 `min_size` 와 동일하게 동작하는 경우 많음) |

스케일링 정책(이후 강의) 없이도 **desired** 만으로 고정 대수 운영 가능.

### 헬스체크 (`health_check_type` · `health_check_grace_period`)

| 항목 | 값 | 설명 |
|------|-----|------|
| **`health_check_type`** | `"EC2"` | EC2 상태만 확인 (기본) |
| | `"ELB"` | **ALB 타겟 그룹** 헬스체크 — `target_group_arns` 와 함께 사용 |
| **`health_check_grace_period`** | 초 | 기동 직후 **앱 준비 시간** — 이 시간 동안은 unhealthy 로 종료하지 않음 |

ALB 뒤 앱 서버(포트 3000)이면 **`health_check_type = "ELB"`** + grace period 권장 (103).

### 네트워크 · 로드밸런서

| 항목 | 이 프로젝트 |
|------|-------------|
| **`vpc_zone_identifier`** | `[aws_subnet.public_subnet_1a.id, aws_subnet.public_subnet_1c.id]` — ALB와 동일 AZ |
| **`target_group_arns`** | `[aws_lb_target_group.alb_target_group.arn]` — **attachment 리소스 불필요** |

ASG에 타겟 그룹을 연결하면 새 인스턴스가 **자동으로 타겟에 등록**된다.

---

## 이 프로젝트와의 관계

| 항목 | 현재 (`terraform/`) |
|------|---------------------|
| **단일 EC2** | `aws_instance.app_server` — **주석 처리** (appserver.tf) |
| **타겟 등록** | `aws_lb_target_group_attachment.instance` — **주석 처리** (elb.tf) |
| **기동 템플릿** | `aws_launch_template.app_lt` — **작성 완료** (101) |
| **ALB·타겟 그룹** | `aws_lb.alb`, `aws_lb_target_group.alb_target_group` (elb.tf) |
| **서브넷** | `public_subnet_1a`, `public_subnet_1c` (network.tf) |
| **ASG** | **미작성** — 103에서 추가 |

| Before (단일 EC2) | After (ASG) |
|-------------------|-------------|
| `subnet_id` on instance | `vpc_zone_identifier` on ASG |
| `aws_lb_target_group_attachment` | `target_group_arns` on ASG |
| `aws_instance` | `aws_launch_template` + `aws_autoscaling_group` |

## HCL 개념 예 (103 미리보기)

```hcl
resource "aws_autoscaling_group" "app_asg" {
  name                = "${var.project}-${var.environment}-app-asg"
  vpc_zone_identifier = [
    aws_subnet.public_subnet_1a.id,
    aws_subnet.public_subnet_1c.id,
  ]
  target_group_arns = [aws_lb_target_group.alb_target_group.arn]

  min_size         = 1
  max_size         = 3
  desired_capacity = 1

  health_check_type         = "ELB"
  health_check_grace_period = 300

  launch_template {
    id      = aws_launch_template.app_lt.id
    version = "$Latest"
  }

  tag {
    key                 = "Name"
    value               = "${var.project}-${var.environment}-app-ec2"
    propagate_at_launch = true
  }

  tag {
    key                 = "Project"
    value               = var.project
    propagate_at_launch = true
  }

  tag {
    key                 = "Env"
    value               = var.environment
    propagate_at_launch = true
  }

  tag {
    key                 = "Type"
    value               = "app"
    propagate_at_launch = true
  }
}
```

Launch Template `tag_specifications` 와 ASG `tag` (`propagate_at_launch = true`) 를 **맞추는** 패턴이 일반적.

## 주의

| 항목 | 내용 |
|------|------|
| **Launch Template 버전** | `$Latest` 는 템플릿 변경 시 **새 인스턴스만** 새 버전 — 기존 인스턴스는 **instance refresh** 등으로 갱신 |
| **`health_check_type = "ELB"`** | 타겟 그룹 **헬스체크 경로·포트** 가 앱(3000)과 맞아야 함 — 미설정 시 103에서 추가 |
| **서브넷 AZ** | `vpc_zone_identifier` 는 **ALB 서브넷과 겹치는 퍼블릭 서브넷** — 멀티 AZ |
| **중복 리소스** | ASG + `app_server` 동시 기동 시 **이중 등록·비용** — 하나만 유지 |
| **`mixed_instances_policy`** | Spot·혼합 타입이 아니면 **단순 `launch_template` 블록** 으로 충분 |

---

## 정리

- ASG = **서브넷 + 타겟 그룹 + 기동 템플릿** 을 묶어 **대수·헬스·등록** 을 자동화.
- 필수: **`min_size` · `max_size` · `vpc_zone_identifier`** + **`launch_template`** (또는 launch_configuration / mixed_instances_policy).
- ALB 연동: **`target_group_arns`** + **`health_check_type = "ELB"`**.
- 다음: **103 오토 스케일링 그룹 작성**.
