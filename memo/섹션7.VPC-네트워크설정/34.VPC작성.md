# 34. VPC 작성

VPC(가상 사설 클라우드)를 만들 때 자주 쓰는 인자입니다. 강의 슬라이드의 표를 옮겨 적었습니다.

| 항목 | 타입 | 설명 |
| :--- | :--- | :--- |
| **cidr_block** * | string | IPv4 CIDR 블록 (VPC 주소 범위). **필수** |
| **assign_generated_ipv6_cidr_block** | bool | AWS가 IPv6 CIDR를 자동 할당할지 여부. (슬라이드에는 타입이 string으로 적혀 있을 수 있으나, Terraform AWS Provider에서는 bool) |
| **instance_tenancy** | enum | 테넌시. `"default"` 또는 `"dedicated"` |
| **enable_dns_support** | bool | VPC 내부 DNS 해결(Amazon 제공 DNS) 사용 여부 |
| **enable_dns_hostnames** | bool | 퍼블릭 IPv4를 가진 인스턴스에 DNS 호스트 이름 부여 여부 |
| **tags** | object(map) | Name, Project 등 태그 |

`*` 표시는 슬라이드 기준으로 **필수 인자**를 뜻합니다.

## 실습 코드 위치

이 강의의 VPC 정의는 저장소의 **`terraform/network.tf`** 에 있습니다. 아래 `aws_vpc` 블록에서 표의 항목이 어떻게 들어가는지 비교해 보면 됩니다.

```5:17:terraform/network.tf
resource "aws_vpc" "vpc" {
  cidr_block                       = "10.0.0.0/20"
  instance_tenancy                 = "default"
  enable_dns_support               = true
  enable_dns_hostnames             = true
  assign_generated_ipv6_cidr_block = false

  tags = {
    Name    = "${var.project}-${var.environment}-vpc"
    Project = var.project
    Env     = var.environment
  }
}
```
